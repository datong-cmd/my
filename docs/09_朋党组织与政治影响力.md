# 09 《变身大明》朋党组织与政治影响力

> 参考机制解析。新项目读取既有党争，不创建第二套人物党籍或通用 faction。

## 1. 现有载体

- `pengdang`、`dangkui`、`dangtu` 等 scripted relation；
- `join_pengdang`、`pengdang_leave` 等内部 effect；
- `pengdang_yingxiang_score` 及职位权重；
- 党魁继承、拉拢、退出、权臣阈值和相关事件。

前置的党派影响不是单纯人数：中枢、地方和军事职位有不同权重，党魁与高官可以放大影响。

## 2. 可读取内容

- 当前党魁和少量代表；
- 人物是否具有已验证党争 relation；
- `pengdang_yingxiang_score` 作为临时修正；
- 党魁官位、与主持者/皇帝关系和性格标签。

每项政策最多读取四名代表，不重建全党成员表。

## 3. 风险

- 关系网若按成员两两扫描会平方增长；
- 某些全局作用域和党魁继承流程可能串台；
- 影响力缓存存在更新滞后，不能视为绝对实时事实；
- 前置党派主要是宫廷组织，不等于完整社会阶级。

## 4. 新项目边界

- 不调用加入/退出党派 effect，不增删党籍，不授予党魁；
- 不覆盖 `GM_Pengdang_events` 或 `common/factions/00_factions.txt`；
- 全国八项值表达社会政治力量，既有朋党只作为朝堂放大器；
- 政策结束清理临时 agenda/support/saved scope，不写回上游。
