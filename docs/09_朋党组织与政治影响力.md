# 朋党组织、政治影响力与权臣形成

> 本文分析参考模组的朋党核心。监察、弹劾、刑部审办和锦衣卫将分别在 `09A`、`09B` 深入解析；本文只说明它们怎样成为党争资源和反制渠道。
>
> 前置阅读：[皇权、内阁与诏令](08_皇权内阁与诏令.md)。

## 1. 核心结论

参考模组的朋党系统由四层组成：

1. **关系网络**：党徒之间建立 `pengdang` 关系，党魁与党徒另有定向关系。
2. **组织再生产**：高官建党、分裂旧党、拉拢门生亲友、保举党徒上任、党魁继承和弱党自动补充。
3. **职位折算**：党派影响力只看成员实际占据的中枢、监察、军事和地方职位，不直接看官品。
4. **权臣跃迁**：某党影响力超过“全国官职影响力的一半 + 皇权修正”后，党魁取得 `quanchen` 标记，进而侵蚀皇权、限制皇帝决策，并可能策动宫廷刺杀。

系统最成功之处，是把朋党强弱绑定到具体组织节点，而不是抽象人数或人物好感。科举主考、吏兵二部、内阁、六科、御史、厂卫和督抚任命都会改变长期力量对比。

系统最主要的局限，是朋党仍被建模为纯粹的官员人际网络：没有稳定的经济支持者、地区利益、阶级基础、政策纲领和社会动员。新模组应保留“职位—组织资源”逻辑，并补上物质基础、意识形态与历史阶段。

## 3. 关系数据结构

### 3.1 三种关系

| 关系 | 方向 | 效果 | 用途 |
|---|---|---:|---|
| `pengdang` | 对称 | 相互好感 +5 | 表示同一朋党的普通成员关系 |
| `dangkui` | 党徒指向党魁 | 好感 +10 | 从成员找到领袖 |
| `dangtu` | 党魁指向党徒 | 好感 +10 | 从领袖枚举成员 |

`dangkui` 与 `dangtu` 在 scripted relation 中互为 corresponding；`pengdang` 与自身对应。

党魁还持有 `dangkui` 人物特质，效果包括：对领主好感 -15、每月威望 +4、每月宗族威望 +0.35、对封臣好感 +5。它将组织领袖的独立政治声望和对皇帝的结构性疏离直接写成人物修正。

### 3.2 加入党派

`join_pengdang` 的行为为：

1. 保存现有成员或党魁为 `join`；
2. 保存新成员为 `joiner`；
3. 遍历 `join` 的所有 `pengdang` 关系；
4. 让每名旧成员都与新成员建立 `pengdang`；
5. 查找党魁并建立党魁—党徒关系；
6. `join` 本人也与新成员建立 `pengdang`。

结果是每个朋党成为一个完全连接网络。若有 `n` 名成员，关系边数量约为 `n(n-1)/2`，另加党魁—党徒边。

这种结构的优点是任何成员都能直接用 `every_relation type = pengdang` 得到全体同党，脚本写起来直观。缺点是规模平方增长：20 人约 190 条同党边，50 人约 1225 条，100 人约 4950 条；每次新成员加入还要遍历全党。它会增加存档体积、关系更新和人物界面成本。

### 3.3 退出党派

`pengdang_leave`：

- 删除该角色与所有党徒的 `pengdang` 关系；
- 删除其党魁关系；
- 若本人是党魁，也移除党魁特质。

它没有单独的“党派对象”。朋党的存在完全依赖关系图和某个持有党魁特质的人。

## 4. 党派形成与分裂

### 4.1 作者说明与脚本口径

说明书称：

- 朋党由高阶官员自发建立；
- 同时最多四党；
- 党内没有足够高阶的在任官员时自然解散；
- 朋党不可被永久消灭，皇帝目标是维持平衡。

当前脚本的 AI 建党门槛为：

- 朋党系统开启；
- AI 角色；
- 官品至少第 14 级；
- 最高实际头衔至少公爵／`tier_duchy`；
- 顶级领主为大明皇帝；
- 本人不是党魁；
- 本人无党，或与现党党魁互为宿敌／低好感；
- 全世界党魁数量不超过 3。

因此第四个党可以产生。这里的“四党上限”用全局党魁计数实现，没有限定大明，兼容多政权时会互相占用名额。

### 4.2 新党从何而来

建党者若原本有党，会先退出旧党并保存旧党魁。随后把以下角色拉入新党：

- 自己的门生；
- 至交／朋友；
- 子女；
- 对自己好感至少 90、且原属旧党者。

脚本依次遍历大明的封臣、皇帝廷臣，以及封臣的廷臣。符合者先退旧党，再加入新党。

这实现了党派分裂的社会机制：高官与原领袖失和并不是独自离党，而会带走私人依附网络。门生、家属、属官和朋友是组织裂变的基本单位。

### 4.3 皇帝暴政促进结党

AI 拉拢通常有 12 个月冷却；当皇帝暴政至少 60 时缩短到 5 个月，并放宽招募纽带到：

- 子女和门生；
- 下属、朋友、至交；
- 同宗族；
- 内阁、九卿和高阶地方官。

这形成正确的反作用：皇帝以暴政压制个别官员，会使更广泛的精英出于共同安全需求加速组织化。

### 4.4 拉拢对象

常规拉拢对象需：

- 是大明政治共同体内的 AI 男性在任官员；
- 成年、有行为能力、无党、非党魁；
- 不是害羞或安于现状者；
- 对招募者有较高好感，或有亲属、门生、隶属、科举身份等纽带；
- 若是有地统治者，政府必须属于布政／卫所体系。

脚本还在招募结束后扫描全世界，把仍与招募者有党关系、但成为角色池人物或已离开大明者直接设为招募者雇员。这是为了修复成员漂移，却可能把已经在外政权任职的角色强行拉回，属于高风险兜底逻辑。

## 5. 党派的自动补充

`buchong_pengdang` 被挂在角色生日 on_action 上，但只有生日角色是大明皇帝时才运行，故大致每年执行一次。

当全世界党魁少于四名时，依次尝试把以下无党人物设为新党魁：

1. 皇帝廷臣中的大学士；
2. 大明下属中公爵级以上的布政／卫所官；
3. 没有合适地方官时的吏部尚书。

新党魁刚出现时可以没有党徒，随后依赖 AI 拉拢逐渐组织化。

这相当于给政治系统设置“生态位回填”：即使皇帝清洗掉一个朋党，只要国家仍有高官和人事网络，新的组织中心就会形成。它符合“朋党源自制度结构，不是几个坏人的阴谋”的方向。

当前实现的问题是：党魁计数仍是全局扫描，且补充触发条件没有显式检查 `Pengdang_switch`；若系统开关语义或多大明实例发生变化，可能产生异常。

## 6. 党魁继承

### 6.1 触发时机

旧党魁死亡、退休、罢免、丁忧、离开大明等多个流程会调用 `dangkui_jicheng`。进入继承要求：

- 系统开启；
- 当前角色是党魁并有党徒；
- 党内没有另一个党魁；
- 自己没有指向其他党魁的关系。

### 6.2 候选资格

候选必须：

- 仍属于大明政治共同体；
- 非旧党魁、非其他党魁；
- 未退休、未罢免、未丁忧；
- 官品至少第 12 级；
- 通常需要是实际在任的县级以上地方官，或满足高官品的替代分支。

没有合格候选时，脚本删除全党关系并解散朋党。

### 6.3 继任评分

`dangkui_houbu_score`：

| 因素 | 分数 |
|---|---:|
| 威望等级 1／2／3／4／5+ | +200／400／600／800／1000 |
| 已退休或罢免 | -5000 |
| 王级地方官 | +1800 |
| 公爵级地方官 | +1200 |
| 伯爵级地方官 | +500 |
| 男爵级地方官 | +250 |
| 在任大学士 | +2200 |
| 在任九卿 | +1800 |
| 近亲是大学士或九卿 | +300 |
| 年龄至少 45 | +100 |
| 每名对其好感大于 85 的党徒 | +50 |

选分最高者继任。因此领袖产生于正式职位、公共威望、家族关系、资历和内部支持的合成，而不是单纯世袭或随机。

### 6.4 权臣状态的继承

若旧党魁有 `quanchen` 标记，新党魁会立即继承；随后把所有党徒的党魁关系改指新人。新党魁在自己的生日运行 `QC_CHECK` 时才按当前影响力重新确认。

这使“权臣”首先是一党控制的结构性地位，而非旧党魁的个人特质。短暂继承也避免领袖死亡当日让所有制度限制瞬间消失。

但由于重算可能延迟近一年，已经跌破阈值的党仍可暂时保留权臣能力。新设计应改成党魁变更当日立即重算，同时用数季度滞后防止频繁跳变。

## 7. 党派政治影响力

### 7.1 计算原则

`pengdang_yingxiang_score` 对党魁本人和其全部 `pengdang` 关系成员逐一检查实际职位。官品本身不直接贡献，只有占据组织节点才产生影响力。

这是本系统最值得继承的设计：一个有大量闲散高品官的朋党，不如控制吏部、内阁、厂卫和督抚的较小朋党。

### 7.2 中枢与特殊职位权重

| 职位 | 单人基础影响力 |
|---|---:|
| 司礼监掌印 | 200 |
| 锦衣卫都督／带 `jinyiwei` 身份 | 200 |
| 内阁首辅的额外值 | 200 |
| 内阁次辅的额外值 | 80 |
| 六部九卿 | 150 |
| 翰林／中枢特定京职 | 130 |
| 五军都督府掌印 | 150 |
| 宗室亲王 | 150 |
| 六科给事中 | 80 |
| 巡按御史 | 60 |

### 7.3 阁臣的动态权重

每名阁臣还按当前阁臣总数取得通用权重：

| 阁臣总数 | 每名阁臣通用值 | 通用总量 |
|---:|---:|---:|
| 6 | 200 | 1200 |
| 5 | 240 | 1200 |
| 4 | 300 | 1200 |

这使“内阁作为整体”的总政治资源大致恒定；席位越少，单个阁臣控制的票拟与信息权越集中。首辅和次辅还叠加额外值，所以一人内阁中的首辅可能达到约 1400。

### 7.4 地方与军事节点权重

### 7.5 其他京官按层级

| CK3 层级 | 影响力 |

| 伯爵级 | 35 |

| 帝国级 | 120 |

这套标尺大体保持：控制中枢关键机构的价值远高于同级普通京官，控制地方层级越高则掌握越多执行与人事资源。

## 8. 权臣阈值

### 8.1 公式

`zong_yingxiang_score` 逐项统计大明所有关键职位，使用的权重基本恰好是党派权重的一半，再加入：

```text
皇权修正 = 皇权 × 1.5
```

因此可概括为：

```text
权臣阈值 ≈ 全国有效官职影响力总和 ÷ 2 + 皇权 × 1.5
```

当某党：

```text
pengdang_yingxiang_score > zong_yingxiang_score
```

党魁获得 `quanchen` 标记，否则移除。

### 8.2 政治含义

阈值不是固定点数，而是“是否控制国家过半组织资源”的近似。扩张国家、增加有效机构和官员会提高总盘子；皇权则作为皇帝独立于官职网络之外的额外政治资本。

这比“党徒超过 20 人即成为权臣”更有解释力。一个控制首辅、司礼监、锦衣卫和九卿的少数集团，可以超过由大量县官组成的组织。

### 8.3 更新频率

`QC_CHECK` 挂在全体角色生日 on_action 上，但仅党魁符合触发器。因此每个党大约每年在党魁生日重算一次。它用分散生日避免所有党同一天计算，性能思路合理；代价是职位变动后状态最多延迟近一年。

### 8.4 数值边界

- 高正皇权提高权臣阈值，负皇权会直接降低阈值。
- 当负皇权绝对值很大时，阈值可能显著下降，弱党也能轻易成为权臣。
- 若多个党都超过阈值，脚本逻辑允许多个党魁同时带 `quanchen`；说明书叙述更像只有最高党，但 `QC_CHECK` 是逐党独立判断。
- 阈值代码手动复制了全部职位的半权重，党派权重与全国权重以后若只修改一边，会悄悄失去“过半”含义。

新模组应从一份统一权重表生成党派值和总值，或把各职位的标准影响力封装成同一 scripted value。

## 9. 权臣状态的实际后果

### 9.1 皇权流失

季度皇权公式只要发现任一仍在政治中的 `quanchen`，皇帝每季度 -10；没有权臣则 +0.5。这会形成强烈正反馈：

```text
一党过半
→ 权臣形成
→ 皇权快速下降
→ 权臣阈值继续下降
→ 党派更难失去多数
→ 皇权跌破零并触发首辅摄政
```

### 9.2 决策限制

`quanchen_trigger` 对党魁本人或其同党成员都返回真。大量任免、弹劾与其他政治互动使用它禁止或改变皇帝选择。

这意味着权臣不是只有领袖本人有特权，而是整个主导党获得制度性保护。

### 9.3 封还诏旨

主导党形成后，弹劾处置和进攻战争可能触发内阁封还。皇帝若强推，要消耗大量皇权、获得高暴政并清洗阁臣；退让也会损失少量皇权。详细流程见 `08`。

### 9.4 宫廷刺杀

`GM_Pengdang_events.001` 由年度全局脉冲调用。触发条件包括：

- 玩家／AI 角色为大明皇帝；
- 存在大明权臣党魁，且其对皇帝好感低于 -20；
- 皇帝宫廷中存在锦衣卫、司礼监掌印或宫廷女官等内廷人员。

事件会：

1. 从内廷人员中随机选择刺客；
2. 令刺客加入权臣朋党并建立党魁—党徒关系；
3. 皇权 -30；
4. 以中毒原因杀死皇帝。

只要条件满足，事件没有额外成功率或阴谋进度，年度检查会确定性地完成刺杀。它把“权臣控制外朝 + 渗透内廷”视为政变完成条件，结构清晰，但对玩家而言过于二元，也没有反情报、忠诚和暴露过程。

源码还用全局扫描选择六个阁席人物，没有限定他们属于当前皇帝，存在串台风险。

## 10. 皇帝如何影响党派平衡

参考模组作者明确提出：目标不是消灭朋党，而是“扶弱抑强”。实际杠杆包括：

### 10.1 科举与门生

会试主考官与考生形成师生关系，门生关系又是 AI 拉拢和党派分裂的重要纽带。让某党长期掌握科举，等于持续为其生产未来官员。

### 10.2 吏部与兵部

文武官员的选授和升迁评分会读取吏部、兵部及其党派关系。控制两部既提供直接影响力，也能把低级党徒送入未来的关键职位。

### 10.3 廷推与中旨任命

皇帝可在关键职位上优先任命弱党或中立者；必要时用皇权支付中旨越权成本。短期压低强党，会同时损耗皇帝独立政治资本。

### 10.4 退休与罢免

实际职位丢失后，角色的影响力贡献清零。接受强党高官退休，是低冲突削弱其组织控制的手段；强行罢黜则可能触发封还和暴政反作用。

### 10.5 风宪官与考核

都察院、六科、巡按与京察大计可以选择性揭露、降调或淘汰强党成员。监察机关自身也有党籍，因此“反腐”可能是公共治理，也可能是党同伐异。

### 10.6 锦衣卫

厂卫能在关键时刻精确打击主导党人物，但长期使用会增加群臣反弹、冤狱和对秘密强制渠道的依赖。参考模组已经有调查、提审、抄家和刑部复核事件链，后续文档单独解析。

## 11. 系统层面的优点

1. **职位而非人数决定力量**：真正模拟谁掌握组织资源。
2. **组织会再生产**：清洗领袖不能消灭产生朋党的制度条件。
3. **党魁可更替**：组织力量不随个人死亡清零。
4. **高压催生联盟**：暴政加速拉拢，压制有反作用。
5. **科举创造长期路径依赖**：主考—门生—入仕—升迁构成数十年链条。
6. **皇权与党权相互反馈**：国家权力不是单边数值竞赛。
7. **内外朝都可成为组织节点**：权臣刺杀需要内廷渠道，避免只算文官职位。
8. **玩家主要通过人事而非直接控制党派**：符合皇帝面对自组织官僚网络的处境。

## 12. 已确认的技术与设计风险

### 12.1 平方级关系网

完全图使党派规模翻倍时关系数量接近四倍。新模组若同时加入商人、工场主、士绅和群众组织，不能沿用全体成员互连。

### 12.2 全局作用域串台

以下多处只查全世界而非当前大明：

- 四党上限；
- 弱党补充；
- 权臣存在判断的部分用法；
- 阁臣与党魁随机选择；
- 党外漂移角色修复。

所有正式实现必须统一使用大明帝国头衔或顶级领主限定。

### 12.3 多权臣与说明不一致

脚本按每个党独立比较阈值，理论上允许多个权臣；作者说明把权臣描述为最高影响力党魁。需要在策划层明确：

- 若只允许一个主导党，应比较第一名、第二名和阈值；
- 若允许多个寡头集团，应把状态命名为“否决集团”，另选“主导集团”。

### 12.4 状态更新滞后

权臣只在党魁生日重算。关键任免、党魁继承和大规模罢黜后应立即标脏，并在短延迟后重算缓存。

### 12.5 党派没有政策内容

党派主要由关系、职位和人物态度形成，不能回答：

- 为什么支持或反对某项税制、土地、工商业和军事改革；
- 其资源来自地主、商人、军队还是城市组织；
- 生产方式变化后为何分裂或转型；
- 党徒之间在具体议题上为何可能跨党合作。

### 12.6 权臣危机容易雪崩

一旦过半，季度 -10 皇权会继续降低阈值；刺杀又是满足条件后的确定性死亡。缺少妥协政府、联合内阁、政策让步、组织分裂和合法退出时，长期博弈会退化为“提前阻止，否则读档”。

### 12.7 调试交互与正式 AI 混杂

玩家建党、拉拢和保举交互带 `debug_only = yes`，正式党派运行主要靠名为 `debug_mode_decision.txt` 的 AI 决议。功能能运行，但代码组织不利于维护。新模组应把正式 AI、玩家调试和自动修复分开。

## 13. 历史唯物主义转译

朋党不是抽象观念共同体，也不只是“好感高的一群官”。它应被理解为不同统治集团和社会力量争夺国家组织资源的政治形式。

### 13.1 组织资源向量

每个政治集团至少保存：

```text
官僚职位控制
+ 地方士绅网络
+ 土地与租赋资源
+ 商业信用与工场资本
+ 军事指挥与军饷债权
+ 宫廷信息渠道
+ 学校、书院、出版与意识形态生产
+ 城乡群众组织联系
```

同样总影响力的两党可能完全不同：

- 官僚—地主党擅长拖延税制与地方执行；
- 商人—技术官僚联盟能提供信用和投资，却要求市场特许；
- 军事集团掌握即时暴力，但依赖持续军饷；
- 宫廷集团能快速下达中旨，却难以在地方形成稳定执行；
- 城市改革联盟能动员舆论与劳工，但可能缺少传统官僚经验。

### 13.2 党派是阶段性的联盟

历史阶段变化会改变结党基础：

| 阶段 | 主要组织基础 | 核心矛盾 |
|---|---|---|
| 财政—租赋农业帝国 | 科举、宗族、土地与官职 | 中央征取与地方士绅控制 |
| 商业化与白银化 | 盐商、长途贸易、税关与债权 | 土地税秩序与流通资本扩张 |
| 原工业化 | 工场主、包买商、手工业行会 | 身份束缚、市场统一与劳动控制 |
| 财政—军事国家 | 军饷债权、技术官僚、军工 | 高税动员与地方／民生承受力 |
| 工业转型 | 制造资本、产业工人、工程官僚 | 资本积累、劳动权利与国家控制 |
| 革命／反革命 | 群众组织、军队分裂、地方政权 | 国家机器归属与生产关系重组 |

党名可以延续，但其物质基础、政策和成员构成会改变；旧党也可分裂为地主保守派与工商改良派。

### 13.3 意识形态不是装饰标签

意识形态应发挥三种作用：

1. 解释共同利益，把不同成员组织为联盟；
2. 降低内部协调成本，提高动员；
3. 规定哪些政策被视为合法、可想象或不可接受。

物质利益不自动生成唯一立场。同一商人集团可能支持皇权保护市场，也可能支持地方自治；选择取决于战争、税制、组织经验和既有思想资源。

## 14. 新模组的党派数据结构

### 14.1 保留四个可见主集团，取消完全图

为保证 CK3 性能和 UI 可读性，帝国层面最多显示四个主要政治集团，另设“中立／未组织力量”。每名关键人物只保存：

- 一个集团成员身份；
- 一条指向集团领袖的关系；
- 必要时一条副集团／议题联盟变量；
- 个人忠诚度和激进度。

党魁只需通过 `dangtu` 关系枚举成员，不再让所有党徒两两建立 `pengdang`。

### 14.2 集团槽位

可在大明帝国头衔上维护四个固定槽：

```text
ming_bloc_1_leader
ming_bloc_1_bureaucracy
ming_bloc_1_land
ming_bloc_1_commerce
ming_bloc_1_military
ming_bloc_1_court
ming_bloc_1_popular
ming_bloc_1_cohesion
ming_bloc_1_radicalism
ming_bloc_1_program
```

其余槽位同构。人物用特质或小整数变量记录所属槽位。领袖死亡只需更新领袖和其线性成员关系。

### 14.3 影响力公式

建议：

```text
集团有效影响力 =
  官职控制 × 官僚渠道效率
+ 地方基础 × 地方自治度
+ 经济资源 × 财政依赖系数
+ 军事资源 × 军队政治化
+ 信息资源 × 信息不透明度
+ 群众组织 × 动员环境
```

每个渠道都受历史阶段和制度影响。例如：商业化提高商业信用权重；常备军和军工扩张提高军饷债权权重；印刷和城市化提高舆论与群众组织权重。

### 14.4 权臣不再只看一条过半线

集团成为主导集团应同时满足：

1. 总有效影响力超过统治联盟可用资源的某一比例；
2. 领先第二名至少一定差额；
3. 至少控制两个关键渠道；
4. 状态持续 2–4 个季度；
5. 皇帝没有形成足以抵消的跨集团联盟。

退出主导状态使用更低阈值和持续时间，形成迟滞区，避免一项任命造成状态来回闪烁。

## 15. 党派行为与反制

### 15.1 非暴力手段

- 争夺主考和学校网络；
- 推荐、保举、廷推和部门任命；
- 预算交换与政策捆绑；
- 封还、部覆、辞职和消极执行；
- 舆论、书院、商会和宗族动员；
- 提供财政贷款或拒绝认购国债；
- 组建跨党议题联盟。

### 15.2 升级手段

- 大规模弹劾和司法清洗；
- 厂卫侦缉、罗织罪名、抄家；
- 地方抗命、税款截留；
- 军队拒命、兵变和拥立；
- 罢市、罢工、抗粮和群众运动；
- 宫廷政变、刺杀、革命或反革命。

升级不由随机事件直接决定，而由“组织度、受压程度、预期胜算、合法渠道是否关闭、危机强度”共同评分。

### 15.3 皇帝的长期选择

皇帝不能只点“打压强党”。可选策略应包括：

| 策略 | 短期效果 | 长期风险 |
|---|---|---|
| 扶持弱党 | 降低单党垄断 | 国家职位被当作平衡筹码，能力下降 |
| 跨党技术政府 | 提高执行与政策质量 | 各党都可能缺乏完全忠诚 |
| 中旨与厂卫 | 快速突破否决 | 程序合法性、信息质量和继承稳定下降 |
| 扩大社会联盟 | 获得商人、军人或群众支持 | 旧精英联合反扑，新盟友要求制度权利 |
| 制度化竞争 | 降低暴力清洗 | 皇帝让渡议程权，集团组织更稳定 |
| 分化主导党 | 快速削弱集团凝聚 | 交易成本高，可能产生更激进分支 |

## 16. 性能与更新策略

### 16.1 禁止做法

- 不建立党徒完全图；
- 不在 tooltip 中遍历全国官员；
- 不在每个角色生日重算整个国家；
- 不在季度脉冲扫描全世界活人；
- 不为普通社会成员创建 CK3 角色；
- 不把四党权重分别复制成四套易漂移代码。

### 16.2 推荐更新

- 任命、罢免、死亡、退职、入党、退党：只调整相关集团的渠道缓存并设 `dirty`。
- 每季度：只重算被标脏集团和权力状态。
- 每年：做一次全量校验，修复缓存漂移。
- 历史阶段变化：重算各渠道系数，不逐人重建。
- UI：只读帝国头衔上的缓存分数和前十名关键成员。

### 16.3 单一权重接口

每个关键职位通过统一 scripted value 返回：

```text
office_bureaucratic_power
office_military_power
office_information_power
office_fiscal_power
office_local_power
```

党派缓存和全国总量都调用同一接口，避免参考模组“党派权重一份、阈值半权重再抄一份”的维护风险。

## 17. MVP 实现边界

### 17.1 必须实现

1. 四个集团槽与中立力量。
2. 线性党魁—党徒关系，不建立完全图。
3. 党派形成、拉拢、退出、分裂、合并、领袖继承。
4. 官僚、地方、经济、军事、宫廷五个影响渠道。
5. 科举、吏部、内阁和督抚任命对长期力量的影响。
6. 主导集团迟滞判定与至少两种非暴力否决手段。
7. 皇帝暴政促进反对组织化。
8. AI 会扶弱、结盟、打压、分裂和接受妥协。
9. 任命事件驱动缓存、季度脏重算、年度全量校验。

### 17.2 第二阶段

1. 商人、地主、军事和城市组织作为集团社会基础。
2. 政策纲领、议题联盟与意识形态转化。
3. 秘密组织、地下化、政变和刺杀进度。
4. 革命与反革命中的党派—阶级—军队重组。
5. 印刷、教育、交通和工业化对组织成本的影响。

### 17.3 暂不实现

- 无限数量小党；
- 每个官员独立模拟完整意识形态量表；
- 每个县建立党支部；
- 满足条件即确定性刺杀；
- 单一“政治影响力”吞并所有经济、军事和群众资源；
- 让玩家直接拖动党派点数。

## 18. 验收清单

- [ ] 任何集团成员数翻倍时，关系数量保持线性增长。
- [ ] 四党上限只作用于当前大明实例。
- [ ] 党魁死亡当天能选出继任者并重新计算主导状态。
- [ ] 没有合格继承人时，集团按规则解散或重组，不遗留关系。
- [ ] 官职的党派贡献和全国总量来自同一权重函数。
- [ ] 一项任命只标脏相关缓存，不触发全世界扫描。
- [ ] 主导集团形成需要优势、渠道和持续时间，不因 1 点波动切换。
- [ ] 高压能短期打击公开组织，同时提高地下组织、激进化或跨党联盟概率。
- [ ] 科举主考影响未来人才网络，但不会让全部考生自动永久入同一党。
- [ ] AI 可在 50 年观察中形成、分裂、衰落和重组集团。
- [ ] 没有集团永远天然代表“进步”或“反动”；立场取决于其社会基础与具体政策。
- [ ] 力量对比在商业化、再工业化和革命阶段会改变渠道权重。

## 19. 与后续文档的接口

- `09A` 将分析言官、六科、都察院、弹劾与刑部—大理寺—都察院审判链，解释党争怎样进入正式监察程序。
- `09B` 将分析锦衣卫调查、提审、抄家、冤狱和秘密强制，解释皇帝怎样建立替代信息与暴力渠道。
- 财政与社会文档将为党派增加地主、商人、工场、军饷债权和城乡群众的物质基础。
- 历史阶段文档将规定不同生产方式下哪些组织渠道升值、哪些旧集团分裂或衰落。

朋党模块最终不是一个“保持四条进度条平衡”的小游戏，而应是国家机构、社会阶级和动员技术不断重组的政治表现层。
