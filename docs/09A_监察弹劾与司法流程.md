# 09A 《变身大明》监察、弹劾与司法流程

> 参考机制解析。现行重大案件规格见 `23C` 与 `30E`；本文不授权直接调用前置案件事件链。

## 1. 已有流程

```text
弹章/举报 → 候选与触发 → 皇帝处置 → 吏部查覆或刑部审办
→ 罪名/量刑 → 大理寺复核或三法司重审 → 执行/平反
```

- 六科、都察院、刑部、大理寺等由现有职位、事件和互动表现；
- 普通官员、高级官员和战败责任有不同入口；
- 前置把人物属性、主审者和事件分数映射为罪名和处罚；
- 部分流程允许复核、释放或复职。

## 2. 可读取内容

- 当前监察、刑部、大理寺等有效职位持有者；
- 人物是否在廷、入狱、死亡、失位；
- 已验证的案件相关 trait/modifier/罪名；
- 事件主题、书信表现和朝审叙事样板。

## 3. 已确认风险

- 某些事件链使用未保存或复用的 scope，可能案件串线；
- 存在重复 ID、人数遗漏和清理不完整风险；
- 分数直接等同罪名会把证据、程序和政治压力塌缩；
- 直接触发 `GM_Tanhe_events` 可能缺必需内部状态。

## 4. 新项目边界

- 不调用或覆盖 `GM_Tanhe_events`，不写内部 flag；
- 只为改革责任和大型事件保存证据、程序、压力、恐惧四值；
- 正式撤职、监禁和前置职位变更仍由《变身大明》原界面完成；
- 无合适办案者时匿名化为机构，不生成证人或官员群体；
- 全国同时最多一个本项目案件槽。
