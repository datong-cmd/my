# 23C 朋党—监察—司法—厂卫实施计划

## 1. 不制造第二套朋党

《变身大明》已有 `pengdang`、`dangkui`、`dangtu` 关系，`join_pengdang`/`pengdang_leave` 效果和 `pengdang_yingxiang_score`。这些是人物的长期政治组织。本模组不再新增 A1–A6 六类联盟；只为当前政策保存至多四个既有人物代表及其临时立场，人物可以因议题不同与同党意见不一。

兼容约束：不调用 `join_pengdang`，不直接增删上述关系，不授予 `dangkui`，不触发或覆盖 `GM_Pengdang_events`。

## 2. 政策联盟数据

不另建六类集团、组织度或逐集团账本。每项政策只从八项全国值中选择最多三个相关输入，并加既有官位、朋党和人物关系修正：

`联盟支持 = 相关全国值 + 既有党争/官位修正 + 路线承诺 - 政策代价`

现有朋党只提供朝廷放大器：党魁的 `pengdang_yingxiang_score` 与其官位可把某一社会诉求转化为更强的廷议票；它不能凭空创造社会力量。每项政策最多记录 4 个既有党派代表及其临时 `agenda/support`，政策结束即清理。

## 3. 监察案件槽

本模组只处理“改革执行产生的政治案件”，不复制日常弹劾。案件只能在共用前台锁为空时启动；若由政策失败转入，先结算并清理政策槽，再把摘要值写入案件初值：

| 字段 | 范围 | 含义 |
|---|---:|---|
| `minghm_case_stage` | 0..5 | 举报、初核、调查、廷议、结案 |
| `minghm_case_evidence` | 0..100 | 可验证证据 |
| `minghm_case_pressure` | 0..100 | 要求办案的政治压力 |
| `minghm_case_procedure` | 0..100 | 程序完整度 |
| `minghm_case_fear` | 0..100 | 恐惧与寒蝉效应 |
| `minghm_case_target` | saved scope | 被调查者 |
| `minghm_case_investigator` | saved scope | 主办者 |

入口仅限：政策纸面落实、明确阻挠、资源侵吞或玩家从兼容决议立案。目标必须是明制内部角色。

## 4. 机构选择

机构不是换皮按钮：

| 路线 | 读取的既有身份 | 优点 | 长期代价 |
|---|---|---|---|
| 都察院/给事中 | 左都御史、巡按、六科职位 | 程序与公开性高 | 易受党争和拖延影响 |
| 刑部—大理寺 | 刑部尚书、大理寺卿 | 证据门槛稳定 | 对紧急政治危机反应慢 |
| 锦衣卫 | `court_jinyiwei_position`/`jinyiwei` | 调查快、突破阻挠 | 恐惧、伪证和信息失真风险 |
| 内廷协调 | `chief_eunuch_court_position` | 可绕过外朝封锁 | 提高内廷俘获和派系反扑 |

若对应职位空缺，该路线禁用或显著降效；不自动生成官员。

## 5. 与原弹劾系统的安全边界

`GM_Tanhe_events.001` 等事件依赖调用者预先保存的 `tanheren`、`beitanheren` 等作用域，并操纵原模组冷却与关系。直接从外部触发容易破坏作用域，因此 MVP 不调用这些事件 ID，也不改 `yibei_tanhe` 等原 flag。

本模组案件结案只处理“改革项目责任”：政策信用、专员去留、本模组政治资本、关系/威望和有限合法基础效果。需要正式弹劾、革职、监禁时，事件文本引导玩家转用《变身大明》既有互动；本模组在下一次季度检查目标是否已失位、入狱或死亡，并据此完成政治反馈。将来只有在参考模组公开稳定 scripted effect 接口后，才增加一键移交适配。

## 6. 结案矩阵

| 条件 | 结果 |
|---|---|
| 高证据 + 高程序 | 可信追责，降低俘获，小幅提高国家能力 |
| 高证据 + 低程序 | 快速清除阻力，但提高恐惧和未来消极执行 |
| 低证据 + 高压力 | 政治清洗；短期支持者满意，长期政策信用下降 |
| 低证据 + 低压力 | 无法结案；调查成本与机构声望损失 |

党派是否支持只改变压力和舆论后果，不改变证据本身。厂卫能提高调查速度，但不能自动把低证据变成真相。

## 7. AI、性能、验收

- AI 在证据低于 35 时原则上不重罚，除非专断/多疑且政治压力极高。
- 季度只推进活动案件；调查者/目标状态用直接 saved scope 检查，不扫描世界。
- 验收包含目标死亡、职位空缺、皇帝继承、原弹劾同时发生、政策转案件以及结案释放共用前台锁。
- 确认本模组未定义 `pengdang` 关系、原党魁特质、原冷却 flag 或 `GM_Tanhe_events` 命名空间。
