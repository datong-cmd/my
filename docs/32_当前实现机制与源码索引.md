# 32 当前实现机制与源码索引

> 用途：这是交给后续编码 AI 或人工维护者的“已写代码”入口。它只描述仓库中已存在的实现；策划全貌仍以 [27A](27A_AI编码交付总实施书.md) 为准，尚未实机验证的接口见 [31](31_技术疑难与探针记录.md)。

## 1. 快速边界

- 启动前提：CK3 1.19、溥天之下 DLC 与《变身大明》同时启动；本模组位于《变身大明》之后。
- 生效范围：唯一入口是《变身大明》的 `is_GM_emperor`，因此只对 `title:h_greatming` 的持有者生效。
- 数据范围：长期数据只保存在 `h_greatming`，全部使用 `minghm_` 前缀；活动数据只占一个前台槽。
- 明确未做：财政、市场、商品、人口、区域生产、跨国通用化、新 3D、建筑、兵种、派系、逐人物意识形态与革命战争 CB。仅有六枚固定 64×64 的行内 2D 图标，不含窗口或动态 GUI。

## 3. 长期数据字典

| 字段 | 范围 | 含义 |
|---|---:|---|
| `minghm_state_capacity` | 0..100 | 国家组织、命令落地能力 |
| `minghm_reform_momentum` | 0..100 | 改革组织和承诺 |
| `minghm_conservative_power` | 0..100 | 守成力量 |
| `minghm_popular_mobilization` | 0..100 | 民众动员能力，不代表人口 |
| `minghm_elite_cohesion` | 0..100 | 精英协调能力 |
| `minghm_military_autonomy` | 0..100 | 军政自主，不代表军队数量 |
| `minghm_commercial_influence` | 0..100 | 承办组织的政治议价力，不代表财富或市场 |
| `minghm_crisis_pressure` | 0..100 | 全国压力 |
| `minghm_historical_stage` | 0..4 | 稳固、失衡、改革竞争、双重权力、新秩序整合 |
| `minghm_industrial_stage` | 0..4 | 技术组织方式，不代表产能 |
| `minghm_new_order_commitment` | flag | 由技术/革命成果写入，整合严重失败会移除 |

所有政策、案件、危机、技术和革命的临时字段在 `minghm_cleanup_active_content_effect` 中删除。长期值只通过 `minghm_change_power_effect` 修改并立即钳制。

年度阶段改变经两年确认后会排队一次普通事件摘要；它只显示结果，不产生额外数值、人物或月度处理。

## 4. 已实现的玩家入口

| 决议 | 条件概要 | 核心对抗 | 结算影响 |
|---|---|---|---|
| 推行官僚考成重整 | 明帝、空槽 | 内阁协调/朋党阻力 | 国家能力、改革、守成、精英 |
| 建立民生协商与动员程序 | 能力与改革达到门槛 | 动员支持/守成阻力 | 民众动员、改革、危机 |
| 规范承办组织政治参与 | 商业影响至少 25 | 承办支持/政治阻力 | 商业影响、能力、民众反作用 |
| 推行军制重整 | 军事自主或危机较高 | 统帅协调/军政自主 | 能力、军事自主、危机 |
| 整合新秩序承诺 | 已有承诺 | 整合支持/反弹 | 阶段稳定或撤销承诺 |
| 启动程序性案件 | 大理寺或锦衣卫在廷 | 证据、程序、压力、恐惧 | 能力、精英、守成、危机 |
| 处置军政失衡 | 战争或军政失衡 | 朝廷响应/危机力量 | 军事自主、能力、危机 |
| 处置朝局失衡 | 精英协调低、守成高或危机高 | 朝廷响应/守成压力 | 精英、守成、能力、危机 |
| 设立技术译审与试制网络 | 工业阶段 0 | 官署/承办协作 | 开启技术阶段 1 |
| 推进技术组织革新 | 技术阶段 1..3 | 支持、承诺、阻力 | 技术阶段、改革、商业/军事反作用 |
| 处置双重权力危机 | 阶段 3 且组合门槛 | 动员力量/朝廷响应 | 恢复、妥协、持续双重权力或断裂 |
| 查看大明局势 | 任意明帝 | 秩序、变革、解体三分数 | 只读显示 |
| 撤回当前政治行动 | 当前有前台内容且非继承待复核 | 行政信誉/危机代价 | 清除当前单槽，避免项目死锁 |
| 定立新君朝局方针 | 取得大明后的无活动状态 | 皇权、官僚或民生动员 | 为新君写入轻量力量偏向 |

五条长期改革共享 `minghm_policy_id = 1..5` 与同一季度脉冲。无活动内容时没有新增季度循环。

官僚考成的“授权首辅内阁承办”只在《变身大明》当前存在 `diange_daxueshi` 且具 `zhongjidian_daxueshi` 的大臣时显示。授权后，季度脉冲只检查这一条既有 relation：首辅仍在任则支持 `+2`，失位则阻力 `+3`。这把前置内阁的实际变动纳入博弈，但不创建专员、不保存人物 scope、不改写任何上游官职或特质。
