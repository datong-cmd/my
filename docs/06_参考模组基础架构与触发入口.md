# 06 《变身大明》基础架构与触发入口

> 本文只保留架构事实和可借的安全模式，不保留早期区域、人口或独立系统设计。

## 1. 核心对象

- `h_greatming`：前置定义的大明最高标题；
- `is_GM_emperor`：判断持有该标题的皇帝 trigger；
- `yan_found_GM_effects.txt`：建立大明、变更 title holder 与 primary title；
- 官位、品级、朋党、军功等事实分别存在角色 trait、position、relation、variable 和 modifier 中。

精确路径和引用行由 `01/02` 维护。

## 2. 前置使用的触发类型

| 入口 | 前置用途 | 新项目边界 |
|---|---|---|
| `on_game_start_after_lobby` | 初始化 | 只检查 `h_greatming`，不得扫独立统治者 |
| `on_title_gain` | 建国/继承 | 只在获得 `h_greatming` 时延迟执行 |
| 年度/季度 pulse | 官员、皇权、党争等 | 新项目无活动时仅年度低成本判断 |
| 生日 | 分散人物考课 | 新项目不为所有明制人物挂宏观逻辑 |
| 战争/战斗/攻城 | 军功与政治反馈 | 只在重大链或候选标记需要时读取 |
| interaction/decision | 玩家主动入口 | 使用独立 `minghm_*` 定义 |

## 3. 状态载体评价

- title 变量适合继承后仍存续的全国状态；
- 人物 trait/position/relation 是前置身份事实，新项目直接读取；
- global variable 容易串台，不用于活动临时值；
- saved scope 必须有死亡、失位、离廷和结局清理；
- 现成事件与 scripted effect 可能有隐藏依赖，不能只凭名称调用。

## 4. 性能与兼容风险

- 不照搬前置的全人物扫描、高频候选构建和公共 GUI 覆盖；
- 不建立人口清理器、地区缓存或第二套身份数据库；
- 新 on_action 文件使用独立子 ID 和 `minghm_*` 键；
- 同一事件只保留一个权威状态作用域，禁止人物和 title 双写。

现行初始化、调度、继承和复国合同见 `22A`。
