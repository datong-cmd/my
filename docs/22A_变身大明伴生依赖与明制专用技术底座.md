# 22A 《变身大明》伴生依赖与明制专用技术底座

## 1. 强制环境

| 层 | 固定要求 | 作用 |
|---|---|---|
| 游戏 | CK3 `1.19.0.6 (Scribe)` | 发布基线，升级必须重测 |
| DLC | All Under Heaven | 事件、美术和脚本样板来源 |
| 前置 | 《变身大明》1.0，Workshop `2713348525` | 提供 `h_greatming`、明制身份与美术 |
| 本项目 | 最后加载 | 只新增 `minghm_*` |

`descriptor.mod` 声明 `dependencies={ "变身大明" }`。不维护独立运行或无 DLC 降级分支。

## 2. 唯一国家载体

`V-M`：前置在 `common/landed_titles/99_yan_titles.txt` 定义 `h_greatming`，并在建国 effect 中把它交给皇帝。所有长期状态统一写在：

```text
title:h_greatming.var:minghm_*
```

不写 `h_china`、人物、区域头衔或 global variable。皇帝更替不搬迁全国状态。

## 3. 明制 trigger

`V-M`：前置已有 `is_GM_emperor`。本项目只包装：

```text
minghm_is_ming_emperor_trigger
minghm_is_ming_realm_ruler_trigger
minghm_is_ming_employed_official_trigger
minghm_is_ming_system_actor_trigger
```

- 皇帝：持有 `h_greatming`；
- 内部有地角色：`top_liege` 为明帝；
- 受雇官员：`employer` 或其 `top_liege` 为明帝，并有已验证的明制职位/流官信号；
- 不以文化、地理、华夏身份或天朝政体判定。

所有 decision、event、on_action 入口第一层必须通过上述包装 trigger。外国华夏角色、独立藩属和其他政体不得初始化任何 `minghm_*`。

## 4. 冻结数据

长期 title 数据仅有：

- schema/version；
- 八项全国力量：国家能力、改革动量、保守力量、民众动员、精英凝聚、军事自主、商业政治影响、危机压力；
- `minghm_historical_stage = 0..4`；
- `minghm_industrial_stage = 0..4`；
- 单政策、单案件、单危机和单重大链的状态字段；
- 必要的年度 marker、冷却和继承恢复标记。

不建立区域锚点、财政承诺、人口、商品或外国状态。

## 5. 初始化与调度

新增独立 on_action 文件，不改前置文件：

| 入口 | 根作用域 | 行为 |
|---|---|---|
| `on_game_start_after_lobby` | empty | 若 `h_greatming` 有 holder，则初始化/迁移一次 |
| `on_title_gain` | 新持有者 | 仅获得 `h_greatming` 时延迟 1—3 日初始化或继承恢复 |
| 年度 fallback | 明帝 | 旧档缺 schema 时初始化；否则只做一次低成本候选判定 |
| 战争结束 | 已验证战争作用域 | 仅更新战争相关候选或当前链固定节点 |
| 决议/事件结算 | 明帝或明制 actor | 推进单一活动槽 |

不注册全国月度宏观 tick。活动节奏优先由 `trigger_event` 延迟推进；只有已激活的普通政策/案件允许低频季度检查。

## 6. 生命周期

### 皇帝继承

- title 状态保留；旧皇帝上的展示 story 结束；
- 新帝获得 title 后重建玩家展示，继续当前阶段；
- 延迟读取前置最终官位，避免与前置同日初始化竞争；
- 无有效主持者时重新选择现有人物或退化成匿名机构。

### 大明无 holder

- 所有入口立即失败；活动设置休眠，不补算时间；
- 不把状态转移给征服者或其他中国皇帝。

### 复国

- 变量仍在则恢复并给一次“制度断裂”修正；
- 变量缺失则按固定默认值初始化；
- 不重建区域、人口、财政历史，也不逐年补算。

## 7. 上游读写

| 对象 | 规则 |
|---|---|
| 皇权 | 读取 `huang_quan_value`；消费只经已验证 `HQ_change` 适配器 |
| 官位/品级 | 读取现有 trait/position；不建平行官阶 |
| 朋党 | 读取关系与 `pengdang_yingxiang_score`；不改原党派 |
| 厂卫/司法 | 读取现任身份；不调用未证明安全的 `GM_Tanhe_events` |
| 军功/五军 | 读取现有军职与信号；不重置军功或建新军制 |
| 美术 | 引用 theme/background/icon 键；不复制二进制 |

调用任何前置 scripted effect 前必须确认根作用域、必需参数、隐藏副作用、幂等性和扫描成本；不能证明则不用。

## 8. 文件与命名

所有新键、namespace、flag、modifier、GUI type 和文件名使用 `minghm_`。可以放在同类目录，但不能与本体、DLC 或前置同文件名、同数据库键。禁止覆盖 HUD、角色窗、理事会窗、原转盘窗口和前置事件文件。

## 9. 首批探针

| ID | 验证 | 失败回退 |
|---|---|---|
| B01 | 依赖和加载顺序，无重复键 | 修 descriptor，不开发机制 |
| B02 | `h_greatming` 变量经继承/存读档保留 | 停止开发并重新选择稳定载体 |
| B03 | `on_title_gain` 只初始化一次 | 增加 marker/延迟 |
| B04 | 旧档年度 fallback 一次完成 | 加明帝手动修复决议 |
| B05 | 明制 actor 不误收外国 | 收紧 employer/top_liege 分支 |
| B06 | 皇权/职位/党争只读正确 | 关闭相应修正，保留基础数字 |
| B07 | story 在继承时安全结束与重建 | 关闭可见 story，逻辑仍在 title |
| B08 | 无 holder 零运行、复国只恢复一次 | 加休眠与恢复锁 |

B01—B08 通过后才可扩展批量事件。
