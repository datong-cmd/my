# 《变身大明》伴生依赖与明制专用技术底座

> 目标：给所有后续机制提供唯一、可执行的运行边界。本文中的路径和键已在本地 CK3 1.19.0.6、《溥天之下》和 Workshop 2713348525 中定位。

## 1. 依赖与加载顺序

### 1.1 强制环境

| 层 | 要求 | 本地证据 | 本项目行为 |
|---|---|---|---|
| 游戏 | CK3 `1.19.0.6 (Scribe)` | `launcher/launcher-settings.json` | 发布基线；升级后重测 |
| DLC | All Under Heaven | `game/dlc/dlc022_ep4/dlc022.dlc`；`has_tgp_dlc_trigger` | 强制；缺失不支持 |
| 前置 Mod | 《变身大明》1.0 | `descriptor.mod`，remote id `2713348525` | 强制；提供明制世界和资源 |
| 伴生 Mod | 本项目 | 新 descriptor | 最后加载，只新增 `minghm_*` |

推荐启动器顺序：

```text
1. 变身大明
2. 本项目：明代权力与社会转型扩展
```

DLC 由游戏管理，不在 Mod 顺序中重复列出。

### 1.2 Descriptor 施工要求

新 descriptor：

```text
version="0.1.0"
name="明代权力与社会转型：变身大明伴生扩展"
supported_version="1.19.0.6"
dependencies={
    "变身大明"
}
tags={
    "Events"
    "Decisions"
    "Gameplay"
}
```

`dependencies` 名称必须与前置 Mod descriptor 的 `name="变身大明"` 完全一致。发布说明同时写 Workshop ID，避免同名本地 Mod 误判。

### 1.3 不支持的启动方式

- 只开本项目；
- 只开 DLC 不开《变身大明》；
- 把本项目放在《变身大明》前；
- 与另一个重定义 `h_greatming` 或同名明制政体的大型 Mod 混用；
- 未测试的 CK3 小版本。

## 2. 明制核心对象

### 2.1 国家标题

`V-M`：《变身大明》定义：

```text
common/landed_titles/99_yan_titles.txt
h_greatming = { ... }
```

建国效果在：

```text
common/scripted_effects/yan_found_GM_effects.txt
title:h_greatming = {
    change_title_holder = ...
    set_variable = { name = GM_liguo value = current_year }
}
set_primary_title_to = title:h_greatming
```

因此 `title:h_greatming` 是最稳定的国家状态载体：皇帝死亡、改朝内战和人物替换不会删除标题对象上的变量。

### 2.2 皇帝判定

`V-M`：参考 Mod 已定义：

```text
common/scripted_triggers/99_yan_triggers.txt
is_GM_emperor = {
    has_title = title:h_greatming
}
```

本项目不重定义该 trigger，只包装：

```text
minghm_is_ming_emperor_trigger = {
    is_GM_emperor = yes
}
```

包装的意义是隔离未来前置 Mod 改名；所有本项目文件只调用 `minghm_*` 公共入口，adapter 内部才直接调用 `is_GM_emperor`。

### 2.3 明制内部角色

新建三个严格 trigger：

```text
minghm_is_ming_realm_ruler_trigger
minghm_is_ming_employed_official_trigger
minghm_is_ming_system_actor_trigger
```

逻辑合同：

| Trigger | 合法对象 | 判断 |
|---|---|---|
| `minghm_is_ming_realm_ruler_trigger` | 有地角色 | 自身是皇帝，或 `top_liege` 是皇帝 |
| `minghm_is_ming_employed_official_trigger` | 有地/无地官员 | employer 存在，且 employer 或 employer.top_liege 是皇帝；并具有参考 Mod 明制职位/流官标志之一 |
| `minghm_is_ming_system_actor_trigger` | 所有需要参与本项目的角色 | 上述两者之一；不能只凭华夏文化或中国地理区判断 |

外国华夏角色、清/八旗角色、独立藩属和失去大明雇主的前官员不自动获得明制状态。

## 3. DLC 门槛

`V-D`：本体 DLC trigger 在：

```text
game/common/scripted_triggers/10_tgp_triggers.txt
has_tgp_dlc_trigger = { ... all_under_heaven ... }
```

本项目 adapter：

```text
minghm_has_required_stack_trigger = {
    has_tgp_dlc_trigger = yes
    title:h_greatming ?= { exists = holder }
}
```

因为《变身大明》是 descriptor 强制依赖，不需要在运行时判断“是否安装参考 Mod”。若其数据库未加载，`title:h_greatming` 引用本身就会暴露错误；这属于加载配置错误，不设计独立降级逻辑。

## 4. 状态作用域

### 4.1 全国状态

统一写入：

```text
title:h_greatming.var:minghm_*
```

包括：schema、调度 marker、8 类集团、6 联盟、阶段、改革槽、项目槽、危机槽、预算承诺和国家聚合能力。

禁止写在：

- 当前皇帝角色：继承会丢失；
- `h_china`：建国效果可能销毁并复制历史；
- 任一总督/巡抚：任命变化会丢失；
- global variables：若未来多个大明历史实例/测试档并存会污染；仅构建版本号可用 global。

### 4.2 区域状态

首版建立 8 个 `title` anchor。锚点来自《变身大明》现有行政头衔，不新建地图、不改 de jure。每个锚点保存八轴和少量缓存：

```text
<anchor>.var:minghm_land_concentration
<anchor>.var:minghm_labor_coercion
<anchor>.var:minghm_commercialization
<anchor>.var:minghm_wage_labor
<anchor>.var:minghm_industrial_capacity
<anchor>.var:minghm_state_direct_control
<anchor>.var:minghm_transport_integration
<anchor>.var:minghm_extraction_pressure
```

具体 8 个 title key 必须在地图适配阶段从《变身大明》实际总督/巡抚头衔选定，并进入配置 scripted list；在选定前不得把“江南”等自然语言写死成无效 scope。

### 4.3 人物状态

人物只保存短期角色和记忆：主联盟、改革执行者、危机代表、承诺、镇压/背叛记忆。官职、品级、朋党、军功、皇权相关人物事实直接读取《变身大明》，不复制成 `minghm_*`。

## 5. 初始化入口

### 5.1 新游戏/已有大明

`V-B`：`on_game_start_after_lobby` 是 empty scope，可识别玩家和规则。新增文件：

```text
common/on_action/minghm_core_on_actions.txt
```

挂接行为：

```text
on_game_start_after_lobby
→ title:h_greatming ?=
→ holder ?=
→ minghm_initialize_or_migrate_state_effect
```

不得扫描所有独立统治者。

### 5.2 游戏中建立大明

`V-B/V-M`：本体 `on_title_gain` 存在；《变身大明》也通过独立文件向它添加 `on_title_gain_GreatMing`。本项目新增唯一子 on_action：

```text
minghm_on_title_gain
```

根 scope 为获得头衔的角色，第一层：

```text
has_title = title:h_greatming
```

成立后调用同一 `minghm_initialize_or_migrate_state_effect`。不修改 `yan_found_GM_effects`。

### 5.3 旧存档中途加本项目

`yearly_global_pulse` 已被参考 Mod使用，且根 scope 可检查 `is_GM_emperor`。本项目挂一个低成本 fallback：若是大明皇帝且 `title:h_greatming` 未初始化，则初始化。初始化后只做一次 `has_title + flag` 判断。

## 6. 调度入口

| 频率/事件 | 根 scope | 第一层过滤 | 调用 |
|---|---|---|---|
| 月度 | 大明皇帝 | `is_GM_emperor` + initialized + 活动截止存在 | 只处理期限/锁 |
| 季度 | 大明皇帝 | 同上 | 改革、危机、AI 议程 |
| 年度 | 大明皇帝 | 同上 + year marker | 8 区域→阶级→联盟→阶段 |
| 五年 | 年度内判断 | `current_year % 5` 的已验证替代或 marker | 人物池和慢变量校准 |
| 获得 `h_greatming` | 新 holder | `has_title` | 初始化/迁移/继承摘要 |
| 皇帝死亡 | 死者 | `is_GM_emperor`（死亡时 scope 行为需探针） | 只记录继承窗口；主恢复靠 title gain |
| 战斗/攻城/战争结束 | 参与角色/战争 | 能追溯大明阵营 | 军功/军费/危机增量 |
| 活动完成 | activity owner | 是大明皇帝或明制官员 | 科举/巡察反馈 |

月/季/年入口必须复用一个 `title:h_greatming` 调度器，不能给每个明制角色注册宏观 tick。

## 7. 生命周期

### 7.1 皇帝继承

- 主状态留在 `h_greatming`，不搬迁；
- 新皇帝获得 title 后刷新皇帝 saved scope、联盟关系和承诺继承；
- 参考 Mod 自行处理年号、流官、内阁等；本项目延迟 1—3 日读取最终职位，不抢同日执行顺序；
- 活动改革不自动取消，但 legitimacy、路线惯性和承诺需要重新审核；
- 只弹一次“新朝议程”摘要。

### 7.2 大明灭亡/标题无 holder

- 所有周期 trigger 自动失败；
- `h_greatming` 上变量保留；
- 改革、项目、危机槽设置 `suspended`，不继续计时扣费；
- 不把本项目机制转移给征服者；
- 外国不继承阶级/阶段状态。

### 7.3 大明复国

- 若 schema 一致，恢复原状态并结算中断年数为“制度断裂”，不逐年补算；
- 若标题变量被参考 Mod 重建后仍在，继续使用；
- 若缺失，按当前明制地图重新初始化；
- 复国本身增加国家危机、精英分裂和军事集团力量，不给予免费稳定。

## 8. 与参考 Mod 的读写边界

### 8.1 默认只读对象

| 对象 | 读取方式 | 禁止 |
|---|---|---|
| 皇权 | `08A` 已定位的变量/value/trigger | 直接 change 内部变量 |
| 军功 | `jungong` 等人物变量/既有 effect | 重置或另建平行军功 |
| 朋党 | 关系、党魁、党派影响缓存 | 把六联盟写回四党 |
| 流官品级/职位 | traits、court/council positions、记忆 | 自建重复官阶 |
| 锦衣卫/案件 | 既有互动、事件、罪名和诏狱状态 | 复写同事件或修改原罪名 |
| 国库 | treasury 原生接口 | 建第二套国库余额 |
| 明制政体 | government flags/keys | change 成新重复政体 |

### 8.2 可以调用的公开面

只有在专项施工文档确认后，才允许调用参考 Mod 已有 scripted effect。调用前检查：

1. 根 scope；
2. 必需 saved scopes/参数；
3. 是否写入超出预期状态；
4. 是否触发 GUI/事件链；
5. 是否幂等；
6. 是否存在高成本扫描。

若无法证明安全，复制其算法思想并用 `minghm_*` 新效果实现，不能复制原键或整段代码。

## 9. 文件层兼容规则

新增文件示例：

```text
common/on_action/minghm_core_on_actions.txt
common/scripted_triggers/minghm_core_triggers.txt
common/scripted_effects/minghm_core_effects.txt
common/script_values/minghm_core_values.txt
events/minghm_core_events.txt
```

文件可以位于与参考 Mod相同目录，但文件名不得相同。数据库定义键、事件 namespace、变量、flags、modifiers、GUI types 和资源名全部使用 `minghm`。

## 10. 首批技术探针

| ID | 探针 | 成功证据 | 失败处理 |
|---|---|---|---|
| B01 | descriptor 依赖/加载顺序 | 两 Mod 同启、无未定义 `h_greatming` | 修正依赖名和顺序 |
| B02 | title 变量持久 | `h_greatming` 变量经继承/存读档保留 | 改用另一稳定 title；不进入机制开发 |
| B03 | on_title_gain 初始化 | 建国后只初始化一次 | 增加 marker/延迟，不修改前置 Mod |
| B04 | 旧档年度 fallback | 中途加 Mod 后一年内初始化一次 | 增加玩家 decision 手动初始化 |
| B05 | 大明内部 actor trigger | 皇帝、总督、官员命中；外国不命中 | 调整 employer/top_liege 分支 |
| B06 | 参考职位/变量只读 | 能读取且不改变前置 Mod 状态 | 改用 trait/position 替代信号 |
| B07 | DLC 国库/功绩/影响力 | 在大明环境中读取正确 | 记录前置 Mod 对原生资源的改写 |
| B08 | 休眠/复国 | 无 holder 时零 tick，复国恢复一次 | 加 suspended 状态和恢复锁 |

完成 B01—B08 后，明制专用底座才视为可用。

## 11. 验收

- [ ] 启动器明确要求并按顺序加载《变身大明》；
- [ ] 本项目没有定义任何前置 Mod 已有标题、政体、官职或资源键；
- [ ] 大明不存在时不产生本项目变量、事件或周期成本；
- [ ] 大明皇帝、内部统治者和受雇官员的 trigger 不误收外国华夏角色；
- [ ] 全国状态只在 `h_greatming`；
- [ ] 皇帝继承不搬迁全国状态；
- [ ] 大明灭亡后全部休眠，征服者不继承玩法；
- [ ] 复国不逐年补算导致事件风暴；
- [ ] 两 Mod 同启 error.log 无新增重复键、未定义 scope 或同路径覆盖；
- [ ] 首批探针全部通过后才开始具体机制实现。
