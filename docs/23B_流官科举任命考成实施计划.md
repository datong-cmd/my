# 23B 流官—科举—任命—考成实施计划

## 1. 兼容原则

科举、品级、授官和原有考成继续完全由《变身大明》负责。本模组只回答两个新增问题：现有官僚结构能否执行某项改革；皇帝如何在能力、忠诚、出身与利益代表之间选择改革团队。

直接读取：`juren`、正从一至九品特质，大学士特质，六部/都察院/锦衣卫/司礼监职位，以及原模组已经给出的职位结果。禁止覆盖 `keju_events`、`liuguan_events`、`shouguan_events`、`liuguan_kaoke` 或 `succession_appointment` 文件。

## 2. 有界官僚快照

每年一次或发起政策时，从明帝宫廷抓取：关键官位持有人、当前廷臣中至多 12 名符合 `juren`/品级/官职条件的候选人、已任命的改革专员。候选人按 scripted value 排序/随机加权抽取，不扫描全体在世人物。

快照只在发起政策时即时计算，并作为当前政策槽的临时值保存；政策结束由统一清理 effect 删除。它们不是 `24/27A` 冻结数据字典之外的长期状态：

| 变量 | 0..100 含义 |
|---|---|
| `minghm_bureau_competence` | 行政/学识与品级经验 |
| `minghm_bureau_cohesion` | 立场一致与协作能力 |
| `minghm_bureau_capture` | 官僚对既得利益依附 |
| `minghm_bureau_information` | 中央获得真实反馈的能力 |
| `minghm_bureau_rotation_cost` | 换人和制度学习损失 |

人物的社会倾向是临时计算值，不永久贴“阶级本质”标签：只由现有官位职责、性格、党派、与皇帝关系和本次政策直接得失决定，不读取土地、财富或商贸账户。

## 3. 改革专员

用新的角色互动 `minghm_appoint_reform_commissioner_interaction` 选人，只授予期限 3 年的 `minghm_reform_commissioner` modifier 和 saved scope，不占用或篡改原有官位。

候选评分：

`能力 35% + 政策适配 25% + 对皇帝忠诚 15% + 官署协调 15% − 利益冲突 10%`

玩家看到前四名及主要优缺点。选择亲信可降低政治风险但不保证执行能力；选择能员可能增强其党派和个人政治资本。人物死亡、离廷、入狱或不再属于明制内部时自动解除并要求补任。

## 4. 考成反馈而非第二套考成

每项本模组政策结束后产生一次“项目考成”：

- 充分落实：专员获得执行声望和政治资本；官僚能力小幅积累。
- 折衷落实：按玩家事前接受的妥协分配功过。
- 纸面落实：提升信息失真/官僚俘获；可开启调查。
- 失败：区分能力不足、政治阻挠、资源不足，避免一律惩罚个人。

这些效果只使用本模组 modifier、威望/关系和宏观值；不直接升降原模组品级。若玩家要正式罢免或授官，仍使用《变身大明》界面和规则。

## 5. 任命与政策博弈

主持人并非越强越好：高阶重臣有协调权但可能代表既有网络；低阶新进缺乏执行链但更愿承担激进方案。每次任命会改变：

- `policy_capacity`：能力和官署匹配；
- `policy_support`：其党派、人脉与皇帝关系；
- `policy_resistance`：受损官署和利益力量反应；
- `bureau_rotation_cost`：频繁换人造成的损耗。

连续撤换专员会叠加“朝令夕改”，防止玩家无代价刷最优人选。

## 6. 技术文件与钩子

```text
common/character_interactions/minghm_reform_interactions.txt
common/scripted_values/minghm_bureaucracy_values.txt
common/scripted_effects/minghm_bureaucracy_effects.txt
common/modifiers/minghm_bureaucracy_modifiers.txt
events/minghm_bureaucracy_events.txt
```

不用修改原科举事件来“发毕业通知”。年度快照自然吸收新科举结果；本模组互动产生的任免立即刷新一次有限快照。原模组若未来提供稳定 on_action，再经版本探针加入可选适配，不把它作为 MVP 前提。

## 7. 验收

- 科举/任官照常运行，新进人物最迟下一年度进入候选池。
- 品级、官位、考试特质均只读；无重复职位或品级特质。
- 候选池上限生效，年度性能不随全世界人口线性增长。
- 专员死亡、继承、离廷和政策结束均无悬挂 saved scope。
