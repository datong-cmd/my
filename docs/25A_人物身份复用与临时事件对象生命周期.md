# 25A 人物身份复用与临时事件对象生命周期

## 1. 技术结论

CK3 的 `create_character` 创建真实角色；把他以 `death = { death_reason = death_disappearance }` 结束，只会变成死亡人物，仍保留在存档谱系中，不等于释放对象。参考模组也使用该模式，例如：

- `events/yan_nvguan_events.txt`：`create_character` 后以 `death_disappearance` 结束未采用的相亲对象；
- `common/character_interactions/99_yan_qita.txt`：创建替身人物并保存作用域。

因此“事件结束后释放”必须主要理解为：清除 saved scope、flag、modifier、变量和事件槽；不能把不断创建/杀死角色当性能方案。

## 2. 角色来源优先级

每个事件按以下顺序找参与者：

1. 《变身大明》现有关键官位持有人；
2. 明帝现有廷臣中带原模组品级、科举、朋党、军职、宦官、锦衣卫身份的人；
3. 明制内部现有封臣/官员中按条件抽取至多 4 名候选人；
4. 无需肖像的“匿名群体”，仅用事件文本、固定插图和全国数字表示；
5. 只有必须在后续事件持续互动的独特领袖，才允许 `create_character`。

禁止为了“商人代表、农民代表、工匠代表、士绅代表”等普通发言者各生成角色。普通集团是数值和本地化叙事，不是 CK3 人物。

## 3. 复用的原模组身份

| 用途 | 读取对象 |
|---|---|
| 文官层级 | 正从一至九品特质、`juren`、大学士身份 |
| 中央官署 | 六部、都察院、大理寺、通政司等原职位 |
| 朋党代表 | `pengdang/dangkui/dangtu` 关系和党魁影响力 |
| 内廷 | `huanguan`、`chief_eunuch_court_position` |
| 监察强制 | `jinyiwei`、`court_jinyiwei_position` |
| 军事行动者 | 原模组军职、五军都督府和军功身份 |

本模组只增加短期职责 modifier：主持改革、公开反对、调查主办、危机代表。modifier 必须有期限并在结束 effect 再次显式清除。

## 4. 临时状态清单

活动开始允许创建：

- `minghm_*_actor` saved scopes，最多 8 个；
- `minghm_event_role_*` character flags，带期限；
- `minghm_*_temporary` character modifiers，带期限；
- `h_greatming` 上当前政策/危机/案件变量；
- 事件局部临时变量。

统一清理 `minghm_cleanup_active_content_effect`：

1. 对仍存在的 saved scope 人物移除本模组 flags/modifiers；
2. 清除 title 上全部活动槽变量与 saved scopes；
3. 保留八项长期数值、历史阶段和已完成内容 flag；
4. 清除互斥锁；
5. 写入一次 debug 完成标记，下一季度移除。

清理 effect 必须从成功、失败、取消、皇帝死亡、目标死亡、大明失主六条出口调用。年度维护只兜底清理“锁存在但活动类型不存在”的孤儿状态。

## 5. 极少数生成角色的规则

只有满足全部条件才创建：

- 是重大危机/革命链的命名领袖，而非一次性群众代表；
- 未来至少三个事件需要其人物属性、关系或肖像；
- 当前找不到合适的现有人物；
- 全国此类本模组生成角色存活不超过 2 名；
- 创建后有 `minghm_generated_major_actor` flag，纳入死亡/离廷清理。

创建模板直接用明帝/明文化作用域：`culture = root.culture`、`faith = root.faith`、`employer = root`、`dynasty = none`，只给必要技能和本体性格，不复制复杂家族。不得每次打开事件重新创建。

事件链结束时：

- 若角色已成为有官职、婚姻、领地或重要关系的真实政治人物，则保留，仅移除本模组临时状态；
- 若仍是无关系廷臣，可 `move_to_pool = yes`，但不宣称其已从存档释放；
- 不以强制死亡作为通用垃圾回收，避免死亡人口和谱系膨胀。

## 6. 无角色事件表现

匿名社会力量使用本体/DLC/参考模组已有背景、主题、音效和图标：左侧显示明帝或主持官，右侧可无肖像；文本称“地方士绅的联名”“军中诸将的意见”“京师商人的请愿”。数值提示承担身份区分，不为每种群体制作 trait 或 3D。

## 7. 验收

- 连续触发 100 次普通政策/危机事件，角色总数不因本模组线性增加；
- 活动结束后无 `minghm_event_role_*`、临时 modifier 或 saved scope 悬挂；
- 生成领袖上限 2 生效，且同一链不会重复创建；
- 原模组职位和身份只读，没有重复 trait/关系定义；
- 皇帝死亡和大明失主后，下一次维护能完成孤儿状态清理。
