# 22 施工版冻结原则

> 本文是现行设计的最高层边界。仓库已删除与本边界冲突的早期经济、区域、人口、跨政权和通用框架文档。

## 1. 产品形态

固定加载组合：CK3 `1.19.0.6` + All Under Heaven +《变身大明》（Workshop `2713348525`）+ 本伴生扩展。本项目最后加载，只服务 `h_greatming`；前置或 DLC 缺失时不支持运行。

## 2. 必须满足的实现门槛

每项进入编码队列的机制必须同时具备：

1. 可定位的本体、DLC 或《变身大明》脚本载体；
2. 明确的 `h_greatming` 入口和非明零运行规则；
3. 新数据的作用域、范围、更新频率、初始化和清理；
4. 玩家选项、AI 选择、成功/妥协/失败结果；
5. 现成 UI、背景、图标或角色身份的复用路径；
6. 性能上限与失败降级；
7. 可执行测试。

只有概念、史论或公式而无游戏载体的内容不得编码。

## 3. 唯一允许的新增模型

- `h_greatming` 上八项 `0..100` 全国政治力量；
- 五档历史阶段和五档工业组织阶段；
- 政策、案件、危机和重大事件链共用一个“前台复杂内容锁”；各自仍有独立字段，但同一时间只有一种内容推进；
- 少量角色临时 flag/modifier/saved scope；
- 明制专用决议、事件、scripted trigger/value/effect；
- 一个不覆盖公共 GUI 的只读局势转盘事件 widget；
- 经探针验证后才允许启用的独立革命战争桥。

## 4. 明确不实现

- 财政、税种、国库预算、内帑账户；
- 区域产量、土地、矿业、作坊、商品、物流、市场和价格；
- 人口、职业、迁徙、逐领地阶级结构；
- 其他国家、天朝通用化、外国宏观状态、朝贡和殖民体系；
- 新政体、新官阶、新文化/信仰、建筑、兵种和科技树；
- 全国 situation、activity 或 scheme 状态机；
- 全地图、全人物、逐领地循环和常态月度宏观结算；
- 公共 GUI 覆盖、3D 复制和大批新 2D。

相关历史矛盾只能用全国政治数值、匿名群体文本和事件结果表达。

## 5. 参考模组读写边界

直接读取：`h_greatming`、`is_GM_emperor`、皇权、官位、品级、朋党、厂卫、军功和现有人物。默认不写上游变量，不复制其身份和数据库键。只有专项文档确认根作用域、副作用和版本探针后，才可调用公开 scripted effect。

## 6. 人物与美术

- 优先使用前置模组已有官员、军职、宦官、锦衣卫和党争人物；
- 普通阶级与社会组织使用匿名群体，不创建人口角色；
- 最多同时保存 3 个重大链角色作用域，单活动候选上限 12；
- 只有无法用现有人物承载的长期危机领袖可生成，存活上限 2；CK3 角色死亡后仍留存档，因此不宣称“释放角色”；
- 3D、服装、建筑、兵器、场景全部直接引用已加载资源；MVP 新 2D 为 0。

## 7. 性能原则

- 无活动时年度一次简单判定；活动使用延迟事件或低频季度结算；
- 全国最多一个前台复杂内容；重大链运行时暂停普通政策/案件/危机结算，每链 6—10 个固定事件、最多 4 个条件插曲；
- 禁止 `any_living_character` 和全地图循环；
- 数字只在入口、固定阶段和结局重算；
- 性能不达标时删减参与者、频率或分支，不追加缓存层掩盖复杂度。

## 8. 证据等级

| 标记 | 含义 | 编码权限 |
|---|---|---|
| `V-B` | 本体脚本已定位 | 可按样板实现，仍需实机验证 |
| `V-D` | DLC 脚本/资源已定位 | DLC 强制启用时可用 |
| `V-M` | 《变身大明》对象已定位 | 前置加载时可读 |
| `N` | 本项目新增，已有语法样板 | 可实现 |
| `P` | 仍需最小探针 | 只写探针和安全回退 |
| `X` | 明确弃用 | 不得实现 |

## 9. 权威顺序

编码入口为 `27A`。实现边界依次为 `22/22A/24/25A/27`，机制规格为 `23—26B`、`29—30G`，源码事实由 `01—10` 按需反查。任何旧提交、聊天摘要或外部草稿与此冲突时，以当前仓库为准。
