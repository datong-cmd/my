# 重大事件与危机博弈框架

> 文档性质：所有重大政治、财政、军事、灾害和社会事件的统一运行规范  
> 目标：事件不直接讲完历史，而是暴露矛盾、迫使联盟投入资源，并把结果写回阶级、制度和区域结构。  
> 明代情景模板见 [16A](16A_重大事件族与明代情景模板.md)。

## 1. “事件”和“危机”不同

普通事件是一项局部选择或反馈；重大危机是一组持续状态、参与者、资源、行动和可能结算。灾荒、继承、战争或财政缺口只有在改变联盟、国家能力或生产关系时，才升级为重大危机。

```text
冲击 ≠ 危机
危机 = 冲击 × 既有脆弱性 × 有组织的利益冲突 × 国家应对
```

同一旱灾在仓储、财政和地方合作良好时只是赈济事件；在银税、债务、欠饷、土地集中和党争同时存在时，可能触发逃亡、军粮中断、民变与政权危机。

## 2. 七阶段危机状态机

### C0：压力积累

结构变量越过安全区，但社会尚未把问题识别为全国议题。只显示弱提示或年度报告。

### C1：信息暴露

巡察、战败、价格、奏疏、事故、举报或民众行动使问题公开。参与者争夺“问题是什么”和谁应负责。

### C2：议程形成

宫廷、部院、地方、军队、商人和民众组织提出方案；玩家决定调查、任命、预算和谈判，而不是立即选择最终结局。

### C3：资源动员

联盟投入国库、影响力、军队、信用、粮食、舆论和组织。承诺写入状态，未来必须兑现。

### C4：公开对抗

法令受阻、罢市、抗税、弹劾、兵变、内战或外敌进攻。参与者可改变阵营，局部结果开始不可逆。

### C5：结算与制度选择

形成妥协、镇压、改革、赔款、换阁、军管、自治、革命或分裂。结算分别处理成本承担、权力分配和制度改变。

### C6：记忆与后效

危机退出 UI 主阶段，但保留债务、仇恨、成功经验、组织承认、工程损坏、人口迁徙和政策先例。数年后同类问题按新结构重现，不重复原脚本。

## 3. 危机对象的数据字段

每个危机在设计和代码中至少有以下字段：

| 字段 | 含义 |
|---|---|
| `crisis_type` | 财政、战争、继承、灾害、党争、劳动、边疆等类型 |
| `scope` | 国家、区域、头衔、战争、机构或政治联盟 |
| `origin` | 最初冲击和结构脆弱性 |
| `stage` | C0—C6 |
| `severity` | 实际损害和紧迫性，0—100 |
| `visibility` | 国家和公众掌握问题的程度，0—100 |
| `polarization` | 方案与集团对立程度，0—100 |
| `state_capacity` | 国家解决问题的财政、人员、信息和强制能力 |
| `social_capacity` | 地方、商人、军队和民众组织的自救/对抗能力 |
| `participants` | 发起者、承担者、受益者、执行者、否决者和调停者 |
| `commitments` | 预算、职位、赦免、地权、特许、工资和政治承诺 |
| `clocks` | 截止期、季节、军粮、疾病、饥饿或继承时间窗 |
| `escalation_paths` | 当前阶段允许的升级条件 |
| `settlement_paths` | 可达妥协、改革、镇压、战争和分裂 |
| `memory_flags` | 影响未来 AI 和事件的成功、背叛、镇压和制度先例 |

危机不保存所有参与者列表。使用 situation 参与组、临时 scope 和最多若干关键角色；群众与区域支持用聚合值。

## 4. 六个危机仪表

玩家界面最多展示六项：

| 仪表 | 作用 | 高值不必然是坏事 |
|---|---|---|
| 严重度 | 生计、财政、军队和秩序实际受损 | 高值迫使快速决策 |
| 可见度 | 问题、账目和责任被掌握 | 高值提高改革可能，也提高政治冲突 |
| 极化度 | 各方案不可兼容程度 | 高值使妥协成本上升 |
| 国家应对力 | 财政、官僚、情报和军队 | 可用于赈济，也可用于镇压 |
| 社会组织力 | 地方、商会、村社、军镇和劳动组织 | 可协作救灾，也可挑战国家 |
| 时间压力 | 季节、军粮、债务、继承和外敌期限 | 高值缩短谈判并增加错误 |

隐藏变量只用于计算，不能让玩家猜一串不可见门槛。

## 5. 参与者角色

同一个集团可同时扮演多个角色，但重大危机必须明确：

| 角色 | 权利/责任 | 典型主体 |
|---|---|---|
| 受害/承担者 | 承受损失、税役、失地、欠饷或伤亡 | 小农、劳动者、军户、地方城市 |
| 受益者 | 从现状、危机或应对方案获利 | 地主、特许商、军需商、内廷机构 |
| 决策者 | 能通过法令、预算、任命和战争 | 皇帝、内阁、部院、议政机构 |
| 执行者 | 掌握地方信息、官僚、军队或组织 | 督抚、县官、将领、会馆、村社 |
| 否决者 | 能拖延、隐匿、罢市、拒税或兵变 | 士绅、商人、军队、政治运动 |
| 调停者 | 提供信用、合法性、妥协和通信 | 太后、宗室、清望官、商会、宗教/地方领袖 |

玩家若只控制决策者而没有执行者和承担者合作，法令会变形。

## 6. 可投入资源

| 资源 | 能解决什么 | 使用后的政治后果 |
|---|---|---|
| 国库 | 赈济、军饷、工程、补偿 | 预算挤压、债务和受款集团支持 |
| 私人黄金/内帑 | 快速救急、绕开预算 | 公私边界、恩庇和宫廷权力 |
| 影响力 | 任命、议程、联盟和交易 | 形成债务与职位承诺 |
| 功绩/信誉 | 让执行者和公众相信方案 | 失败会损害制度信用 |
| 军队/强制 | 镇压、护运、征收、抗敌 | 伤亡、军队政治化和民众记忆 |
| 粮食/物流能力 | 赈灾、供军和城市供给 | 控制仓储者获得谈判力 |
| 商业信用 | 借款、采购和运输 | 债权人获得合同、利息或代表 |
| 信息 | 识别对象、降低浪费与欺诈 | 揭露责任者并提高极化 |
| 社会组织 | 分发、互助、请愿、罢市和动员 | 组织获得合法地位和未来行动能力 |
| 外交/朝贡 | 援军、贸易、粮食和停战 | 产生保护、贸易、领土或名分义务 |

危机资源不是消耗后清零的抽象点。投入者会要求回报，拒绝兑现会形成背叛记忆。

## 7. 决策不是三色按钮

每个重大选择由四层构成：

```text
目标：要缓解什么
工具：钱、军队、法律、市场、组织或外交
承担：谁支付成本
治理：谁执行并监督
```

例如“赈济”至少可分：国库平粜、地方仓储、商人采购、强制征粮、减免税役和村社互助。它们都降低饥饿，但对国库、价格、商人、地主、小农和地方组织的影响不同。

### 7.1 选项必须显示

- 立即成本和时间；
- 所需执行者；
- 预期受益/受损集团；
- 成功区间而非伪精确百分比；
- 可能变形；
- 新承诺与冷却；
- 对危机仪表的方向影响。

## 8. 谈判与承诺

重大危机的核心是交换。常见承诺：

- 预算份额、俸饷、税役减免；
- 官职、科举名额、地方自治、审计豁免；
- 商业特许、关税、合同、债务偿还；
- 土地、减租、赈济、工资和劳动条件；
- 赦免、退兵、军功、军队整编；
- 政治代表、议政席位、公开调查和问责。

承诺有期限、受益者和验证条件。兑现提高合作与制度信用；违约提高不满、组织和未来谈判成本。不能用一次好感修正代替长期义务。

## 9. 升级逻辑

危机升级取决于损害、组织和应对，而非固定随机概率：

```text
escalation_pressure
  = severity
  + time_pressure
  + grievance
  + polarization
  + failed_commitments
  - relief_effect
  - credible_concessions
  - cooperative_capacity
```

强制镇压另算：

```text
repression_success
  = loyal_coercion + intelligence + isolation_of_opposition
  - opposition_organization - terrain/logistics - elite_split
```

镇压成功降低公开行动，但会提高长期记忆、合法性损失和地下组织；若配合实际让步，后效较轻。

## 10. 多维结算

结算不只写“成功/失败”，至少分别记录：

| 维度 | 问题 |
|---|---|
| 生计 | 饥饿、伤亡、工资、土地和迁徙是否改善 |
| 财政 | 谁花钱、谁负债、税基和信用如何变化 |
| 国家能力 | 信息、人员、预算和军队是否加强 |
| 阶级力量 | 哪些集团增加物质基础、组织和国家入口 |
| 制度 | 法律、契约、产权、任命和代表是否改变 |
| 地域 | 核心/边疆、城市/农村、沿海/内地如何分化 |
| 合法性 | 谁被认为解决、利用或制造危机 |
| 记忆 | 哪些承诺、背叛、镇压和先例进入未来事件 |

玩家可以“财政成功但民生失败”“战争胜利但军头化”“改革通过但地方转嫁”，这正是重大事件的长期博弈价值。

## 11. 危机耦合

同时最多显示 2 个全国重大危机和 2 个区域重大危机，其他压力进入队列或合并。危机之间通过桥接效果耦合：

| 主危机 | 可能引发 |
|---|---|
| 战争 | 军费、欠饷、粮价、难民、边疆自治、技术投资 |
| 灾荒 | 税役危机、土地兼并、迁徙、民变、信用紧缩 |
| 继承 | 组阁、军队忠诚、边疆试探、改革承诺重议 |
| 财政 | 加派、矿税、借款、俸饷、工程停工和党争 |
| 清丈/税改 | 士绅反制、地方舆论、小农负担、官僚考成 |
| 工业化 | 粮价、劳动、信用、区域不平等、军工政治 |
| 革命 | 外部干预、供给、军队裂解、复辟和地方分裂 |

耦合只传递结构变量和少量事件，不让每对危机交叉生成独立事件链。

## 12. 信息不完全

玩家不应默认知道真实田亩、灾情、军额、仓储、商人资产和民众支持。

- 报表显示估计区间和信息质量；
- 不同来源可冲突：地方官、巡按、商人、军队和民众组织各有偏差；
- 调查提高可见度，但给被调查者时间隐匿和结盟；
- 快速决策减少损害，错误目标风险高；
- 公开信息提高社会监督，也可能加剧挤兑、抢购和政治极化。

虚假信息不是纯随机。上报者根据考成压力、利益、恐惧和联盟立场选择隐瞒、夸大或如实报告。

## 14. CK3 技术载体

| 需求 | 载体 |
|---|---|
| 长期全国危机 | 独立 situation 或社会转型 situation 子阶段 |
| 单一危机任务链 | story cycle / character or title variable |
| 区域状态 | 区域锚点变量 + county modifier |
| 行动者组 | situation participant group、关系、家族和职位 |
| 谈判 | character interaction、event、contract、law |
| 调查 | activity_survey、scheme、task contract |
| 截止期 | variable date、cooldown flag、quarterly pulse |
| 战争升级 | faction、custom CB、war contribution |
| 工程解决 | great project、building、treasury |
| 后效记忆 | character memory、title variable、global history log |

不为每个危机新造全屏 GUI。优先使用 situation 主界面、事件、重要行动和提示；只有多危机总览确有必要时才做独立 scripted GUI。

## 15. AI 选择

```text
option_score = immediate_survival
             + class_and_family_gain
             + office_and_alliance_gain
             + ideology_match
             + feasibility
             + expected_public_credit
             - resource_cost
             - personal_risk
             - empowered_rival_risk
             - broken_commitment_cost
```

AI 依据自身情报评估，不读取隐藏真实值。危机中允许改主意，但要付信誉、联盟和沉没成本。强硬、谨慎、贪婪、仁慈等性格改变权重，不绕过物质利益。

## 16. 事件节流与性能

- 同类重大危机国家冷却 8—15 年，但结构压力可保留；
- 同一危机每季度最多一个主动弹窗，其他变化进入日志；
- 非玩家 AI 用聚合结算，只有邻国、宗主、盟友和重要对手展开完整事件；
- 参与人物最多 20—40 名重要角色，群众用区域值；
- 区域事件每年每区最多一个，且不保证弹窗；
- 战争占领和灾害按季度/年度批处理，不逐日触发；
- 危机结束清理临时 scope、参与组和无用变量，保留摘要记忆；
- 任何 `every_living_character`、全世界头衔或全地图 barony 扫描禁止进入常规脉冲。

## 17. 错误恢复与存档

危机每阶段都有幂等初始化和清理：

- 缺失领袖时重新选举/任命，不销毁危机；
- 国家分裂时把危机移交合法继承头衔或拆成区域后效；
- 主头衔销毁时从稳定全局/替代锚点恢复摘要；
- DLC 缺失或版本不兼容时不初始化；
- 危机变量越界时钳制到 0—100并写调试日志；
- 重复事件用唯一阶段标记防止多次扣款和多次授予头衔。

## 18. 样例：财政—军饷危机

### C0

军费份额、债务压力和欠饷同时上升；只在年度财政报告提示。

### C1

边军上书、军需商停止垫款或一次小规模哗变使问题公开。地方官、户部、将领和商人给出不同数字。

### C2

方案：削减其他预算、加派、借商款、整顿空额、停战、内帑拨款或军队缩编。每项确定承担者。

### C3

玩家必须取得部院、宫廷、商人或将领至少两方合作；承诺利息、职位、审计或军功。

### C4

若时间钟耗尽，军队罢战、哗变、劫掠或投向地方将领；若强征，农村生计和民变压力上升。

### C5

结算可能是国家化足饷、军商债权国家、军头自治、裁军妥协、战争失败或军事政变。

### C6

记录谁垫款、谁违约、哪些军队可靠、哪种预算先例成立。下一次战争直接读取这些记忆。

## 19. 验收清单

- [ ] 重大危机按 C0—C6 运行，普通冲击不自动升级；
- [ ] 危机对象有来源、范围、参与者、承诺、时间钟、升级和结算；
- [ ] UI 只展示六项可决策仪表并能解释变化；
- [ ] 决策包含目标、工具、承担和治理四层；
- [ ] 每个承诺有对象、期限、验证和违约后果；
- [ ] 镇压降低短期行动但留下合法性和组织记忆；
- [ ] 结算分别处理生计、财政、国家、阶级、制度、地域、合法性和记忆；
- [ ] 多危机通过结构变量耦合，不生成组合爆炸；
- [ ] 信息来源有利益偏差，玩家看见区间而非真实上帝数值；
- [ ] AI 不读取隐藏真值，并考虑物质、联盟、思想、风险与承诺；
- [ ] 同类事件有冷却、弹窗节流和参与者上限；
- [ ] 危机支持领袖死亡、国家分裂、主头衔变化和重复加载恢复。
