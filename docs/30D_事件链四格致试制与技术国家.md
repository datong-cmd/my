# 事件链四：格致试制与技术国家

> 把“再工业化”抽象成知识组织、标准化和试制制度的政治斗争；不模拟产量、商品、市场、工厂或财政。

## 1. 触发与持续时间

- 入口一：事件链三以“技术—训练转向”收束；
- 入口二：玩家决议“设格致试院”，且改革动量不低于 45、国家能力不低于 40、工业阶段低于 4；
- AI 入口：满足条件后按年低频评估，且全国没有重大事件链；
- 预期持续 4—8 年，事件间隔 180—360 天；只在 `h_greatming` 与现任明帝有效。

## 2. 核心矛盾与数字

- `support`：官僚、军方和士人支持；`resistance`：保守抵制、部门争权与利益俘获；
- `commitment`：玩家承诺投入，取 30/60/90 三档；
- `route`：1 标准度量、2 军器操练、3 译制传播、4 综合试院；`stage`：0—6。

```text
支持 = 改革动量 + 国家能力/2 + 工业阶段×10
阻力 = 保守力量 + 军事自主/3 + 路线难度
```

所有除法向下取整并钳制到 0—200。数字表现力量对比，不代表真实产能。

## 3. 固定事件骨架

| ID | 展示 | 内容与选择 |
|---|---|---|
| `minghm_technical_chain.4000` | 主题化角色事件 | 《格致奏议》：接受、暂缓或驳回；确定初始承诺 |
| `.4010` | 普通角色事件 | 以翰林、部院官或武职主持试院，保存 `lead` |
| `.4020` | 普通角色事件 | 选择标准度量、军器操练、译制传播或有限综合 |
| `.4030` | 书信事件 | 首次失败报告：隐瞒、复验、公开纠错 |
| `.4040` | 普通角色事件 | 士人、军官与承办者争夺定义权 |
| `.4050` | 普通角色事件 | 皇帝检阅，以数字门槛判定演示成败 |
| `.4060` | 书信事件 | 事故、泄密或冒功三选一插曲 |
| `.4070` | 普通角色事件 | 知识归属：个人秘术、部院成例、公开程式 |
| `.4080` | 普通角色事件 | 有真实战争则读取记录，否则进行抽象校验 |
| `.4090` | 主题化角色事件 | 《试院定制》，结算阶段、力量与承诺 |

每次事件只调整 2—4 个全国值，每项通常为 5/10/15；不创建建筑、单位或器物。

## 4. 结局

`净胜 = 支持 + 承诺/2 - 阻力`：

- `>=80` 制度化成功：工业阶段 `+1`（最多 4），能力与动量小升；
- `40—79` 有限试院：仅低阶段可 `+1`，留下保守妥协；
- `0—39` 部门俘获：不升级，军事自主或精英固化上升；
- `<0` 仪式化失败：改革动量下降，危机压力上升。

工业阶段是知识生产组织方式的阶段标签，不是科技树；一次事件链最多提升一级。

## 5. 本体边界、AI与验收

- 容器使用统一 `story_cycle`；推进使用延迟事件、战争结束回调或抽象验证；
- 人物只取现有翰林、六部、军职和朝臣；匠役用匿名群体文本；
- 复用本体行政、学识、战争背景及书卷/工具/军械图标；
- 禁止新创新、文化传统、兵种、建筑、商品、地图实体和逐件发明模拟；
- AI 只读全国值和当前路线，不计算人物全表；无主持者时退化为匿名试院；
- 必测四路线、真实战争/抽象校验、四结局、继承续链及阶段上限。
