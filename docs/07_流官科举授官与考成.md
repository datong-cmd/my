# 07 《变身大明》流官、科举、授官与考成

> 参考机制解析。新项目不复制官阶、科举或补官系统，只把现有身份作为改革和事件链人物来源。

## 1. 已有数据模型

| 事实 | 载体 |
|---|---|
| 文武宦官路线 | 流官相关 trait/trigger |
| 官品 | 分级 trait group |
| 实际职务 | court/council position |
| 科举、监生、恩荫等出身 | trait、relation、memory 与事件状态 |
| 履历、丁忧、致仕 | memory、flag、modifier、event |

官品与实际职位分离，不能只用一项代表官员能力或权力。

## 2. 已有流程

- 科举由决议/事件分阶段组织，包含乡试、会试、殿试和文武分流；
- 授官通过人工/自动路径选取候选，职位和品级共同参与；
- 文官、武官、宦官有不同考课与升迁入口；
- 京察、大计和军政考选形成周期性淘汰与晋升；
- 朋党、门生、皇权和中旨任命会改变政治可行性。

具体事件 ID 与 effect 调用见 `02/02A`。

## 3. 新项目允许读取

- 当前内阁、六部、监察、厂卫与五军职位持有者；
- 已验证的品级、流官路线、出身和党争信号；
- 人物是否在世、在廷、入狱、受雇于明制；
- 前置考成结果留下的稳定 trait/modifier。

## 4. 禁止事项

- 不创建第二套十八级官品、科举、补缺或退休系统；
- 不批量生成进士或候选人；
- 不写前置的官品、履历和职位变量；
- 不为模拟社会流动建立人口或地区数据；
- 不直接触发需要内部 saved scope 的前置事件链。

实施时最多从现有人物中抓取 12 名候选，展示前四名；无合适者则使用匿名机构或延后，而不是造人填坑。
