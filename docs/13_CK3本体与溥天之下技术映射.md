# CK3 本体与“溥天之下”技术映射

> 文档性质：新 Mod 技术底座、DLC 依赖和脚本接口选型  
> 基准环境：CK3 `1.19.0.6 (Scribe)`；参考模组声明支持 `1.19.0.6`  
> DLC：`All Under Heaven`，本地目录 `game/dlc/dlc022_ep4`，脚本功能标记 `all_under_heaven`  
> 证据标记：本体或 DLC 脚本已确认用 **S1**；新项目选型用 **D**；版本、兼容或实现风险用 **R**。  
> 同路径覆盖、资产和借鉴边界另见 [13A](13A_兼容覆盖与资产复用清单.md)。

## 1. 结论先行

“溥天之下”已经把 CK3 从主要依赖封建头衔的框架，扩展到可表现东亚官僚帝国的状态。天朝政府原生拥有功绩、影响力、国库、俸禄、行政省、官职任命、强大家族、家产、政治运动、朝贡契约、科举、巡察、王朝周期和大型工程。参考模组早年用自定义变量、人物交互和覆盖 GUI 补出的许多能力，在 1.19.0.6 已有更稳定的原生接口。

**D：新 Mod 不复制参考模组的整套国家模拟层，而采用“三层架构”。**

```text
CK3 / All Under Heaven 原生层
  政体、头衔、国库、功绩、影响力、任命、家产、朝贡、局势、活动、工程、军队
                    ↓
明代制度适配层
  六部/都察院/内阁语义、考成、财政预算、卫所、边疆契约、重大事件接口
                    ↓
历史唯物主义博弈层
  阶级力量、生产关系、组织度、意识形态、改革联盟、危机、阶段转型
```

这意味着：人物和政治关系仍由 CK3 擅长的角色系统表现；宏观社会只保存少量聚合变量；政治集团通过原生政治运动和强大家族获得人物载体；改革不是按钮加永久增益，而是改变收入、任命、组织和动员规则，并制造受益者、受损者和反制联盟。

## 2. 版本与依赖基线

### 2.1 已确认版本

| 项目 | 已确认值 | 证据位置 | 影响 |
|---|---|---|---|
| 游戏版本 | `1.19.0.6 (Scribe)` | `launcher/launcher-settings.json` | 所有接口结论只对该快照负责 |
| 参考模组 | `1.0`，支持 `1.19.0.6` | 参考模组 `descriptor.mod` | 可在同一版本比较，但不证明与 DLC 机制无冲突 |
| DLC | `All Under Heaven` | `game/dlc/dlc022_ep4/dlc022.dlc` | 本项目的正式依赖 |
| DLC 功能标记 | `all_under_heaven` | `has_tgp_dlc_trigger` 最终检查该标记 | DLC 专用入口统一走门槛触发器 |

**D：首个正式版本直接要求“溥天之下”，不维护无 DLC 降级分支。** 无 DLC 兼容会迫使项目重做政体、家产、功绩、国库、任命、局势和工程，既增加代码量，也制造两套难以同时平衡的游戏。启动或首次载入时应检查 `has_tgp_dlc_trigger = yes`；若缺失，仅给出明确说明，不继续初始化国家变量。

**R：版本支持必须使用精确小版本。** `supported_version` 可声明 `1.19.*` 方便启动器识别，但发布说明与测试矩阵必须保留实际验证号。更新游戏后先做解析和烟雾测试，再修改兼容声明。

## 3. 核心机制选型矩阵

| 设计需求 | 首选 CK3 载体 | 采用方式 | 不采用的旧式方案 |
|---|---|---|---|
| 天朝国家结构 | `celestial_government` | 直接复用并以法律、契约、修正和事件扩展 | 新造一套平行政体和自定义国库 |
| 朝鲜、越南等官僚邻国 | `meritocratic_government` | 作为非天朝功绩官僚政体 | 一律改成天朝封臣 |
| 官员资历 | 原生 `merit` | 表示制度认可的任职资格和履历 | 每个官员自建“官阶变量”全套替代 |
| 运作政治资本 | 原生 `influence` | 任命、结盟、请托和政治交易货币 | 威望充当万能政治货币 |
| 国家财政 | 原生 `treasury` + 少量预算变量 | 国库保存总量，预算变量保存承诺和结构 | 人物黄金与国家资产混用 |
| 流官与部长任命 | `succession_appointment` | 扩展候选分，保留原生候选 UI 和 AI | 定期全世界扫描后瞬移头衔 |
| 官僚家族基础 | `east_asian_estate`、强大家族、家族志向 | 家产表现私人制度资本和关系网 | 给每个官员创建虚构地产头衔 |
| 科举 | `activity_imperial_examination` | 扩展考试结果与阶级流动反馈 | 单一决议批量刷角色 |
| 巡按、考成、勘灾 | `activity_survey` | 新增或复用巡察焦点和结算 | 每月遍历所有省份随机触发 |
| 朝贡秩序 | `tributary_celestial` 等 subject contract | 用契约档位表达义务、特权和自主 | 二元变量 + 强制接受全部命令 |
| 国家阶段与政治阵营 | 新建独立 `situation`，借鉴王朝周期 | 聚合阶段、集团、领袖、力量和催化剂 | 覆盖原生 `dynastic_cycle` 或每月事件链 |
| 大型建设与再工业化项目 | `great_projects` | 工期、出资者、分项和完工回调 | 瞬时支付后加永久修正 |
| 军工产能 | 火药库上限 + 自定义聚合军工能力 | 建筑提供能力，军队消耗能力 | 只用科技解锁无限新军 |
| 改革路线 | 法律/政策 + 局势催化剂 + 事件链 | 改规则、改联盟和改力量分配 | “改革成功=全属性永久加成” |
| 民众与阶级 | 国家/区域聚合变量 | 年度抽样更新，重大事件即时修正 | 为每个平民生成角色 |

## 4. 政体、国库、功绩与影响力

### 4.1 天朝政府是正式底座

**S1：** `common/governments/00_government_types.txt` 中的 `celestial_government` 是行政制、可无地游玩、使用合法性和强大家族的政府。它设置：

- `treasury = yes`、`merit = yes`、`replace_gold_cost_by_treasury = yes`；
- `domicile_type = east_asian_estate`；
- 行政省与任命、头衔兵士、家族志向、强大家族和县级贵族家族；
- `celestial_vassal` 契约组；
- 政府标记 `government_is_celestial`、`government_has_merit`、`government_has_influence`、`government_has_treasury`、`government_has_estate` 等；
- 功绩等级上限 9，适合比一般功绩官僚政体更长的职业梯度。

`meritocratic_government` 同样使用功绩、影响力、国库、家产和行政任命，但功绩上限较低，且直接受 `has_tgp_dlc_trigger` 控制。

**D：** 大明国家层直接使用 `celestial_government`。内阁、六部、都察院、厂卫、卫所等不是新政府，而是该政府中的职位、契约、法律、机构修正和政治集团。只有当国家发生制度断裂，例如军阀化、军事共同体掌权或革命政权彻底重构官僚制时，才考虑政府切换。

### 4.2 四类资源不可互相替代

| 资源 | 语义 | 主要收入 | 主要支出 | 设计限制 |
|---|---|---|---|---|
| 人物黄金 | 私人可支配财产 | 领地、家产、赏赐、商业关系 | 家产、私人网络、贿赂、自费行动 | 不得直接代表国家财政 |
| 国库 | 国家公共财政能力 | 省级上缴、朝贡、专卖、借款 | 俸禄、军费、赈济、工程、行政 | 必须受预算承诺和信用约束 |
| 功绩 | 制度承认的履历与任职资格 | 科举、考成、军功、重大任务 | 通常不直接花费，决定候选资格和排序 | 不能等同实际政治权力 |
| 影响力 | 可动用的政治关系资本 | 职位、家族、运动、结盟、恩庇 | 任命竞争、请托、结党、政策推动 | 不等同阶级力量或意识形态 |

**D：** 阶级力量是另一层聚合状态，不新增一种可随意购买一切的“阶级点数”。官僚可能功绩高而阶级组织弱，也可能功绩低但家族、资本和军队关系强；这正是长期政治博弈的来源。

### 4.3 国库和预算的分工

**S1：** 天朝封臣契约原生包含国库上缴、部院预算、俸禄和俸禄品级。本体已经把私人收入与公共财政区分开。

**D：** 新 Mod 只额外保存少量国家预算承诺，例如 `ming_budget_military_share`、`ming_budget_relief_share`、`ming_budget_projects_share` 和 `ming_debt_pressure`。季度或年度结算把原生国库净流量映射到这些结构变量。不要为盐、粮、布、铁、木、铜逐笔建立全球库存账；仅在关键资源成为危机瓶颈时使用聚合储备或项目条件。

## 5. 任命、科举与官僚再生产

### 5.1 原生任命评分已经足够丰富

**S1：** `common/succession_appointment/` 中有 `celestial_minister.txt`、`celestial_governor.txt`、`meritocratic_governor.txt` 等。候选分可读取功绩、影响力、年龄、能力、职业履历、家产建筑、家族志向、关系、密谋、宫廷和议会职位、家族与宗族、政治运动以及王朝周期阶段。文官与武官还能使用不同的功绩履历变量。

**D：** 不新建“候选池数据库”。沿用原生候选 UI，只在评分中加入明代制度项：

```text
最终候选分 = 原生任命分
           + 科举/军功/考成履历
           + 对任命者与主政联盟的政治可靠性
           + 区域经验与回避规则
           + 改革路线匹配
           - 贪腐案、失职、党争污点和利益冲突
```

这些项目必须出现在可解释的候选分明细中。玩家能看见“为什么此人有资格”和“为什么此人政治上可行”，而不是面对隐藏随机数。

**R：** 修改本体已有任命类型通常需要覆盖同一数据库键，易受补丁影响。MVP 优先用候选角色修正、履历变量、法律和 scripted value 被原生评分读取；只有无法挂接时才覆盖对应任命文件，并把覆盖集中到兼容模块。

### 5.2 科举与巡察采用活动系统

**S1：** `activity_imperial_examination` 直接要求 `has_tgp_dlc_trigger`，并有完整参加者、阶段、事件和 `on_complete`。`activity_survey` 有边疆、官僚和直属领地等焦点，官僚焦点进一步包含税收、发展与控制；活动结束有统一清理和结算。

**D：**

- 科举负责制造和筛选个人官僚角色，并改变社会流动、士绅再生产和政治运动成员；
- 巡察负责把宏观省情转成有限的人物事件和考成证据；
- 考成采用“年度轻量评分 + 必要时巡察复核”，不是让每位官员每月跑一条事件链；
- 改革派可扩大科举来源或专业科目，但会损害旧式经学网络和既有家族的候选优势；
- 军功、技术官僚、胥吏升迁应成为并行但受争议的入仕路径。

## 6. 家产、强大家族和私人制度资本

**S1：** `east_asian_estate` 是东亚行政角色的原生 domicile。`00_chinese_estate_buildings.txt` 提供书库、税务、部院办公、档案、舆图、征募、吏员厅、稽核、雕版、造纸、考试、客舍、工坊、常平仓、磨坊、度量衡、灌溉等大量建筑及图标。

**D：** 家产不是国家工业区，而是官僚家族的私人制度基础。它可以决定：

- 候选分、功绩积累和信息能力；
- 养士、门生、书院与技术人员网络；
- 私人借贷、行商、地产和工坊利益；
- 在改革或危机中能动员的关系、现金和追随者；
- 家族从士绅向官僚资本、商人资本或军事承包者转化的路径。

大型工厂、全国军器生产和交通网不能直接用单个家产建筑代表，应通过省级建筑、国家项目和聚合产能共同表示。家产只提供投资主体、技术节点或政治受益者。

## 7. 政治运动、局势与历史阶段

### 7.1 王朝周期可借鉴，不宜直接改写

**S1：** `common/situation/situations/tgp_dynastic_cycle.txt` 定义了：

- 核心子区域和参与者组；
- 亲君、进取、扩张、保守和未定政治运动；
- 运动成员、领袖、人物力量和集团总力量；
- 稳定、扩张稳定、进取稳定、不稳定、征服型不稳定和混乱等阶段；
- 灾害、战争、合法性、国库债务、贪腐、科举间隔、工程、军队落后、任命失当等大量催化剂；
- 阶段进入/退出效果、集团参数、年度漂移和 UI 状态。

这证明局势系统可以承担长期、多方、可逆的国家状态机，并把人物政治与宏观阶段连接起来。

**D：** 新建独立局势，例如 `ming_social_transformation`，不要覆盖 `dynastic_cycle`。原生王朝周期负责天命和帝国稳定；新局势负责生产方式、阶级力量、改革联盟和社会转型。两者只通过少量桥接催化剂相互影响：财政崩溃、内战和灾荒可同时加剧两套危机；有效赈济、制度重组和生产恢复可缓和王朝危机，但未必缓和阶级冲突。

### 7.2 新局势的职责边界

```text
原生 dynastic_cycle
  谁拥有天命？帝国是稳定、不稳定还是混乱？

新建 ming_social_transformation
  哪种生产关系占主导？哪些阶级有组织力？改革联盟是否能执行？
  社会处在商业化、财政军事化、再工业化、资本扩张、革命危机或复辟整合的哪一阶段？
```

**D：** 新局势参与集团不是简单复制“保守/进取”。至少应让官僚士绅、宗室宫廷、军功军事集团、城市商人/工场主、乡村地主、编户小农/佃户和城市劳动者的力量先在聚合层计算，再选择少量有角色代表的政治联盟进入 UI。具体模型在 `14` 和 `15` 文档定义。

**R：** 同一角色进入多个 situation participant group 是引擎支持的常见用法，但高频重算会有成本。成员归属按季度或重大事件重算；力量值按年度基线加即时增量，不要每天遍历全帝国角色。

## 8. 朝贡、藩属和跨政权秩序

**S1：** `subject_contract_groups.txt` 原生定义 `celestial_vassal`、`tributary_celestial` 和 `tributary_hegemonic`。天朝朝贡契约可以控制黄金、威望和贸易、婚姻、政治册封特权；天朝内臣契约则包含省份、国库、俸禄、部院预算与品级。

**D：** `12` 中提出的承认、依赖、负担、保护信用与自主性，不应全部另造变量。首选映射为：

| 设计状态 | 原生载体 | 额外缓存 |
|---|---|---|
| 正式关系与继承 | subject relation / contract | 仅保存稳定关系 ID 或主从引用 |
| 贡赋负担 | 黄金/威望契约档位 | 灾害豁免、拖欠和特别贡物 |
| 册封特权 | trade/marriage/politics 档位 | 临时承诺与事件期限 |
| 保护义务 | 战争和互动规则 | `protection_credit` 聚合信用 |
| 自主与离心 | 契约、意见、实力和政治事件 | 必要时保存一项压力值 |

原生关系是事实来源，缓存变量只补充它无法表达的长期信用，不能形成两套可能不同步的宗主关系。

## 9. 大型工程、基础设施与再工业化

### 9.1 大型工程接口能力

**S1：** `common/great_projects/types/_great_project_types.info` 和现有长城、运河、强化首都、军营工程证明大型工程类型支持：

- 展示条件、规划条件、地点过滤和持续有效性；
- 项目所有者、发起人、所在地和图像；
- 规划费用、脚本化工期、出资截止点和出资冷却；
- 封臣、朝贡国、领主、盟友、项目所有者等出资者过滤；
- 强制与可选出资分项；
- 规划、开工、完工、取消、失效回调；
- 对出资者和项目结果分别施加效果。

**D：** 这是运河整治、海塘、边墙、漕运、矿冶、国家军器局、标准化工场和后期铁路/蒸汽试验工程的首选载体。工程必须包含：

1. 谁提出并控制项目；
2. 谁出钱、出粮、出劳役或出技术；
3. 强制分项与政治交换分项；
4. 工期内的腐败、事故、抵制和超支；
5. 完工后收益归谁、谁被征用、谁获得组织经验；
6. 国家崩溃、领土易手或财政中断时的暂停/失败规则。

单个项目的完成可以改变建筑和聚合产能，但不能直接把全国推进一个历史阶段。阶段转型还需要技术扩散、财政承受、劳动组织、市场范围和政治联盟条件。

### 9.2 工程与阶级博弈的连接

| 出资/承担者 | 短期收益 | 长期政治后果 |
|---|---|---|
| 国库 | 统一调度、快速开工 | 债务、增税和预算挤压 |
| 地方官府 | 地方控制和政绩 | 截留、摊派与地方自主增强 |
| 官僚家族 | 候选分、合同和声望 | 私人制度资本扩张 |
| 商人资本 | 合同、市场和特许 | 要求法权、垄断或政治代表 |
| 小农/佃户劳役 | 可能获得水利和市场收益 | 征发、迁徙、失地和反抗风险 |
| 城市劳动者 | 工资、技能和组织经验 | 形成议价与集体行动能力 |
| 军事集团 | 军工和交通优势 | 控制战略设施并提高政变能力 |

## 10. 火器、军工产能与兵种

### 10.1 本体已经有“建筑支持军团”的产能上限

**S1：** 晚期中世纪火药创新设置 `unlock_late_medieval_gunpowder_units` 和 `unlock_powder_magazines`。`powder_magazine_01` 至 `08` 可在合适地产建造，并强化驻扎火药兵与攻城武器。`00_men_at_arms_values.txt` 通过遍历直属省份的火药库等级，计算可支持的火药部队数；火枪手和火枪骑兵招募条件要求现有火药军团数低于建筑支持上限。脚本注释明确表达“一座火药库支持若干火药军团”的容量思想。

**S1：** 本体和 DLC 可用兵种包括 `handgunners`、`fire_lancers`、`rocket_carts`、`cannon` 等；东亚火炮还检查中国、朝鲜或日本文化传承。火箭车和火炮已有图标、成本与战斗定位。

**D：** 军事现代化沿用“技术解锁 + 生产建筑容量 + 预算维持 + 政治控制”四条件：

```text
可部署现代军团数
  = min(技术与训练上限, 军工作坊/火药库容量, 可持续军费, 政治可靠部队槽位)
```

军工能力不足时不能只靠黄金瞬间扩军；财政不足时可以短期透支但积累欠饷、倒卖军器和兵变风险；军事集团控制军工节点时，皇帝也未必能无条件调动。

**R：** 原生 `gunpowder_raise_limit_existing_gunpowder_units` 只计个人直属的特定兵种和直属省份，未必自动覆盖新兵种、行政头衔兵士或全国军工。新 Mod 应新增带前缀的聚合 scripted value，把原生火药库和国家军工项目映射成统一能力；不要直接改写通用本体值，避免与其他模组冲突。

### 10.2 近代兵器的抽象边界

“机枪”“铁路炮兵”等超出 CK3 美术和战斗模型的内容不应要求逐件装备或新 3D 模型。后期可用一个标准化新军兵种、一个火力支援兵种和若干军队修正抽象工业火力；决定胜负的不只是伤害值，还包括补给、军费、军官体系、铁路/水运、兵员动员和政治忠诚。

## 11. 重大事件与危机的原生接口

| 危机 | 原生检测/载体 | 新设计作用 |
|---|---|---|
| 财政债务 | 国库、预算、王朝周期催化剂 | 欠饷、停工、信用危机、利益集团争夺 |
| 灾荒疫病 | natural disaster、epidemic、巡察 | 赈济选择、人口迁徙、粮价与地方组织 |
| 边患和战争 | war、MAA、朝贡契约 | 动员分配、军功集团增长、军费与合法性 |
| 官僚腐败 | 秘密、任命、巡察、考成 | 不是固定道德值，而是监督、俸禄和网络的结果 |
| 党争 | 政治运动、家族、关系、影响力 | 联盟重组、政策阻挠、任命交易 |
| 民变与革命 | faction、war、situation | 由生计、组织、军队倒戈和替代制度共同决定 |
| 继承与摄政 | succession、diarchy、legitimacy | 政策承诺重议、联盟断裂和重大机会窗口 |

**D：** 重大事件只改变已有结构或迫使参与者选择，不凭空制造结果。每条事件链至少读取财政、阶级力量、组织度、政治联盟和地方状况中的两项；结算至少改变其中两项，并留下可追踪的赢家、输家和后续义务。

## 12. 脚本分层与建议目录

```text
common/
  scripted_triggers/     # 只判断，不改状态
  scripted_values/       # 阶级力量、预算、产能、候选分的纯计算
  scripted_effects/      # 唯一写入路径、结算与迁移
  on_action/             # 轻量入口，调用调度器
  situation/             # 社会转型局势、集团、阶段和催化剂
  laws/                  # 制度规则与改革落地
  modifiers/             # 可见状态，不承担唯一事实源
  great_projects/        # 国家工程与工业化项目
  buildings/             # 省级生产和军工节点
  men_at_arms_types/     # 少量新增兵种
  character_interactions/# 谈判、任命、结盟和政治行动
events/
  ming_core/             # 初始化、迁移、调度和错误恢复
  ming_politics/         # 权力运行和联盟事件
  ming_economy/          # 财政、生产和劳动事件
  ming_crises/           # 重大危机事件链
  ming_reform/           # 改革与革命路线
localization/simp_chinese/
gui/                     # MVP 尽量为空；优先 scripted GUI 或原生局势 UI
```

**D：事实源规则。**

- 政府、头衔、关系、契约和人物资源以原生对象为事实源；
- 国家/区域宏观状态只由带统一前缀的变量保存；
- scripted modifier 用于展示和派生效果，不作为唯一事实源；
- 所有写入通过少数 scripted effects，事件选项不得散落复制计算；
- 缓存值必须标注重算频率、失效条件和恢复方法。

## 13. MVP 技术选择

首个可玩版本只做以下本体映射：

1. 大明使用天朝政府、国库、功绩、影响力、家产、原生任命和天朝契约；
2. 新建一个社会转型 situation，先放四个可见政治联盟，底层计算七类阶级力量；
3. 科举和巡察各接入一个明代结算扩展；
4. 国库外只保存四项预算结构、一项债务压力和三类生产能力；
5. 采用原生火药库容量，增加国家军工能力桥接值；
6. 制作两个大型工程样板：漕运整顿和国家军器局；
7. 制作一条财政—欠饷—兵变危机链和一条土地兼并—灾荒—民变链；
8. 不做自定义全屏 GUI，不覆盖原生政府、战争、议会或角色窗口。

只有这套垂直切片证明“结构变量—人物联盟—政策选择—危机结果—阶段变化”能完整闭环后，才增加再工业化、资本兴起和革命路线。
