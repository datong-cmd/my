# 26B 革命、反革命与实际战争桥接方案

## 1. 革命不是随机起义按钮

革命危机仅在历史阶段 3 且满足组合门槛时出现：民众动员高、精英协调低、危机压力高，并且改革势头或军事自主中至少一项形成替代权力。革命方与王朝方都可能胜利、妥协或分裂。

```text
革命力量 = popular_mobilization + reform_momentum/2 + crisis_pressure/2 + 组织选择
王朝力量 = state_capacity + elite_cohesion + 原模组皇权折算 + 军队服从修正
```

前三阶段仍是事件化政治博弈；只有累计差值达到战争门槛，才桥接 CK3 实际战争。

## 2. 领袖选择

按 `25A` 优先级：先从现有明制封臣/军职/党魁/改革专员中选择有动机者。只有没有任何合格者且革命已到战争门槛时，才创建一名 `minghm_generated_major_actor`，计入全国存活上限 2。

匿名农民、工匠、商人不创建人物。革命领袖一旦生成就是可能长期存活的真实政治人物，不承诺事件后释放。

## 3. 两级实现

### MVP：事件化政变/妥协

不定义战争。根据双方差值产生：王朝镇压、强制改革、权力分享、革命接管失败、长期割裂五类结果。它适合先验证数值闭环、AI 与清理。

### 第二阶段：唯一自定义革命战争

新增唯一键 `minghm_revolution_war`，文件 `common/casus_belli_types/minghm_revolution_cb.txt`；使用本体 `start_war`，语法已在本体事件和《变身大明》互动中证实。禁止覆盖参考模组 `GM_casus_belli_types.txt`。

调用结构：

```text
scope:revolution_leader = {
  start_war = {
    casus_belli = minghm_revolution_war
    target = title:h_greatming.holder
    target_title = title:h_greatming
  }
}
```

实际编码前必须做 T0 探针确认 1.19.0.6 对 `casus_belli`/`cb` 参数名、无地攻击者合法性和目标 title 作用域；若无地攻击者不合法，则只允许现有有地角色开战，不运行时临时授地。

## 4. 战争结果只写政治状态

| 结果 | 本体处理 | 本模组处理 |
|---|---|---|
| 革命胜利 | 经验证的 title/vassal change 或事件接管 | 历史阶段转新秩序整合，重置精英协调并写入路线承诺 |
| 王朝胜利 | 战争失败/领袖被俘 | 民众动员下降、守成/军事自主按镇压方式上升 |
| 白和平 | 不改国号 | 进入双重权力余波，危机压力保持高位 |
| 失效 | 继承/目标消失 | 清战争桥接 flag，重新判定而不自动补战 |

革命胜利不能直接创建新国家，因为本项目只维护明制兼容版；它表示大明内部权力结构被改造。若未来要改国号/政体，应另立独立扩展，不塞进本兼容模组。

## 5. 反革命机制

反革命不是专属阵营对象，而是三种可选动员：皇权—内廷、官僚—士绅、军事镇压。每种都提高当期王朝力量，同时分别付出精英分裂、守成力量、军事自主的长期代价。玩家不能无成本把所有力量叠加。

革命方若取胜，也必须处理军队、官僚和民众组织之间的矛盾；新秩序整合阶段连续两年国家能力过低会退回双重权力。

## 6. 性能与兼容红线

- 全国最多一场本模组革命战争；战争期间暂停本模组普通政策季度结算；
- 不新增 faction 类型，不覆盖参考模组已整体替换的 `common/factions/00_factions.txt`；
- 不扫描所有农民派系或地图省份；只检查明帝相关当前战争和保存的领袖；
- 不复制军队 3D；军队与兵种全部使用本体/参考模组已有内容；
- CB 未通过实机 T0/T1 探针前，默认关闭实际战争桥接，MVP 事件闭环仍可运行。

## 7. 验收

- 低民众动员或高精英协调时不会凭单一随机数爆发革命；
- 革命和王朝双方都有至少三种策略、妥协路线和长期代价；
- 实际战争只有在探针通过且有合法领袖时启动；
- 四类战争结束均清理 crisis/war saved scopes；
- 无新增 faction 覆盖，无非明革命事件。
