# 大明专用王朝局势：实现说明与运行边界

## 已实现的结构

本模组新增唯一的 `minghm_dynastic_cycle`，但绝不改写 DLC 原有的 `dynastic_cycle`。它的唯一服务对象是《变身大明》的 `title:h_greatming`。

| 项目 | 实现 |
|---|---|
| 局势对象 | `common/situation/situations/minghm_dynastic_cycle.txt` |
| 参与者 | 大明皇帝 1 名；与该皇帝保持 `diange_daxueshi` 关系的现任大学士，通常至多 6 名 |
| 明确排除 | `h_china`、天命、朝贡、丝路、外国统治者、臣属自动加入、地图行动 |
| 阶段来源 | 已有 `minghm_historical_stage = 0..4`；年度八项力量公式与两年确认保持不变 |
| 同步频率 | 每年一次；只有阶段实际变化才调用一次 `change_phase` |
| 局势自身更新 | 无 `on_monthly`、无 `on_yearly`、无 catalyst、无 duration、无 movement power |
| 长期数据 | 仍仅写在 `h_greatming` 的 `minghm_*` 变量；局势不另存一套数值 |

## 五阶段映射

| 原有数值阶段 | 大明局势 phase | DLC 复用 phase 图标 | 含义 |
|---:|---|---|---|
| 0 | 秩序整合 | advancement | 制度仍可协调运行 |
| 1 | 制度失衡 | instability | 危机与守成阻力累积 |
| 2 | 改革竞争 | expansion | 多种改革组织争夺制度方向 |
| 3 | 双重权力 | chaos | 国家整合下降、不同正当性竞争 |
| 4 | 新秩序整合 | conquest | 新组织承诺进入整合考验 |

图标只借其视觉语义；本模组不继承原 DLC 对征服、扩张、天命的游戏效果。

## UI 和美术

“查看大明局势”决议现在挂载一个只读 event widget：

- 静态轮盘、边框和五枚 phase 图标直接引用 TGP 已加载的 DDS；不复制资源。
- 中央显示当前大明局势 phase 图标；不调用 `SituationDynasticCycleWindow` 的 C++ 旋转、地图模式或天朝参与者代码。
- 现有六枚自制 64×64 图标继续只用于阶段通知、文字图例、技术组织与新秩序提示，不替代 DLC 轮盘。
- 没有新 3D、肖像、建筑、单位、美术定义或全屏窗口。

## 运行流程

```text
明帝取得/持有 h_greatming
  -> 初始化 h_greatming 的八项长期数值
  -> 若不存在，建立 minghm_dynastic_cycle
  -> 手动登记皇帝 + 当前大学士
  -> 年度八项数值判定并两年确认历史阶段
  -> 同步对应 situation phase
  -> 玩家打开“查看大明局势”时读取该 phase 与轮盘纹理
```

## 需要实机确认的项目

静态验证已确认 situation 定义、关系接口、DLC 纹理路径和 GUI 部件注册方式。晚间实机应确认：新局/旧档是否创建局势、大学士是否仅以关系加入、打开总览时 widget 是否显示，以及 `error.log` 是否无 situation/GUI scope 错误。若 widget 显示异常，可暂时移除 event 的 `widgets` 块；数值阶段与局势同步仍可独立运行。
