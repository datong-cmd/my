# 性能、缓存、存档与测试预算

> 文档性质：CK3 1.19.0.6 实现硬约束、性能验收和版本迁移规范  
> 原则：先限制对象和频率，再优化公式；使用相对本体基线，不承诺脱离机器和模组组合的绝对毫秒。

## 1. 硬规模上限

| 对象 | 标准上限 | 超出处理 |
|---|---:|---|
| 功能区域 | 8，扩展最多 12 | 合并相近区域，不增加每县矩阵 |
| 基础社会集团 | 8 | 新身份映射到既有集团或作为临时分支 |
| 可见政治联盟 | 6，危机临时联盟最多 +2 | 合并低力量联盟为“未定/地方” |
| 全国主改革 | 2 | 新项目进入候选队列 |
| 同时大型工程 | 3，全国级最多 1 | AI/玩家需暂停或完成旧项目 |
| 全国重大危机 | 2 | 其余压力缓存或合并 |
| 区域重大危机 | 2 | 低优先级进入年度摘要 |
| 单危机重要人物 | 20，硬上限 40 | 只保留领袖、执行者、否决者和代表 |
| 国家阶级代表角色 | 8—16 | 死亡后按需补选 |
| 阶段历史记录 | 最近 10 次 | 更早压缩为摘要计数 |
| 承诺记录 | 每联盟/国家 12 条活动承诺 | 结算后压缩成信誉记忆 |
| 年度主要变化原因 | 每指标 3 条 | 低权重合并为“其他” |

上限属于设计规则，不允许 AI 生成代码时擅自扩大。

## 2. 更新频率表

| 频率 | 允许内容 | 禁止内容 |
|---|---|---|
| 即时事件 | 任免、战争、法令、建筑、工程、死亡、重大选择的增量 | 全图重算 |
| 月度 | 只有活动中的局部倒计时、围城/饥饿等极少状态 | 全国阶级、区域经济、联盟全员扫描 |
| 季度 | 活动危机、改革承诺、预算偏差、联盟成员和阶段出口 | 世界人物扫描、全地图地产扫描 |
| 年度 | 8 区结构、8 类阶级、七驱动值、AI 主议程 | 每人物复杂事件链 |
| 五年 | 地图抽样校准、长期阶段和产业网络检查 | 全量 barony 逐项深算 |
| 生日/死亡 | 单人物考成、利益画像失效、代表补选 | 从该入口遍历所有官员 |
| GUI 打开 | 读取缓存、合法性便宜判断 | 改状态、复杂遍历和随机事件 |

## 3. 调度器

### 3.1 单一国家调度入口

每个合格天朝政权只有一个年度和一个季度主调度器。on_action 只做廉价过滤：

```text
是否启用 Mod
→ 是否目标政体/主头衔
→ 是否最高相关统治者
→ 是否本周期尚未执行
→ 调用对应调度 effect
```

子模块不得各自在同一 on_action 上重复扫描国家。

### 3.2 分散执行

多个 AI 天朝政权按主头衔 ID/月份散列到不同季度或月份，避免同一天集中年度结算。人物考成优先使用生日分散入口，首层 trigger 排除绝大多数非官员。

### 3.3 幂等锁

年度、季度和迁移都有周期标记；重复加载或多个 on_action 命中时不重复扣款、加值或触发事件。

## 4. 遍历成本等级

| 等级 | 范围 | 政策 |
|---|---|---|
| O(1) | 当前人物、头衔、区域锚点、缓存 | 常规允许 |
| O(k) | 固定 8 区、8 类、6 联盟、有限候选池 | 允许，按预算运行 |
| O(realm) | 本国重要人物、直属省、战争参与者 | 仅季度/年度或事件驱动，严格过滤 |
| O(map) | 全国所有县/地产 | 初始化或五年抽样替代；常规禁用 |
| O(world) | 所有在世人物、所有统治者、全世界头衔 | 核心 Mod 禁用 |

### 4.1 明确禁止

- `every_living_character`、`any_living_character` 作为常规脉冲入口；
- 先 `any_*` 找到目标，再 `every_*` 重扫同一全集；
- GUI 实时运行多层 `every_*`；
- 为一次候选选择对全世界人物安排延迟事件；
- 每季度扫描所有 barony 建筑；
- 同一事件选项重复调用完整阶级/区域重算；
- 用随机杀人物模拟宏观人口损失。

## 5. 缓存目录

### 5.1 事实源与缓存

| 数据 | 事实源 | 缓存 | 刷新 |
|---|---|---|---|
| 政体、头衔、关系、契约 | 原生对象 | 是否合格天朝 | 政体/头衔/关系变化 |
| 国库、功绩、影响力 | 原生资源 | 年度财政摘要 | 预算事件/年度 |
| 区域结构轴 | 区域锚点变量 | 派生生计/流动 | 即时增量 + 年度 |
| 阶级七维 | 帝国主头衔变量 | power、压力和前三原因 | 年度/重大事件 |
| 政治联盟 | situation 成员和领袖 | 联盟总力量/立场 | 季度/成员变化 |
| 历史阶段 | situation phase | 七驱动值和接近阶段 | 年度/重大危机 |
| 工业能力 | 区域、建筑、项目 | 五类国家能力 | 项目完工/年度 |
| 危机 | situation/story state | 六仪表与 UI 摘要 | 季度/危机事件 |
| AI 信息 | 调查、职位和情报 | 估计区间 | 调查/季度 |

### 5.2 缓存失效事件

- 统治者死亡或主头衔改变；
- 政体、法律、契约和任命改变；
- 战争开始/结束、首都或关键区域易手；
- 大型工程、关键建筑和工业项目完成/失效；
- 灾害、革命、国家分裂和统一；
- 区域锚点或版本迁移；
- 调试命令显式重建。

失效只标记 dirty；除必须即时展示的值外，在下一合适脉冲批量重算。

## 6. 变量所有权

每个变量在 `18` 数据字典登记：

| 字段 | 要求 |
|---|---|
| 键名 | 冻结前缀，唯一 |
| scope | global、主头衔、区域头衔、人物、危机 |
| 事实/缓存 | 明确二选一 |
| 类型与范围 | 布尔、整数、定点；0—100 等边界 |
| 创建点 | 初始化、迁移或首次危机 |
| 唯一写入 effect | 禁止事件散写 |
| 读取者 | trigger、value、AI、GUI |
| 刷新周期 | 即时、季度、年度、五年 |
| dirty 条件 | 具体 on_action/事件 |
| 清理/继承 | 死亡、结束、头衔变化或永久保留 |
| 迁移默认 | 缺失时如何生成 |
| 调试断言 | 合法范围和依赖 |

## 7. 脚本原子化

### 7.1 trigger

只判断，不写变量、不触发事件、不产生随机。昂贵 trigger 前先放政体、头衔、标记等便宜条件。

### 7.2 scripted value

尽量纯计算。复杂值由年度/季度缓存，GUI 不重复计算。对空 scope 和缺失变量给安全默认。

### 7.3 effect

每类状态只有一个正式写入口，例如：

```text
minghm_change_class_dimension_effect
minghm_change_region_axis_effect
minghm_recalculate_national_class_power_effect
minghm_start_crisis_effect
minghm_advance_crisis_stage_effect
minghm_settle_crisis_effect
```

事件选项只调用这些 effect，不复制边界、原因日志和桥接规则。

## 8. 性能目标

绝对帧时受 CPU、存档、地图和其他 Mod 影响，因此使用同机同档 A/B 基线：

| 测试 | 目标 | 阻断线 |
|---|---:|---:|
| 100 年观察局平均推进时间 | 相对本体+DLC ≤ +5% | > +10% |
| 年度结算峰值 | 无可感知长停顿；相对季度均值 ≤ 2 倍 | 连续明显冻结 |
| 玩家国家打开主 UI | 缓存读取，主观即时 | 打开时出现全图扫描/卡顿 |
| 存档体积增量 | 100 年相对基线 ≤ +5% | > +10% 或持续超线性增长 |
| 活跃危机事件量 | 常态每年 4—10 个需玩家选择 | 常态 > 16 或单季度 > 4 |
| error.log | 新增解析/重复键/无效 scope = 0 | 任一新增错误 |

若基线本身波动大，至少跑三次取中位数。性能目标是设计门槛，不保证所有大型模组组合都达到。

## 9. 弹窗与认知预算

| 信息 | 呈现 |
|---|---|
| 年度阶级/区域变化 | 一份年度报告，前三原因 |
| 普通区域波动 | 日志/提示，不弹窗 |
| 危机季度变化 | 只有需要决策时弹窗 |
| AI 国家内部事件 | 聚合结算，除非直接影响玩家 |
| 改革进度 | 项目 UI 和里程碑，不逐县弹窗 |
| 承诺到期 | 一次集中提醒 |
| 错误/恢复 | 调试日志；正式版仅在无法恢复时提示 |

事件数量是玩法性能的一部分。即使 CPU 足够，过多点击也视为失败。

## 10. 存档架构版本

主头衔或全局保存：

```text
minghm_schema_version = <integer>
minghm_initialized = yes
minghm_last_annual_tick = <date/year marker>
minghm_last_quarter_tick = <marker>
```

### 10.1 迁移原则

1. 每次破坏性数据变化提高 schema 版本；
2. 迁移按版本逐级执行，不直接从任意旧版跳最终逻辑；
3. 新变量有基于旧数据的默认值；
4. 变量重命名先复制、验证，再清理旧键；
5. 删除机制先进入弃用期，旧存档仍能加载；
6. 迁移 effect 幂等，重复运行不重复加值；
7. 迁移完成写日志和新版本；
8. 无法安全迁移时停止相关模块，不损坏原生存档。

### 10.2 人物和头衔变化

- 国家长期变量依附稳定主头衔，不依附统治者；
- 区域变量依附固定锚点头衔；
- 人物利益、立场和记忆随人物死亡清理/继承指定部分；
- 国家分裂时按区域和制度复制必要摘要，不复制全部国级变量；
- 统一时按区域权重合并，不简单相加超过 100。

## 11. 不变量断言

年度调试检查：

- 所有 0—100 变量在范围内；
- 每个区域恰有一个锚点且不重复；
- 每个人物最多一个同类型联盟成员身份；
- 全国主改革 ≤ 2，大型工程 ≤ 3；
- 活跃全国危机 ≤ 2、区域危机 ≤ 2；
- 每个危机有合法 stage、scope 和至少一个决策主体；
- `power` 与七维来源一致或已标 dirty；
- situation phase 与阶段缓存一致；
- 缺 DLC 时不存在半初始化状态；
- schema 迁移没有旧新变量双写。

正式版不每年跑完整断言，只在 debug game rule 开启时运行。

## 12. 调试工具

提供仅调试可见的决议/交互：

- 初始化/重新初始化指定国家；
- 重算一国阶级、区域、工业和阶段缓存；
- 输出状态摘要到日志；
- 把某一结构轴设为测试值；
- 启动/推进/结算指定危机；
- 模拟统治者死亡、国家分裂、联盟领袖死亡；
- 执行 schema 迁移；
- 清理孤儿变量和参与者；
- 开关 AI 决策理由日志；
- 统计本周期事件、扫描和候选人数。

调试工具必须用 `debug_only = yes` 或等价门槛，不出现在正常游戏。

## 13. 测试层级

### 13.1 静态检查

- 括号、编码、重复键、缺本地化、无效路径；
- 自定义键前缀；
- 禁止同路径覆盖与 `replace_path`；
- 禁止世界级迭代器；
- 变量写入只来自登记 effect；
- `.info`、备份文件和本体资产不打包。

### 13.2 单模块情景

- 阶级年度重算；
- 区域轴增量和五年校准；
- 任命、继承和代表补选；
- 改革五种执行结果；
- 工业七环断裂；
- 危机 C0—C6；
- 革命与复辟；
- 朝贡和国家分裂。

### 13.3 组合情景

- 幼主 + 多线战争；
- 清丈 + 灾荒；
- 新军 + 欠饷；
- 海贸 + 信用危机；
- 工业化 + 劳动组织；
- 革命 + 外部挑战者；
- 复辟 + 已改变的土地和军队。

### 13.4 长跑

- 10 年快速烟雾；
- 50 年 AI 路线和事件量；
- 100 年性能、存档和阶段多样性；
- 200 年稳定性、变量泄漏和重复危机；
- 玩家/AI、强国/弱国、和平/战争、分裂/统一各至少一局。

| 维度 | 最小组合 |

## 15. 发布阻断条件

以下任一存在即不发布：

- 新增解析错误、重复数据库键、无效 scope 或缺失本地化；
- 核心脉冲出现世界人物/全地图扫描；
- 同路径 C4/C5 覆盖或未登记 C3 覆盖；
- 存档加载重复扣款、重复事件或变量越界；
- 继承/死亡后国家状态丢失；
- 国家分裂或统一造成总值爆炸；
- 100 年观察局超过性能阻断线；
- 常态事件量超过认知预算；
- AI 在数年内反复启动/取消同一改革；
- 缺 DLC 时出现半初始化；
- 发布包含本体、DLC 或参考模组复制资产。

## 16. 验收清单

- [ ] 所有核心对象有硬上限及超限处理；
- [ ] 即时、月度、季度、年度、五年、生日和 GUI 职责明确；
- [ ] 每个国家只有一个年度/季度主调度器；
- [ ] 多国家执行分散，不集中在同一天；
- [ ] 常规逻辑只用 O(1)、O(k) 和受限 O(realm)；
- [ ] 缓存有事实源、刷新和 dirty 条件；
- [ ] 每个变量有唯一写入口、范围、清理和迁移规则；
- [ ] 性能用同机同档 A/B 和三次中位数测量；
- [ ] 100 年推进、存档、弹窗和 error.log 有阻断线；
- [ ] schema 逐级幂等迁移，旧存档不会半初始化；
- [ ] debug 模式可重算、推进危机、输出理由和检查不变量；
- [ ] 静态、单模块、组合、长跑和矩阵测试齐全；
- [ ] 发布阻断条件自动化程度在 `18/19` 中落实。
