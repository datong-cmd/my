# 27 美术复用、兼容、性能与测试施工规范

## 1. 加载与依赖

固定组合：CK3 `1.19.0.6` + “溥天之下” + 《变身大明》（Workshop `2713348525`）+ 本兼容扩展最后加载。descriptor 声明依赖名 `变身大明`，不提供脱离前置的兼容分支。

本模组只能新增 `minghm_*` 键和新文件。不得覆盖：

- `h_greatming`、明政府、官职、品级、朋党、厂卫和军功定义；
- 参考模组 `00_factions.txt`、`GM_casus_belli_types.txt`、公共 GUI；
- 参考模组事件 namespace、flag、变量和 scripted effect 定义；
- 本体/DLC/参考模组的二进制美术文件。

## 3. 3D 红线

- 不新增或复制 `.mesh/.asset/.dds` 3D 资源；
- 不修改 portrait modifier、服装、头饰、武器、建筑或单位模型；
- 人物肖像自然继承《变身大明》和 TGP 的文化/官服规则；
- 军事事件展示现有角色和现有武器，不强制换装；
- 新机制不以 3D 资产是否出现作为脚本条件。

参考模组已有明官服、肖像环境、武器和事件场景，兼容扩展通过前置加载自然获得，无须打包。

## 4. 2D 预算

初版原计划新 2D 数量为 0。经玩家明确要求，现已启用固定的六枚小型文本图标，仍不制作自定义窗口、轮盘、动画或 3D。它们只出现在阶段变迁事件、局势总览图例与技术／新秩序入口，不参与任何计算：

- 4 个阶段徽记：稳固、失衡、改革竞争、双重权力；
- 1 个技术组织图标；
- 1 个新秩序整合图标（同时显示第五阶段）；
- 0 个新 trait/modifier 图标、0 张封面。

发布包只含 `gfx/interface/icons/minghm/*.dds` 的六张 64×64 纹理和 `gui/minghm_texticons.gui`。该 GUI 文件只注册 `@minghm_*!` 行内图标，不定义窗口或更新循环。清单见 [33](33_2D图标资产清单.md)。不得从上游复制后改名伪装成新资产。

王朝局势总览按 `29` 直接引用 TGP 的 `dynastic_cycle_wheel*.dds` 和五个 phase icons，不复制文件；原旋转效果依赖专用 C++ 窗口，MVP 改为固定轮盘与当前阶段高亮。

## 5. 兼容探针

| 编号 | 验证 | 失败处理 |
|---|---|---|
| C01 | `title:h_greatming` 与 holder 可访问 | 全系统休眠 |
| C02 | `is_GM_emperor` 可调用 | 使用本模组 title-holder 包装 trigger；仍失败则休眠 |
| C03 | `huang_quan_value`/`HQ_change` 签名与校验常数 | 禁用皇权消费选项，仅只读显示 |
| C04 | 官位/特质键存在 | 缺位按空缺处理，不生成替代身份 |
| C05 | `pengdang_yingxiang_score` 可读 | 党派修正归零，不改原党派 |
| C06 | theme/background 引用显示 | 回退本体主题 |
| C07 | `start_war` 与自定义 CB | 关闭实际革命战争，仅用事件 MVP |
| C08 | 旧存档初始化/复国恢复 | 清理并重建活动槽，不补算时间 |

探针失败只关闭相关桥接，不用猜测键、不覆写前置文件修复。

任何功能突破上限，先改成匿名叙事、缓存数字或减少参与者；不得以未来优化为理由合入。

## 7. 测试层级

1. T0 静态：重复键、括号、namespace、localization、路径、未知引用。
2. T1 启动：四件套按顺序加载，开始新局和旧档，无脚本错误刷屏。
3. T2 Ming gate：明帝可见，其他国家不可见；大明不存在时休眠。
4. T3 垂直切片：一项改革完成提案—廷议—执行—反馈—清理。
5. T4 生命周期：皇帝/主持人/目标死亡、离廷、入狱、继承、复国。
6. T5 兼容：前置科举、任官、朋党、弹劾、厂卫、战争仍正常。
7. T6 AI：观察 20 年，AI 不刷决议，不持续强推失败方案。
8. T7 长跑：观察 100 年，数值钳制、阶段可逆、角色数不线性增长。
9. T8 战争桥：单独启用革命 CB；验证胜/负/白和/失效四出口。

T0–T7 通过才能发布 MVP；T8 未通过则保留事件版革命，不阻塞 MVP。

## 8. 发布阻断

- 修改/覆盖任何前置公共文件；
- 非明获得变量、决议或事件；
- 出现财政、区域生产、市场、人口经济数据；
- 常态创建临时人物或在死亡后宣称已释放；
- 无活动内容仍进行人物/地图循环；
- 数字选项不显示主要修正或只有单向最优解；
- 3D/大批 2D 被复制进发布包；
- 日志持续未知 effect/trigger/scope 错误。
