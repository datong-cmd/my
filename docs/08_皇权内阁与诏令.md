# 08 《变身大明》皇权、内阁与诏令

> 参考机制解析。现行政策状态机见 `23A`；本文只说明前置可以读取什么。

## 1. 皇权

- 前置以 `huang_quan_value` 及相关 scripted value/trigger 表达皇权；
- 皇权受继承、战争、内阁、朋党和违制行为影响；
- 数值存在上下界与版本差异，精确增减入口由 `08A` 维护；
- 新项目默认只读，消费只经探针验证的 `HQ_change` 适配器。

## 2. 内阁与中枢人物

- 内阁大学士、六部九卿、司礼监等由现有 position/trait/事件承载；
- 任命、递升、摄政和亲政均可能改变人物权力；
- 票拟、批答、部覆和封还在前置事件中形成皇帝与官僚的程序性冲突；
- 这些人物可作为政策主持、反对者或调停者，不复制其职位。

## 3. 可借的玩法结构

```text
皇帝提出 → 内阁/部院表态 → 皇权投入或妥协 → 官僚执行 → 结果反馈
```

该结构可以由独立决议、角色事件、书信事件、title 变量和角色 modifier 实现，无需覆盖前置诏令事件或公共 GUI。

## 4. 风险

- 前置部分事件使用全局临时变量或特定 saved scope；不得按事件 ID 盲调；
- 同日继承时前置会调整官位与年号，新项目应延迟 1—3 日读取；
- 皇权强推不能成为免费成功按钮，施工书必须计入官僚阻力和长期反噬；
- 不建立第二套内阁、诏令资源或皇权变量。

精确皇权调用位置见 [08A](08A_皇权数值调用与门槛专项索引.md)。
