# 23 政治权力运行施工总表

> 定位：这是“变身大明兼容扩展”的政治层施工入口。前置边界见 [22A](22A_变身大明伴生依赖与明制专用技术底座.md)。本阶段只增加明制内部的政策博弈，不复制《变身大明》已经实现的皇权、科举、官阶、朋党、锦衣卫和弹劾系统。

## 1. 可交付边界

| 子系统 | 玩家问题 | 本模组新增 | 直接复用/只读 | 明确不做 |
|---|---|---|---|---|
| 诏令—内阁—执行 | 皇帝能否把意志变成结果 | 政策提案槽、复议、强推、执行期限、地方落实率 | `huang_quan_value`、大学士与六部/都察院/司礼监职位 | 第二套内阁和第二个皇权值 |
| 官僚供给—考成 | 谁能执行、为何阳奉阴违 | 官僚年度快照、改革专员、执行团队、利益偏向 | 科举结果、品级特质、原有考成和任官 | 改写科举事件或任官界面 |
| 朋党—政策联盟 | 社会力量怎样进入朝廷 | 围绕具体政策的支持/反对联盟 | `pengdang/dangkui/dangtu` 关系及影响力值 | 第二套永久党籍 |
| 监察—案件 | 监督是治理还是武器 | 改革案件槽、证据/程序/政治压力、结案反馈 | 原有弹劾、刑部、都察院、锦衣卫身份 | 复制原有审判全流程 |

## 2. 一条完整政治回路

`社会矛盾/改革议题 → 皇帝提出政策 → 内阁及关键官署表态 → 朋党与利益力量结盟 → 皇帝协商或耗皇权强推 → 执行团队落实 → 地方/官僚反馈 → 阶级力量、国家能力和下一轮政治条件变化`

这条回路的核心不是“点按钮得增益”，而是资源互换：强推可换速度，却提高政治债务和消极执行；妥协可扩大联盟，却可能牺牲改革深度；监察可清除阻力，也会提高恐惧和信息失真。

## 3. 数据所有权

所有全国政治状态放在 `title:h_greatming`：

| 变量/保存作用域 | 类型 | 范围/清理 | 用途 |
|---|---|---|---|
| `minghm_policy_active` | flag | 单政策期间 | 全局互斥锁 |
| `minghm_foreground_content_active` | flag | 任一复杂内容期间 | 政策/案件/危机/重大链共用互斥锁 |
| `minghm_policy_id` | int | 0..配置表上限 | 当前政策模板编号 |
| `minghm_policy_stage` | int | 0..6 | 提案、审议、裁决、执行、结算 |
| `minghm_policy_support` | int | -100..100 | 朝廷净支持 |
| `minghm_policy_capacity` | int | 0..100 | 执行能力快照 |
| `minghm_policy_resistance` | int | 0..100 | 既得利益阻力 |
| `minghm_policy_radicality` | int | 0..100 | 改革深度 |
| `minghm_policy_deadline` | date/flag duration | 到期清理 | 防止悬挂 |
| `minghm_policy_sponsor` | saved scope | 政策结束清理 | 主持官/专员 |
| `minghm_case_*` | 单案件槽 | 结案清理 | 见 23C |

人物只保存短期角色标记，例如 `minghm_reform_commissioner`、`minghm_policy_opponent`；不永久复制其官阶、党籍或社会身份。

## 4. 固定执行顺序

1. `minghm_start_policy_effect` 校验明帝、前台内容锁为空、政策模板可用，并同时设置两个 active flag。
2. 对有限官位持有人做一次快照，不遍历全世界人物。
3. 计算官署立场、既有朋党输入、社会力量输入。
4. 皇帝选择修改、交换条件、搁置或强推。
5. 锁定执行参数；执行期只在季度脉冲处理一次。
6. 到期按“政策深度 × 能力 − 阻力 − 执行损耗”结算。
7. 更新国家能力、阶级力量和历史阶段指标，并删除临时变量及共用前台锁。

## 6. 性能与失效策略

- 季度只处理当前唯一前台内容；年度只读取明帝宫廷内指定官位与少量候选人。
- 不用 `any_living_character`、全地图伯爵领或逐日脉冲。
- 读取前均用 `exists`/`?=` 防空；职位缺员视为能力下降，而不是报错。
- 参考模组键经版本探针失败时，关联桥接停用并显示兼容警告，不猜测替代键。
- 明帝死亡或大明暂时无持有者：保留全国长期值、清理人物 saved scope、暂停计时；复国后从新一季度继续，不补算多年。

## 7. 验收

- 同时启用本体、溥天之下、《变身大明》与本模组，顺序最后加载本模组，无重复键错误。
- 非 `h_greatming` 持有者不可见决议/互动，非明人物不获得宏观变量。
- 一项政策能完成提案—博弈—执行—反馈闭环；皇权、官僚、朋党至少各改变一个中间参数。
- 原模组的任官、科举、朋党、弹劾仍能独立正常运行。
