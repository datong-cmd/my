# 事件链五：奏疏风暴与厂卫大案

> 用证据、程序、舆论和恐惧四个数字模拟重大政治案件；不接管《变身大明》的官职、弹劾或厂卫系统。

## 1. 触发与边界

- 重大政策失败后，朋党影响或既有案件压力较高；玩家选择“彻查”，或 AI 在高危机时低频触发；
- 全国无其他重大事件链，预期 1—4 年；
- 不调用 `GM_Tanhe_events`，不写其内部 flag，不替玩家撤换其职位。

## 2. 案件状态

- `evidence` 证据、`procedure` 程序、`pressure` 朝堂压力、`fear` 厂卫恐惧，均为 0—100；
- 每阶段只保存一名主办 `lead`、一名被查 `opponent`、可选调停 `mediator`；
- 无合适人物时使用“涉案官僚集团”匿名化，不生成角色。

## 3. 固定事件骨架

| ID | 展示 | 内容与选择 |
|---|---|---|
| `minghm_case_chain.5000` | 书信事件 | 匿名奏疏或实名弹章：接奏、留中、秘密核查 |
| `.5010` | 普通角色事件 | 是否公开立案；公开加程序与压力，秘密加恐惧 |
| `.5020` | 普通角色事件 | 都察院、刑部/大理寺、锦衣卫或内廷主办 |
| `.5030` | 普通角色事件 | 账册、口供与人证冲突：复验、采信或压下 |
| `.5040` | 书信事件 | 消息外泄，朋党联署；公开程序或恐惧压制 |
| `.5050` | 普通角色事件 | 被查者反劾主办：换人、保护或双案并审 |
| `.5060` | 普通角色事件 | 朝审展示四值：证据优先、秩序优先或政治交换 |
| `.5070` | 普通角色事件 | 本模组给出政治结论；正式任免回到兼容模组界面 |
| `.5080` | 主题化角色事件 | 《大狱既结》，结算问责、公信、恐惧与朋党力量 |

可选插曲最多两项：证人翻供、遇刺传闻、御前泄密、地方联名；不同时生成支线。

## 4. 结局判定

```text
可信问责 = evidence + procedure - fear/2
政治失控 = pressure + fear - procedure
```

- 可信问责高、失控低：有限清算，国家能力与合法性上升；
- 两者接近：政治交易，精英凝聚上升但改革动量受损；
- 失控高：恐惧清洗，短期皇权上升，长期危机与保守力量上升；
- 证据低而强行定案：冤狱，朋党分裂和民众动员上升；
- 压案：当前危机下降，留下“未结责任”标记。

所有处分首先是全国值和角色修正；真正撤职交给《变身大明》既有任免机制。

## 5. 本体边界、AI与验收

- 书信事件承载奏疏，角色事件承载审讯；复用奏章、官署、监狱、密探氛围背景；
- 可读朋党关系、官职持有者、锦衣卫与宦官首脑，缺失时安全退化；
- 禁止覆盖原事件、复制职位、批量遍历官员、创建证人群体或证词数据库；
- AI 只按四值、性格标签和权力路线选择；
- 必测各主办机关存在/缺失、关键人物死亡、继承、压案与重启、主要结局及无重复任免。
