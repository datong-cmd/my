# 明代权力运行与历史变迁 Mod 策划资料总目录

## 项目目标

以 CK3 可实现的角色、头衔、政体、法律、修正、事件、决议、互动、派系、契约和界面机制为基础，制作一个围绕明代国家权力、重大事件博弈、改革/革新/革命和社会经济转型的长期策略 Mod。

本项目不追求逐年复刻史实。设计优先级是：

1. 长期且非一边倒的政治博弈；
2. 历史唯物主义的动态解释力；
3. 清楚、可操作、有反馈的玩家机制；
4. CK3 脚本可实现性、性能和 AI 可用性；
5. 史实与文学参考的气氛、矛盾结构和事件素材。

## 分析证据等级

后续每条结论使用以下标记，防止把文件名猜测写成事实：

- **S1 - 脚本确认**：已从实际脚本条件、效果和调用链确认；
- **S2 - 文档确认**：作者说明书明确描述，但尚未完成脚本交叉验证；
- **S3 - 本地化提示**：从 UI 文本可推断用途，仍需脚本验证；
- **D - 新设计**：为本项目提出的机制，不声称参考模组已实现；
- **R - 风险/待核对**：版本、DLC、性能或兼容性尚未确认。

## 固定边界

- 参考模组逻辑脚本做全量分类与机制解析；纹理、网格、字体、音乐只做分类，机制引用时按需定位。
- CK3 本体不全量读取，只追踪参考模组覆盖文件、调用到的接口和新设计需要的机制。
- 史料不做百科式堆积；只提炼行动者、资源、制度约束、冲突结构和可转化的事件素材。
- 历史阶段不是固定科技树，也不预设资本主义或革命必然胜利；允许混合、停滞、倒退、复辟和路径依赖。
- 所有主要机制都必须有 AI 路径、失败状态、反制手段、性能预算和 UI 反馈。
