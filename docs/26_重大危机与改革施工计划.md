# 26 重大危机与改革施工计划

## 1. 统一危机槽

重大危机不用 CK3 struggle、全新 situation GUI 或地区状态机，只使用 `h_greatming` 上一个事件槽：

| 字段 | 范围 | 说明 |
|---|---:|---|
| `minghm_crisis_active` | flag | 全国互斥锁 |
| `minghm_foreground_content_active` | flag | 与政策、案件、重大链共用 |
| `minghm_crisis_type` | 1..6 | 危机模板 |
| `minghm_crisis_stage` | 0..4 | 预警、爆发、对策、决战、余波 |
| `minghm_crisis_force` | 0..200 | 危机一方累计力量 |
| `minghm_response_force` | 0..200 | 朝廷/改革方累计力量 |
| `minghm_crisis_commitment` | 0..100 | 玩家已经公开许诺的改革深度 |
| `minghm_crisis_deadline` | duration | 6–24 个月 |
| `minghm_crisis_leader` | saved scope | 可空，优先现有人物 |

八项长期值继续由 `24` 所有；危机槽只保存本次对抗。只有共用前台锁为空时才能启动危机；危机期间不启动政策、案件或重大链。

## 2. 六个危机模板

| ID | 危机 | 本体/参考模组事实输入 | 核心对比 |
|---:|---|---|---|
| 1 | 皇位更替与摄政争夺 | `on_death/on_title_gain`、明帝继承、大学士/内廷身份 | 新君整合力 vs 外朝/内廷争权 |
| 2 | 朝争与行政瘫痪 | 原朋党影响、关键官位缺员、皇权 | 政策推动力 vs 守成与党争阻力 |
| 3 | 军事失败与边患 | 本体战争状态/结果、五军都督府/军职 | 国家能力+军队服从 vs 军事自主+危机 |
| 4 | 灾荒—民变压力 | 本体灾害/事件可用则读，否则低频随机门槛 | 救济组织力 vs 危机+民众动员失控 |
| 5 | 改革失败与反扑 | 本模组政策失败、原朋党/监察身份 | 改革联盟 vs 守成反攻 |
| 6 | 双重权力与革命 | 历史阶段 3、民众动员、精英分裂 | 革命力量 vs 王朝整合力 |

模板 4 不模拟粮价、库存、区域灾情，只把灾害视为政治压力事件。

## 3. 每阶段的数字对抗

事件先公开：

```text
朝廷响应 = state_capacity + elite_cohesion/2 + 选择的动员修正 + 关键官员修正
危机力量 = crisis_pressure + 模板主变量/2 + 反对集团修正 + 0..10随机
```

玩家每阶段从 2–4 个动作中选一个：妥协、行政处理、皇权强推、军事/民众/商业动员、监察压制。动作只调整两条当期力量和最多两项长期值。第四阶段按累计差值结算，不另建资源。

## 4. 改革项目模板

改革沿用 `23A` 政策状态机，MVP 固定六项：

| 改革 | 主要目标 | 推动力 | 阻力/代价 |
|---|---|---|---|
| 官僚考成与职责重整 | 提高国家能力 | 改革势头、吏部/首辅 | 守成力量、换人损耗 |
| 监察程序化 | 降低俘获和信息失真 | 都察院/刑部/大理寺 | 厂卫与朋党反扑 |
| 军令与军功重整 | 降低军事失控、提高应战 | 皇权、五军职位、军功身份 | 军事集团抵制 |
| 议政与公开承诺 | 提高精英协调/政策信用 | 朋党协调、改革势头 | 皇帝自由裁量下降 |
| 技术—军备组织革新 | 开启再工业化路线 | 国家能力、军职、改革网络 | 官僚负担、军事自主 |
| 扩大政治动员 | 提高改革或革命力量 | 民众动员、商业影响 | 结果脱离皇帝控制 |

没有“点击即升级”。每项都经过提案、廷议、执行、结算，成功与失败改变八项长期值。

## 5. 触发与 on_action

- `on_title_gain`：只检查模板 1；同一继承只触发一次。
- `on_war_ended_*`：仅当明帝或明制核心参战，检查模板 3；不遍历全部战争。
- 政策结算 effect：直接检查模板 5。
- 年度 Ming pulse：检查模板 2/4/6，最多选一个，满足冷却才触发。
- 不挂逐人物生日脉冲，不修改参考模组现有事件文件。

验收要求：六模板均能从预警走到清理；同一时间只有一种前台复杂内容；所有退出路径调用 `25A` 清理 effect 并释放共用锁；事件选项显示双方数值和长期代价；非明不触发。
