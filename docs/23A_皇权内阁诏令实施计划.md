# 23A 皇权—内阁—诏令实施计划

## 1. 复用依据

《变身大明》将皇权保存在明帝人物变量 `huang_quan_value`，`HQ_change` 负责修改并钳制在 `-500..2000`；大学士、六部、都察院、司礼监已有真实职位。因此本模组把皇权当“绕过协商的稀缺政治资源”，不再定义皇权条。

核验源：

- `common/scripted_effects/liuguan_effects.txt`：`HQ_change`
- `common/script_values/liuguan_values.txt`：`huangquan_windows`、`huang_quan_score`
- `common/council_positions/01_ministry_positions.txt` 与 `00_council_positions.txt`
- `common/court_positions/types/99_yan_court_positions.txt`

## 2. 皇权适配器

```text
minghm_hq_raw_value = holder.var:huang_quan_value（不存在则 0）
minghm_can_spend_hq_trigger = 是明帝 AND raw >= COST
minghm_spend_hq_effect = 再校验 → HQ_change { CHANGE = -COST CHECK = 437639088 }
```

`CHECK` 是参考模组内部保护参数，属于强耦合接口。发布前探针必须确认 `HQ_change`、变量名、边界与校验常数未变。任何普通结算不得直接 `change_variable = huang_quan_value`。界面可显示五档（受制、虚弱、常态、强势、专断），不缓存第二个规范值。

## 3. 政策模板定义

每个政策在 `minghm_policy_definitions.txt` 由固定 ID 和 scripted values 定义：

| 字段 | 例 | 作用 |
|---|---:|---|
| `radicality` | 60 | 改变既有关系的深度 |
| `admin_difficulty` | 45 | 对执行能力的要求 |
| `target_class` | 2 | 主要受益/受损力量索引 |
| `base_duration_months` | 24 | 执行周期 |
| `hq_override_cost` | 120 | 强推代价 |
| `required_office_mask` | 配置键 | 哪些官署必须参与 |
| `success_effect`/`failure_effect` | 独立 effect | 只写本模组状态 |

模板不使用动态数组；MVP 固定 6–10 个政策，便于 AI、文本和测试逐项覆盖。

## 4. 审议者与权重

仅抓取当前明帝宫廷的下列职位持有人：首辅/次辅（大学士特质或职位）、吏户礼兵刑工六部、左都御史、大理寺卿、锦衣卫、司礼监掌印。缺员不补生成。

个人立场：

`立场 = 性格倾向 + 官署职责 + 本人得失 + 朋党议程 + 对皇帝关系 + 腐败/恐惧修正`

官署权重固定配置，首辅和直接承办部较高。立场只在政策开始与重大让步后重算，随后保存全国汇总，不每月遍历人物。

## 5. 事件状态机

| stage | 入口 | 玩家动作 | 退出条件 |
|---:|---|---|---|
| 0 提案 | 决议选择政策 | 指定主持官、初始深度 | 保存模板与快照 |
| 1 廷议 | 即时事件 | 公开辩论/私下协调/压制异议 | 得到净支持与条件 |
| 2 复议 | 支持不足 | 降低深度、利益交换、延期 | 最多两轮 |
| 3 裁决 | 复议后 | 接受、撤回、耗皇权强推 | 锁定参数 |
| 4 执行 | 季度脉冲 | 追加资源/更换专员/查办阻挠 | 到期或崩溃 |
| 5 结算 | 到期事件 | 承认结果、甩锅或追责 | 写入反馈 |
| 6 清理 | hidden effect | 清临时变量 | 解除互斥锁 |

强推不等于成功：它只把 `support` 提升至可启动门槛，同时增加 `resistance`、恐惧和信息失真。撤回损失威望及政策信用，但避免长期政治债务。

## 6. 执行公式

`执行得分 = capacity + support/2 + 已投入资源 − admin_difficulty − resistance − 信息失真 + 0..20受控随机`

- ≥70：充分落实；40–69：折衷落实；10–39：纸面落实；<10：失败/反噬。
- 随机项只占小部分，关键参数在选项提示中公开。
- 结果改变本模组的国家能力、阶级力量、政策信用和阶段矛盾；原模组皇权只通过明确选择的 `HQ_change` 改变。

## 7. AI 与兼容验收

- AI 每年最多提出一次政策；战争、破产、活跃政策时降权或禁用。
- AI 强推条件：皇权储备充足、政策重要性高、预期执行得分不低于纸面落实线。
- 测试皇帝继承、职位空缺、首辅死亡、政策执行中大明失主、旧存档初始化。
- 检查日志无未知 `HQ_change`、无跨国决议可见、无原内阁 GUI 覆盖。
