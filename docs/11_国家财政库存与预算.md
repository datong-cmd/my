# 国家财政、库存与预算

> 文档性质：参考模组国家财政专项解析 + 新 Mod 财政状态机设计  
> 证据等级：脚本事实以 **S1** 标注；新设计以 **D** 标注；缺陷与待核风险以 **R** 标注。  
> 县域赋役见 [11A](11A_县域赋役土地矿产与基础设施.md)，私产与市场见 [11B](11B_私产地产行商与市场.md)，军器库存见 [10A](10A_兵种军器与军事现代化.md)。

## 1. 结论先行

参考模组已经证明，可以在 CK3 人物黄金之外构造一套“国家财政”：明帝拥有国库、内帑以及粮、缎匹、工料、军器、藏书五类库存；季度脉冲依次生产、采购、支付官俸宗禄、消耗军需、汇集地方赋税，并用自定义界面调整内帑分成、俸禄标准、中央截留和外购数量。

它的最大价值不是数值平衡，而是把国家剩余的取得与分配显式化：县域农民和土地生成粮税，地方官府留用一部分，中央取得一部分，皇室内帑再截取一部分；官僚、宗藩、宦官和宫廷人员各有领取通道。财政因此天然是权力关系，而不是单一货币条。

现有实现的主要局限是：资源多使用世界级 `global_var`，无法支持多个中国政权；季度顺序、零值边界和透支检查存在风险；许多收支由人物属性或军队规模直接生成，尚未形成生产—征收—运输—分配的闭环；玩家可用滑条频繁改变制度参数，代价主要是暴政值，而不是组织执行、利益集团和信用反应。

**D：** 新 Mod 保留“国库 + 内帑 + 少量实物库存 + 地方留成”的结构，但把核心变量绑定到国家主头衔，压缩为可解释的预算账户，并加入财政信用、税收可达性、拖欠、专项预算和利益集团索取。预算每年决定，季度执行；危机时可以挪用，但每次挪用都会重新分配政治力量。

## 3. 季度执行顺序

**S1：** 明帝的季度可玩角色脉冲依次调用：

```text
1  guoshui_shouru       国税转入内帑、宫廷经营收入
2  guoku_zhuchong       宫廷、宗室、宦官俸给
3  duanpi_shouru        缎匹生产与采购
4  gongliao_shouru      工料生产与采购
5  junqi_shouru         军器生产、采购与战争消耗
6  cangshu_shouru       藏书生产与采购
7  shuiliang_shouru     税粮收入、采购与霉耗
8  shuiliang_zhichu     官员、宗藩粮俸
9  gexiangmu_zhichu     地方赋税、专项支出、外购付款
```

**S1：** `gexiangmu_zhichu` 内部再调用：

```text
waiguan_fengyin_effect  # 县域生产、留成、上解和官员封印
zaxiang_zhichu_effect   # 锦衣卫、赈济、赔偿等杂项
waigou_zhichu_effect    # 五类资源采购付款与价格刷新
```

**R：现金流时序。** 地方赋税在本季度多数资源生产、工资和支出之后才上解，因此它更像为下一季度提供现金；若 UI 只显示季度总额而不说明时序，玩家会难以理解“账面本季有收入却先发生短缺”。

**D：** 新 Mod 明确区分“预算批准—季度执行—年末决算”。季度结算统一采用：期初余额 → 固定收入 → 固定支出 → 战争/灾荒冲击 → 可选应急融资 → 期末拖欠。界面同时显示本期与下期预测。

## 4. 国库与内帑

### 4.1 国税转内帑

**S1：** `guoshui_shouru` 在明帝作用域初始化 `neitang_zhanbi`，默认约 10%。季度转入量以人物月收入乘比例和三个月计算；直接持有县数较多时另乘约 0.3，上限约 1000。效果增加内帑，同时从短期人物黄金扣除等额数值。

县域上解完成时，`waiguan_fengyin_effect` 还会从中央税收中再按内帑比例截取。因此内帑至少存在两个入口：人物收入代理和地方财政结算。

**R：重复计征风险。** 若人物月收入已经包含由县域税收修正产生的收入，再额外从地方上解中截取，就可能重复表达同一税源。必须在新 Mod 中规定每笔收入只进入一次总账。

### 4.2 宫廷经营收入

**S1：** `guoshui_shouru` 会遍历皇帝宫廷中的 `slave` 与 `shouma` 人物。收入依据性别、身份以及乐器、歌咏、礼书、文学、知识、语言等特质经验计算，并进入内帑。

这段逻辑把宫廷中的人身依附直接货币化，属于“皇室家产经营”，不是公共税收。它也让宫廷人口规模成为财政来源。

**R：性能与伦理表现。** 每季度遍历全体廷臣，且收入随人物数量增长，容易鼓励玩家囤积人物。新 Mod 若保留，只应做宫廷产业的聚合等级和政治事件，不逐人计价。

### 4.3 内帑的政治含义

**D：** 内帑不是第二个无色国库，而是皇权的自由裁量资源：

- 可绕过内阁快速赈灾、赏军、收买官员和建立密探；
- 可供养宫廷、宗室、私人武装和皇室产业；
- 过高截留会降低公共服务、官僚合作和财政信用；
- 过低则削弱皇帝在危机中的自主权，并使其依赖内阁或商人融资；
- 将内帑公开化可提高信用，却会让议政机构取得审计权。

内帑占比必须是制度决策或政治谈判，不应是每季度无条件拖动的滑条。

## 5. 宫廷、宗室与官僚俸给

**S1：** `guoku_zhuchong` 在内帑不少于约 100 时，向特定女官、司礼监/掌印体系、地方宦官和皇族发放私人财产。不同性别、婚姻状态、官职和宗室身份对应不同额度，并累计到名为 `guoku_tanwu` 的统计变量。

**R：命名与实义冲突。** `guoku_tanwu` 字面是“国库贪污”，脚本实际累计的却包含正常俸给与宗禄。后续若把该变量解释为腐败，将混淆合法分配和侵吞。

**R：透支。** 逻辑只在进入整段发薪前检查一次内帑是否达到门槛，没有对每一笔付款重新检查余额。人员很多时可能把内帑推为负值，取决于增减效果是否夹限。

**R：遍历成本。** 季度遍历所有下属统治者及其廷臣以寻找地方宦官；规模扩张后成本会明显增加。

**D：** 新 Mod 将领取者聚合为预算项目：

| 项目 | 受益集团 | 未足额支付的后果 |
|---|---|---|
| 文官俸禄 | 官僚、胥吏 | 行政效率下降、寻租与朋党保护上升 |
| 军饷 | 将领、军官、士兵 | 逃亡、劫掠、兵变、倒向地方派系 |
| 宗禄 | 宗藩、皇族 | 皇族结社、抵押资产、干预地方 |
| 宫廷费 | 宦官、女官、近侍 | 皇帝情报与私人执行能力下降 |
| 赈济 | 农民、城市贫民 | 流民、粮价危机、基层组织激化 |
| 公共工程 | 工匠、民夫、地方官 | 长期税基与运输能力下降 |
| 债务本息 | 商人、金融资本 | 信用崩溃、未来融资成本上升 |

不逐个角色季度发放数值；只有关键人物在拖欠事件中代表集团提出条件。

## 6. 五类实物库存

### 6.1 粮食

**S1：** `shuiliang_shouru` 以人物月收入约六倍估算季度粮入；直接持有县过多时乘约 0.25，领地很小时乘约 1.5，并加入外购粮。军粮支出以最大军事力量约 3%为基础，处于战争时翻倍；持有县较多且处于战争时还可能再乘三倍。

库存超过约 3000 时会随机霉损约 2%—5%。户部尚书可从霉损量中取得约 3%—5%的私人财产；满足特定富裕、未婚条件时还可能生成婚配人物。

**R：因果代理过强。** 人物收入既不是粮食产量，也不等于可调运税粮；最大军事力量也不等于实际动员。霉损与官员私人收益绑定有讽刺性，但生成婚配人物与财政链关系薄弱。

### 6.2 缎匹

**S1：** `duanpi_shouru` 的本地产出约等于领地规模三分之一加户部尚书管理能力；若没有对应大臣，本地产出可能归零，再加外购量。

### 6.3 工料

**S1：** `gongliao_shouru` 以领地规模乘工部尚书管理加成为产出；缺少工部尚书时加成变量可能为零，导致本地产出归零。

### 6.4 军器

**S1：** 军器按最大军事力量与工部能力生成，战争时消耗；细节、风险和替代设计见 [10A](10A_兵种军器与军事现代化.md)。

### 6.5 藏书

**S1：** `cangshu_shouru` 以直辖规模加礼部尚书学识为本地产出，缺少礼部尚书时可能归零，再加入外购。

### 6.6 资源模型评价

参考模组用五种物资给不同国家部门配置实物基础，方向正确：粮食支持军民，缎匹支持礼制与赏赐，工料支持工程，军器支持战争，藏书支持教育与文化再生产。

**R：人物缺位即停产。** 大臣职位更适合影响组织效率，而不应决定整个国家是否从零开始生产材料。职位空缺可降低收集和调度能力，但基层生产仍存在。

**R：世界单例。** 这些库存多以 `global_var` 保存；多政权、内战和南北朝廷会共享账本。

**D：首版建议只保留四类国家资源：**

1. 粮储：军队、赈济、城市稳定；
2. 财政信用：货币税、借款和未来收入承诺；
3. 工业物资：工程、军工与再工业化的综合投入；
4. 知识/技术能力：书籍、学校、译介和技术人员的聚合值。

缎匹降为礼制型奢侈品或事件资源；军器作为军事子系统库存。这样既保留政治含义，又避免玩家管理六七条彼此同质的进度条。

## 7. 外购、出售与价格

**S1：** 京仓界面允许皇帝设定每季度购买粮、缎匹、工料、军器和藏书的数量，并可出售库存。`waigou_zhichu_effect` 汇总“数量 × 当期价格”，从国库/人物黄金扣款，随后刷新随机价格。出售价格通常低于购入价，约为九折，形成交易损耗。

**R：价格是随机参数，不是社会结果。** 现有价格没有稳定读取战争、丰歉、海禁、贸易路线、库存、商人垄断或信用风险。玩家主要是在随机价上做套利，而不是与供应者博弈。

**D：** 新 Mod 只计算年度市场环境档位：宽松、正常、紧张、封锁。其来源包括国内产量、重大灾害、战争海禁、贸易政策和商人关系。每次大宗采购还会提高商人资本力量、债权或特许要求，避免国库把市场当无限售货机。

## 8. 京仓界面的政策杠杆

**S1：** `jingcang_gui.txt` 提供：

- `qiyun_coef`：中央起运/截留系数，约 0—0.9；上调会增加暴政；
- `zonglu_coef`：宗禄系数，约 0.5 起；
- `fenglu_coef`：官俸系数；
- `fengyin_coef`：地方官留成/封印系数；
- `neitang_zhanbi`：内帑占比，约 0.05—0.5；上调会增加暴政；
- 国库向内帑的转移；
- 五种资源采购与出售。

这套 UI 已经把财政分配变成玩家可操作权力，但主要代价是一次性暴政，利益集团没有组织化回应。

**D：** 新 Mod 把滑条改为年度预算议程：玩家先提出方案，受益与受损集团表态，政治影响力决定通过方式。强行颁行仍可用皇权，但会产生执行折扣、隐瞒、拖欠或联合反对。预算通过不等于足额征收到手。

## 9. 专项支出

**S1：** `zaxiang_zhichu_effect` 包含：

- 锦衣卫经费，受相关人物谋略和随机因素影响；
- 赈济支出，受领地规模影响；
- 冤狱赔偿，受刑部/司法大臣学识影响；
- 河工与漕运等变量在当前片段中初始化为零；
- 支出从国库或人物黄金扣除。

专项支出证明脚本能够在常规俸禄外建立政策账户，但没有形成“拨款—执行—验收—政治后果”链。

**D：** 重要专项采用四阶段状态机：

```text
立项 -> 筹资 -> 执行 -> 验收/追责
```

每一阶段允许截留、延期、追加预算、地方配套和集团交易。结果不是简单成功率，而是工程效果、财政信用、受益阶级、腐败网络和行政学习同时变化。

## 10. 财政作为阶级关系

财政不是从“社会”外部抽取一个抽象值。它是生产剩余如何被占有、运输和重新分配的制度化形式。

**D：首版五类财政负担/收益主体：**

| 集团 | 主要缴纳 | 主要取得 | 典型诉求 |
|---|---|---|---|
| 自耕农/佃农 | 田赋、徭役、粮差 | 赈济、水利、治安 | 减免、稳定地权、反加派 |
| 地主士绅 | 地租、部分税与摊派 | 地方控制、官位、工程利益 | 税收豁免、地方留成、地权保护 |
| 商人/金融资本 | 商税、关税、借款 | 利息、特许、采购合同 | 合同司法、低内地关卡、政策发言权 |
| 官僚与胥吏 | 行政劳动/政治忠诚 | 俸禄、规费、寻租 | 稳定预算、任免权、审计豁免 |
| 军官与士兵 | 服役、战争风险 | 军饷、土地、战利品 | 优先支付、军政自治、装备 |

玩家加税时，系统必须明确由谁实际承担、由谁代征、谁可以转嫁以及谁因此组织起来。

## 11. 新 Mod 财政状态机

### 11.1 国家级核心变量

```text
state.var:public_treasury
state.var:imperial_purse
state.var:grain_stockpile
state.var:industrial_inputs
state.var:fiscal_credit
state.var:tax_reach
state.var:arrears_civil
state.var:arrears_military
state.var:debt_service
state.var:budget_policy
```

状态放在国家主头衔或唯一政权载体，绝不使用世界单例保存某国库存。

### 11.2 年度预算

```text
可支配收入 = 潜在税基
             × 税收可达性
             × 合规度
             × 货币化程度
             × 战乱/灾害系数

财政余额 = 可支配收入 + 新借款 + 资产处置
           - 固定俸饷 - 宗禄 - 债息
           - 军费 - 赈济 - 工程 - 改革投入
```

预算不追求会计细目，而要回答三个问题：谁出钱、谁先拿钱、欠谁的钱最危险。

### 11.3 拖欠状态

支出不足不立即消失，而转为三档拖欠：轻微、严重、崩溃。拖欠对象不同，政治后果不同。拖欠文官产生寻租与行政怠工；拖欠军饷产生兵变；拖欠债息摧毁信用；削减宗禄推动宗藩抵制；赈济不足扩大流民和基层组织。

### 11.4 应急融资

玩家可选择：

- 内帑垫付：提高皇帝声望和控制，但消耗私人自主资源；
- 商人借款：立即得钱，增加利息、资本力量和政策条件；
- 强制摊派：短期收入高，降低合规并推动地主/商人联合；
- 货币贬值：缓解名义支出，抬高物价并伤害领薪者与城市贫民；
- 卖官与特许：换取现金，腐蚀官僚再生产并强化新买入者；
- 没收财产：打击特定集团，若执行失败则触发资本外逃和政治结盟。

没有无政治代价的“借钱”按钮。

## 12. 长期博弈结构

### 12.1 皇帝与内阁

皇帝希望保有内帑和自由裁量，内阁希望收入统一入账并取得审计。皇帝过度截留会使公共财政失灵；内阁完全控制预算又可能架空皇权。双方可在战争、赈灾和改革中临时交换权力。

### 12.2 中央与地方

提高中央起运可扩大国家能力，但地方工程、官俸和赈济不足会降低下一期税基与执行意愿。降低起运能稳定地方，却增强地方官绅联盟。最优比例随战争、灾害、交通和商业化改变，不存在永久答案。

### 12.3 国家与债权人

信用允许国家提前动员未来剩余，也让债权人拥有结构性权力。按期还本付息提高融资能力，却挤压民生和军费；违约释放当期财政，却可能造成资本罢市、贸易断供或要求以政治权利换展期。

## 13. AI 策略

AI 不遍历每笔预算，使用四种财政姿态：

| 姿态 | 触发 | 行为 |
|---|---|---|
| 守成 | 无大战、信用正常 | 优先固定支出与小额储备 |
| 战时动员 | 外敌/内战迫近 | 军饷、军器优先，可借债和挪用 |
| 恢复 | 战后、灾后、拖欠高 | 降低扩张，优先赈济、信用和税基 |
| 改革投资 | 稳定且有政治支持 | 承担短期赤字换税制、交通、工业能力 |

AI 根据统治者性格和支持集团选择融资工具，不追求精确最优。每年决策一次，重大危机可插入一次应急重算。

## 14. 性能预算

| 逻辑 | 频率 | 对象 |
|---|---|---|
| 国家总账 | 每季度 | 每个主要中国政权一次 |
| 年度预算 | 每年 | 政权一次 |
| 地方税基汇总 | 每年或变化时缓存 | 8—15 个财政区 |
| 资源价格档位 | 每年/重大冲击 | 区域市场一次 |
| 集团财政反应 | 预算通过时 | 5—7 个集团 |
| 关键人物索取 | 事件触发 | 每集团最多一名代表 |

禁止季度遍历全体廷臣发薪、禁止把全部县逐笔汇总到 UI、禁止以人物数量直接生成宫廷收入。

## 15. 首版最小可行内容

1. 国库、内帑、粮储、财政信用四项核心状态；
2. 文官、军饷、宗禄、赈济、工程、债息六项预算；
3. 中央起运、内帑截留、税率、举债四项政策；
4. 文官拖欠、军饷拖欠、债务违约三个危机状态；
5. 一套年度预算界面和下季度预测；
6. 皇帝—内阁、中央—地方、国家—债权人三组谈判事件；
7. 战争、灾荒、继承和改革启动四种预算冲击。

## 16. 验收标准

1. 每笔核心收入只进入总账一次，不重复计征；
2. 多个中国政权各有独立国库和库存；
3. 国库与内帑用途和政治后果不同；
4. 预算不足产生拖欠状态，而不是把余额无提示推成负数；
5. 玩家能知道本季度实际收支和下一季度预测；
6. 提高中央截留会同时增强中央能力并损害地方再生产；
7. 大宗外购会增强供应者或债权人的政治力量；
8. 大臣缺位降低行政效率，但不会让全国产出归零；
9. 至少三种融资方式各有不同阶级后果；
10. 财政危机既可能推动改革，也可能引发反改革、违约或革命；
11. AI 能维持基本俸饷，并在战争与恢复间转换预算姿态；
12. 国家季度结算不扫描全世界人物或逐个廷臣。
