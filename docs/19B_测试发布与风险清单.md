# 测试、发布与风险清单

> 本文把“感觉能运行”改成可重复的证据。绝对性能数值取决于硬件，因此以相同机器、相同游戏版本、相同开局和相同观察方式的本体基线为准。

## 1. 测试层级

| 层 | 名称 | 何时运行 | 主要问题 |
|---:|---|---|---|
| T0 | 文本与静态检查 | 每次修改 | 解析、命名、重复 ID、本地化、越权写入 |
| T1 | 接口探针 | adapter/本体接口变化 | effect/trigger/scope 是否真实可用 |
| T2 | 原子功能 | 每个公共接口 | clamp、dirty、锁、清理、失败无副作用 |
| T3 | 垂直切片 | 每个机制提交 | 玩家/AI 完整调用链 |
| T4 | 存档与生命周期 | 每个持久状态变化 | 新局、读档、继承、死亡、失位、迁移 |
| T5 | 系统集成 | 每个里程碑 | 多机制耦合、危机槽、承诺和结算顺序 |
| T6 | AI 与平衡观察 | 每次平衡批次 | 路线分布、惯性、误判、失败可玩性 |
| T7 | 性能与稳定长跑 | 里程碑/发布 | 100/200 年速度、对象泄漏、存档增长 |
| T8 | 兼容与发布 | beta/正式发布 | DLC、版本、加载顺序、升级与用户说明 |

## 2. 测试环境记录

每份可比较报告保存：

- CK3 精确版本；
- DLC 启用列表；
- Mod 版本、Git commit 和 schema；
- 加载顺序和同时启用 Mod；
- 开局日期、国家、统治者和随机种子；
- 游戏规则、难度、观察模式和速度；
- 机器 CPU、内存、系统与是否后台负载；
- 本体基线/本 Mod 构建；
- 起止游戏日期、现实时间、存档大小；
- error.log 等日志的清理时间与差分范围。

没有环境记录的“变快/变慢/经常发生”只算线索，不算回归证据。

## 3. T0 静态阻断项

### 3.1 文件与路径

- [ ] 文件编码符合脚本/本地化要求；
- [ ] 无允许清单外的本体同路径碰撞；
- [ ] 无临时文件、测试存档、本体/DLC/创意工坊源文件；
- [ ] 新图像/音频有来源和许可边界；
- [ ] descriptor 支持版本与文档一致。

### 3.2 定义与调用

- [ ] 事件 ID 唯一、号段正确、namespace 存在；
- [ ] scripted trigger/value/effect 定义唯一；
- [ ] 调用者能找到定义；
- [ ] 公共接口已登记在 `18A`；
- [ ] 关键事件登记了入口、scope、锁、冷却和测试；
- [ ] 无未经验证的本体/DLC effect 名；
- [ ] 无发布占位符、省略号、`BLOCKED`、`FIXME`、`TODO`。

### 3.3 状态与所有权

- [ ] 新持久变量有载体、范围、初始、所有者、写入者、清理和迁移；
- [ ] 非所有模块没有直接写源状态；
- [ ] 缓存只由重算 effect 写；
- [ ] UI/AI 不直接写持久变量；
- [ ] 活动槽有完整 clear effect；
- [ ] 所有标准量经公共 effect clamp；
- [ ] 空槽不存在遗留字段。

### 3.4 性能模式

- [ ] 高频入口没有 world/character/map 全扫描；
- [ ] UI value 不进行国家级循环；
- [ ] region 循环最多登记的 8/12 个 anchor；
- [ ] relevant character 常态/硬上限可检查；
- [ ] AI 新议程不是日/周检查；
- [ ] 每个动态对象有截止、冷却和清理。

### 3.5 本地化

- [ ] 每个玩家可见 key 有简体中文；
- [ ] 无重复 key 和非法语言头；
- [ ] tooltip 不隐瞒确定成本；
- [ ] 未知信息未显示虚假精确值；
- [ ] C/P/K 内部编号不直接替代玩家文本；
- [ ] scope 函数和图标语法从本体实例确认。

任一 T0 阻断项失败，不进入实机测试包。

## 4. T1 Adapter 探针

每项原生接口建立最小独立场景：

| 接口 | 正向 | 反向/边界 | 存档 | 缺 DLC |
|---|---|---|---|---|
| 适用天朝/国家识别 | 正确识别目标 | 非目标不初始化 | 读档一致 | 安静禁用 |
| 国家国库 | 增减一次、tooltip 正确 | 不足/负值按原生规则 | 数值一致 | 不建立影子国库 |
| 功绩 | 合法人物读/写 | 非法 scope 安全 | 继承/死亡合理 | 降级 |
| 任命 | 候选/职位接口有效 | 无候选有后备 | 任职状态一致 | 禁用依赖选项 |
| 家产 | 只读/必要写入有效 | 无家产安全 | 继承正确 | 降级 |
| situation | 创建、参与、阶段、关闭 | 上限/无参与者 | 保存恢复 | 固定槽降级 |
| 王朝周期 | 读取阶段/状态 | 非天朝不调用 | 一致 | 不桥接 |
| 工程/火器产能 | 读本体能力 | 未解锁安全 | 一致 | 自有抽象不假冒原生 |

探针代码不进入玩家正式事件，或由 debug game rule 隔离。

## 5. T2 原子接口用例

每个 change effect 至少覆盖：

1. 正常正增量；
2. 正常负增量；
3. 超过 100 clamp；
4. 低于 0 clamp；
5. 无变量初始化行为；
6. 无效 root scope；
7. 无效参数；
8. 重复调用是否按定义叠加/幂等；
9. dirty 请求正确；
10. debug 日志只在调试模式输出。

每个事务 effect 至少覆盖：

- 前置失败不扣成本；
- 锁已占用不重复执行；
- 费用只扣一次；
- saved scope 失效时走后备；
- 完成后锁释放；
- 中途事件链中断可恢复；
- 清理移除全部字段和参与组。

## 6. T3 垂直切片测试模板

| 维度 | 至少一个用例 |
|---|---|
| 玩家成功 | 按预期推进并结算，但有真实代价 |
| 玩家政治失败 | 物质条件足够但联盟/执行失败 |
| 玩家物质失败 | 支持足够但钱、粮、运输或技术不足 |
| 主动回撤 | 保留已付成本、先例和记忆 |
| AI 成功 | 无玩家操作完成全链 |
| AI 妥协 | 根据有限信息选择次优可行路径 |
| AI 误判 | 合理不确定性造成可解释失败 |
| 反制 | 反对者实际改变执行/成本/危机 |
| 空候选 | 无人物/区域候选时安全降级 |
| 并发 | 达到改革/项目/危机上限后拒绝新建 |

测试报告列出实际变量前后值、事件链、日志和是否满足策划预期，不只截图最终界面。

## 7. T4 生命周期与迁移

### 7.1 新游戏

- 初始化一次；
- schema、区域、阶级、联盟、阶段和空槽完整；
- 同日重复 on_action 无第二次初始化；
- 非适用国家无持久垃圾变量。

### 7.2 保存/读取

在下列时点分别存读档：

- 无活动对象；
- 改革提案后；
- 危机 K2；
- 承诺已接受未兑现；
- 年度结算前一个月；
- 年度结算后；
- 统治者死亡事件期间。

读取后检查 scope、槽、期限、锁、dirty、缓存和下一事件。不允许重复弹出或重复扣费。

### 7.3 人物与国家生命周期

| 变化 | 预期 |
|---|---|
| 执行官死亡 | 替换、延期或失败；无 invalid scope |
| 联盟领袖死亡 | 有事件驱动替换；联盟聚合仍在 |
| 统治者继承 | 国家状态保留，人物承诺按规则继承/重谈 |
| 主头衔更名/继承 | state anchor 仍有效 |
| 国家分裂 | 区域/危机按登记规则分配或收口 |
| 国家灭亡 | 停止调度，结算/清除活动对象 |
| 国家恢复 | 按 schema 休眠状态恢复或重新初始化，不能双份 |

### 7.4 Schema 迁移

每个版本保存至少一份最小旧 schema 测试档：

- 逐级迁移；
- 迁移前备份；
- 缺失字段采用文档默认；
- 废弃字段清除；
- 活动对象无法迁移时给出保守结算与一次说明；
- 迁移重复运行无副作用；
- 失败不把 schema 提前改成新值。

## 8. T5 集成不变量

每季度/年度后检查：

- 区域先结算，全国只聚合一次；
- 阶级源状态与缓存没有同一调用链相互覆盖；
- 联盟支持来自最新阶级缓存；
- 阶段审核发生在所有聚合之后；
- 活动改革 ≤2、项目 ≤3、危机符合全国/区域上限；
- relevant characters 常态 ≤20、硬上限 ≤40；
- 已结算危机把承诺和记忆写完后才关闭；
- 取消项目先处理债务、劳动和区域后果；
- GUI 不因 dirty 自行全量重算；
- 原生国库、功绩和任命没有影子真值。

## 9. T6 AI 与平衡

### 9.1 观察指标

| 指标 | 需要回答 |
|---|---|
| 路线份额 | 是否某路线在多数结构下垄断 |
| 路线持续 | AI 是否每季度摇摆，或永不调整 |
| 提案频率 | 是否无事可做/事件过载 |
| 完成/回撤/失败 | 失败是否全部来自资源不足 |
| 承诺兑现 | AI 是否无代价背叛或永不背叛 |
| 联盟变化 | 物质利益变化是否进入政治 |
| 阶段分布 | 是否所有国家自动 P0→P7 |
| 危机复现 | 同一危机是否无结构变化地重复 |
| 结局续玩 | 革命/反革命/分裂后是否仍可治理 |
| 玩家对照 | 难度是否来自规则而非 AI 免费资源 |

### 9.2 种子和初始结构

至少覆盖：

- 中央强/地方弱；
- 中央弱/士绅强；
- 商业繁荣/财政弱；
- 军事强/军饷不稳；
- 小农占优/灾害高；
- 土地集中/劳动强制高；
- 高工业潜力/运输瓶颈；
- 精英分裂/群众组织低；
- 群众组织高/替代能力低；
- 外压高/国家合法性低。

单一开局随机 100 次不能替代结构矩阵；需要同时改变初始社会关系。

### 9.3 平衡警报

- 路线在明显不同结构下仍总是最优；
- 玩家能靠循环同一事件无限刷能力/资源；
- 高压制同时永久降低不满、组织和生产成本；
- 工业能力无视资本、运输、劳动和需求持续上涨；
- 资本形成必然带来国家能力或公共福祉；
- 高 grievance 自动产生革命；
- AI 通过隐藏真值总能选正确选项；
- 难度只给 AI 钱、兵或成功率；
- 失败把玩家锁死在无决策状态超过合理恢复期。

## 10. T7 性能与长跑

### 10.1 基线方法

同环境分别运行：

1. 本体 + 同 DLC；
2. 本体 + 本 Mod；
3. 必要时本体 + 参考模组（只作结构参考，不作为硬性能目标）；
4. 本体 + 本 Mod + 目标兼容组合。

目标沿用 `17A`：100 年观察现实时间相对本体尽量不超过约 5%，超过约 10% 阻断；存档增长同理。若测量噪声大，重复运行并报告区间。

### 10.2 检查点

在 0/10/25/50/100/200 年记录：

- 现实耗时；
- 存档大小；
- 活动改革/项目/危机数；
- 相关人物数；
- lingering flags/modifiers/saved scopes；
- 年度事件数量和玩家弹窗数量；
- error.log 新错误计数；
- 适用国家数量和已休眠国家；
- AI 议程/危机重复频率。

### 10.3 泄漏测试

强制创建并关闭上限数量的：

- 100 次改革；
- 100 次项目；
- 100 次危机；
- 反复更换联盟领袖；
- 反复继承/国家分裂（调试场景）。

每轮后对象、modifier、flags、saved scopes 应回到规定基线，记忆聚合值除外。

### 10.4 性能回归定位

若超预算，按顺序隔离：

1. UI 打开/关闭；
2. AI 议程；
3. crisis scheduler；
4. region annual settlement；
5. class/alliance aggregate；
6. relevant actor maintenance；
7. event director；
8. compatibility bridge。

不先随意砍功能；先用开关和计数确定热点，再缓存、错峰、缩小候选或降低频率。

## 11. T8 兼容矩阵

### 11.1 必测组合

| 组合 | 目的 |
|---|---|
| 本体，无目标 DLC | 验证安全禁用/降级 |
| 本体 +“溥天之下” | 主支持环境 |
| 本体 + 全 DLC | 常见完整环境 |
| 本体 + 本 Mod | 独立运行 |
| 本体 + 参考模组 + 本 Mod | 检查前缀、路径和重复机制冲突 |
| 本体 + 一个常见 UI Mod | 检查界面覆盖风险 |
| 上一公开版存档 + 新版 | 迁移 |

不宣称未经测试的兼容。参考模组同时启用若存在设计层重复，应在兼容说明区分“技术可加载”和“玩法不建议叠加”。

### 11.2 覆盖审计

- 新 Mod 通用同路径覆盖目标为零；
- 数据库同键覆盖目标为零；
- 必须 patch 的个别项进入机器可读 allowlist；
- allowlist 每项注明原因、版本、上游文件 hash/检查和降级；
- 上游版本变化时自动提示重新审计。

## 12. 玩家可读性测试

招募不读策划文档的测试者完成：

1. 找到当前国家主要矛盾；
2. 解释一个阶级“力量高但不满低”和另一个“力量低但不满高”；
3. 选择一项改革并说出谁支付代价；
4. 在危机中识别已知事实和未知风险；
5. 找到一个反对联盟的反制手段；
6. 解释为什么工业增长没有自动变成国家胜利；
7. 失败后找到下一步恢复策略。

若测试者只能看数值但说不出关系，优先改 tooltip、摘要和事件因果，不继续加新仪表。

## 13. 发布阻断条件

以下任一存在即不得公开发布：

- 新增解析错误、invalid scope 高频错误或未定义键；
- 可稳定复现的存档损坏、重复扣费或重复年度结算；
- 活动对象无界增长或清理泄漏；
- schema 迁移覆盖原值/重复奖励；
- 缺 DLC 导致刷屏或无法加载；
- 主要 AI 机制完全不运行或读取隐藏真值；
- 玩家重大选项隐瞒确定成本；
- 一条路线在测试矩阵中严格支配且无结构性代价；
- 100 年性能/存档超过阻断预算且未解释；
- 上传了无权分发的本体、DLC、创意工坊或第三方资产；
- 文档、代码和版本说明描述不一致。

## 14. 可接受的非阻断问题

在 beta 中可接受但必须记录：

- 个别历史文本变体不足；
- 稀有开局的区域初始值不够精细；
- tooltip 排版需要润色但成本信息完整；
- 非主支持语言缺失；
- 不影响状态的动画/音效问题；
- AI 在极端结构下选择次优但可解释；
- 少数兼容组合未测试且明确标记不支持。
