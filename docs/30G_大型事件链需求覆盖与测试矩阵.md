# 大型事件链需求覆盖与测试矩阵

## 1. 需求覆盖审计

| 用户要求 | 落地位置 | 结论 |
|---|---|---|
| 立足 CK3 本体/DLC 可实现 | `30` 的 story、事件、延迟触发与头衔状态 | 已覆盖 |
| 与《变身大明》同时启动 | `30` 的只读适配、`30E` 的任免/弹劾隔离 | 已覆盖 |
| 只对明制生效 | 所有入口要求 `h_greatming` 与明帝包装 trigger | 已覆盖 |
| 无财政、区域生产、市场 | `30D` 只做知识组织阶段；全系列无经济账本 | 已覆盖 |
| 唯物史观、阶级斗争、阶段变迁 | `30B/30D/30F` 的组织能力、力量对比与可逆终局 | 已覆盖 |
| 长期且非一边倒 | 每链至少四类结局，胜利均有代价和后续矛盾 | 已覆盖 |
| 性能优先、允许假数字 | 单链锁、定点重算、整数对抗、无全图扫描 | 已覆盖 |
| 直接复用美术 | 每链指定现成 theme/background，零新 3D | 已覆盖 |
| 可交给 AI 编码 | 固定 ID、字段、公式、入口、清理与批次写入 `27A` | 已覆盖 |

## 3. 通用测试用例

| 编号 | 操作 | 通过标准 |
|---|---|---|
| MC01 | 非明角色观察 20 年 | 无变量、决议、story 或事件 |
| MC02 | 同日满足两条链入口 | 只创建一条，另一条只留候选或冷却 |
| MC03 | 链中皇帝死亡 | 旧 story 结束；头衔状态保留；新帝重建并续链 |
| MC04 | 主办/对手死亡或失效 | 清 scope，选择替代或匿名退化，不刷错 |
| MC05 | 可选 DLC 美术引用失效 | 回退本体主题，不影响逻辑结算 |
| MC06 | 前置职位、党争、厂卫同时运行 | 无 namespace/flag/职位覆盖，无重复任免 |
| MC07 | 100 年 AI 长跑 | 重大链不并发；无角色线性增长；无事件风暴 |
| MC08 | 任一阶段保存读取 | 阶段、数字、日期与人物安全恢复 |
| MC09 | 选择最高承诺路线 | 有更高代价或阻力，不形成无脑最优解 |
| MC10 | 结局后等待一年 | story、临时 scope、锁与 modifier 全清理 |

## 4. 最终边界

- 不使用 `situation`、activity、scheme 维护全国链；
- 不虚构 `narrative_event`，开场/结局使用主题化 `character_event`；
- 不逐日、逐月或逐角色更新力量，只在入口和固定节点重算；
- 不创建第二政府、阶级人口、市场、工厂、商品、区域生产或财政账本；
- 不覆盖本体、TGP、《变身大明》的公共定义和事件命名空间；
- 首个编码切片只做统一容器和事件链一，通过后再生产其余链。

本阶段完成标准：六链均有入口、阶段、选择代价、数字公式、至少四种结局、继承与死亡路径、AI 边界、素材来源和性能上限；索引、编码入口及测试规范能够反向定位。本阶段不包含实际 Mod 代码。
