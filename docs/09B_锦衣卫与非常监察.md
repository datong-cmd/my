# 09B 《变身大明》锦衣卫与非常监察

> 参考机制解析。新项目只读取厂卫身份和复用表现资源，不复制诏狱、抄家或刑期系统。

## 1. 前置机制

- 锦衣卫指挥使等身份由现有职位和事件维护；
- 皇帝互动可发起指定调查、提审或移交；
- 调查方向、强牵制、口供和判决由事件状态推进；
- 前置还包含诏狱界面、罪名、刑期、平反及停废厂卫的政治反馈。

精确事件、interaction 和 GUI 键用 `02/02A` 反查。

## 2. 可读取与复用

- 当前锦衣卫首脑、司礼监/宦官首脑及其有效性；
- 已验证的职位、关系、入狱和角色状态；
- `theme = jinyiwei` 等已经在前置事件使用的表现键；
- 厂卫作为高恐惧、低程序路径的政治叙事。

## 3. 风险

- 前置调查依赖内部变量和 saved scope，不能从外部直接跳入中段；
- 诏狱 GUI 属于前置系统，复制或覆盖会造成兼容冲突；
- 刑期与判决 flag 存在清理和高频扫描风险；
- 抄家、地契和财产分流不属于本项目范围。

## 4. 新项目边界

- 不复制诏狱窗口、罪名、地契、抄家、刑期和调查数据库；
- 不直接任免前置锦衣卫职位；
- 厂卫只改变重大案件的证据潜力、程序、压力或恐惧；
- 每次最多保存一名厂卫主办，死亡/失位立即替换或匿名退化；
- 任何相关全国效果仍只写 `h_greatming` 八项值。
