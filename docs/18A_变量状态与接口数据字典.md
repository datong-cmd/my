# 变量、状态与接口数据字典

> 文档性质：持久状态、缓存和公共脚本接口的唯一登记表。
>
> 命名规则见 [18_AI编码总规范](18_AI编码总规范.md)。机制含义见 `14`—`17A`。
>
> 约束：未在本表登记的新持久变量不得直接进入代码；应先说明所有者、载体、范围、初始值、写入者、刷新和迁移。

## 1. 数据模型总图

```text
global save
  └─ schema / compatibility

applicable state title（每个适用国家一个）
  ├─ core scheduler
  ├─ C1—C8 nationwide class matrix
  ├─ A1—A6 political alliances
  ├─ P0—P9 historical stage + four institutional layers
  ├─ reform/project slots
  ├─ national aggregate caches
  └─ references to region anchors and active crises

region anchor title（标准 8 个，硬上限 12）
  ├─ eight structural axes
  ├─ dominant production relation
  ├─ derived outputs and dirty flags
  └─ regional crisis/project reference

situation/crisis instance（全国 2 + 区域 2）
  ├─ C0—C6 crisis stage
  ├─ six visible meters
  ├─ type/origin/deadlines
  ├─ participant groups and key actors
  └─ commitments / settlement memory

character（只限相关代理人，常态 20，硬上限 40）
  ├─ material-interest profile
  ├─ political role / organization membership
  └─ promise, betrayal and action memory
```

| `minghm_state_instance_id` | state title | integer | 分配 | core | 初始化 | 调试识别，多国并存不相撞 |
| `minghm_last_month_tick` | state title | integer marker | none | core | 月度调度后 | 防重复，不保存复杂日期文本 |
| `minghm_last_quarter_tick` | state title | integer marker | none | core | 季度调度后 | 年×4+季度 |
| `minghm_last_annual_tick` | state title | year | none | core | 年度调度后 | 同年只结算一次 |
| `minghm_last_five_year_tick` | state title | year | none | core | 五年校准后 | 可错峰而非整十年同步 |

## 4. 国家级基础状态

### 4.1 阶级矩阵

持久键模板：

```text
minghm_class_c<C>_<dimension>
```

例如 `minghm_class_c5_economic_base`、`minghm_class_c8_grievance`。共 `8 × 7 = 56` 个源状态，载体均为 state title，范围 `0..100`，所有者为 `class`。

| 字段族 | 初始来源 | 唯一写入者 | 即时变化 | 年度处理 |
|---|---|---|---|---|
| `economic_base` | 区域结构、本体地产/收入抽样、制度 | `minghm_change_class_dimension_effect` | 政策/战争/没收等事件 | 区域聚合与衰减校准 |
| `organization` | 已有机构、联盟、会馆/军镇代理 | 同上 | 组织、镇压、成功行动 | 记忆衰减和阶段修正 |
| `state_access` | 任命、职位、联盟和国家形式 | 同上 | 任免、制度改革 | 官职结构抽样校准 |
| `coercion` | 军队、家丁、治安与动员 | 同上 | 缴械、募兵、兵变 | 战争状态校准 |
| `legitimacy` | 意识形态、救济、胜败和承诺 | 同上 | 事件和结算 | 向制度相容基线缓慢回归 |
| `cohesion` | 共同敌人、组织和内部分配 | 同上 | 分裂、联盟、背叛 | 按内部矛盾校准 |
| `grievance` | 税租债役、工资、镇压、失约 | 同上 | 冲击即时增加/让步减少 | 生计与分配年度校准 |

缓存：

| 键模板 | 范围 | 公式来源 | 刷新 | 用途 |
|---|---:|---|---|---|
| `minghm_class_c<C>_power_cache` | 0—100 | 六项能力维度加权，不含 grievance | dirty 后季度/年度 | 联盟、谈判、阶段 |
| `minghm_class_c<C>_mobilization_cache` | 0—100 | organization、cohesion、grievance、coercion | 危机/季度 | 集体行动，不等于总力量 |
| `minghm_class_c<C>_trend_cache` | -3—3 | 最近结算方向 | 年度 | UI 箭头和 AI 惯性 |

### 4.2 六联盟状态

键模板 `minghm_alliance_a<A>_*`，载体 state title，所有者 `politics`。

| 后缀 | 类型/范围 | 初始 | 含义 |
|---|---|---:|---|
| `support` | 0—100 | 阶级派生 | 全国可动员支持，不是民调百分比 |
| `organization` | 0—100 | 机构派生 | 行动与传播能力 |
| `cohesion` | 0—100 | 50 附近 | 路线一致度 |
| `state_access` | 0—100 | 职位派生 | 影响国家议程能力 |
| `commitment_load` | 0—100 | 0 | 尚未兑现的政治承诺负担 |
| `route_inertia` | 0—100 | 20—50 | 改变路线的成本 |
| `recent_success` | -100—100 | 0 | 数年衰减的行动记忆 |
| `recent_betrayal` | 0—100 | 0 | 对执政者/盟友的不信任 |
| `leader_valid` | flag/cache | no | 当前关键领袖是否仍有效 |

人物领袖用 saved scope 或 situation participant 维护，不用数值变量存 character ID。领袖失效时从相关人物池按五年/事件驱动替换。

## 5. 区域状态

### 5.1 区域登记

标准开局最多 8 个功能区。每个 region anchor 是稳定 landed title，state title 保存：

```text
minghm_region_count = 8
saved scope: minghm_region_1 ... minghm_region_8
```

若动态 saved scope 的持久性经实机验证不足，则使用 title 标志和低频枚举重建索引；不得退化为每月遍历全部省份。

### 5.2 八轴源状态

每个 region anchor 保存：

```text
minghm_land_concentration
minghm_labor_coercion
minghm_commercialization
minghm_wage_labor
minghm_industrial_capacity
minghm_state_direct_control
minghm_transport_integration
minghm_extraction_pressure
```

全部范围 `0..100`，所有者 `region`，唯一写入者 `minghm_change_region_axis_effect` 和年度结算 effect。

### 5.3 区域其他字段

| 键 | 类型/范围 | 说明 |
|---|---|---|
| `minghm_region_index` | 1—12 | 稳定索引，不因控制者变化重排 |
| `minghm_region_function` | enum | 粮仓、商业沿海、军工、矿冶、边疆等功能标签 |
| `minghm_dominant_relation` | 1—7 | PR1—PR7，只作主导分类 |
| `minghm_population_pressure` | 0—100 | 聚合人口/生计压力，不是人口数 |
| `minghm_food_security` | 0—100 | 生产、运输、灾害和征收结果 |
| `minghm_credit_depth` | 0—100 | 信用与融资可得性 |
| `minghm_urbanization` | 0—100 | 城市定居和非农活动聚合 |
| `minghm_environmental_stress` | 0—100 | 矿冶、水利、灾害与资源约束 |
| `minghm_policy_compliance` | 0—100 | 最近政策执行度缓存 |
| `minghm_region_dirty_flag` | flag | 需要年度/事件后重算 |
| `minghm_last_region_settlement_year` | year | 防重复结算 |
| `minghm_active_project_count` | 0—3 | 区域项目上限检查 |

区域不保存 C1—C8 七维完整矩阵。区域对阶级的影响在年度结算中汇总为全国增量。

## 6. 改革与工业项目槽

首版每国同时最多两个重大改革、三个重大工业/基础设施项目。为避免无界动态对象，首版采用固定槽。

### 6.1 改革槽 `r1/r2`

键模板 `minghm_reform_r<R>_*`：

| 后缀 | 类型 | 说明 |
|---|---|---|
| `active` | flag | 槽位占用 |
| `type` | enum | 改革登记表 ID |
| `stage` | 0—6 | 提案、结盟、试点、扩散、反制、结算、后效 |
| `progress` | 0—100 | 实施进展，不等于成功率 |
| `implementation_capacity` | 0—100 | 财政、官僚、地方和组织综合 |
| `opposition` | 0—100 | 可组织反对，不等于单一阶级不满 |
| `distortion` | 0—100 | 地方转嫁、寻租和执行变形 |
| `budget_committed` | 0—100 abstract | 政治预算/资源承诺；真实金钱另由 adapter 结算 |
| `deadline_marker` | integer | 下次里程碑/失效期限 |
| `sponsor_alliance` | 1—6 | 主要推动联盟 |
| `beneficiary_profile` | enum | 预登记分配模板，不保存列表 |
| `dirty` | flag | 待季度审核 |

### 6.2 项目槽 `p1/p2/p3`

键模板 `minghm_project_p<P>_*`：

| 后缀 | 类型 | 说明 |

| `region_index` | 1—12 | 所在区域 |
| `construction` | 0—100 | 建设进度 |
| `operational_capacity` | 0—100 | 投产后实际能力 |
| `labor_regime` | enum | 身份役、募工、军工、契约、合作等 |
| `capital_source` | enum | 国库、内帑、商资、债务、摊派、混合 |
| `supply_stability` | 0—100 | 原料、能源、运输和订单闭环 |

项目关闭时先结算区域轴、阶级记忆和债务，再清空全部槽字段，不能只移除 `active`。

## 7. 危机实例

### 7.1 载体选择

优先使用原生 situation 实例承载危机和参与组；脚本变量写在该实例可用 scope。此能力必须在首个垂直切片中实机验证。若 `1.19.0.6` 不支持预期的 situation 变量写入，则降级到 state title 固定危机槽，参与组仍由 situation UI 表达。不得临时改用随机人物作为危机载体。

### 7.2 必需字段

### 7.3 承诺槽

每危机最多 6 项活动承诺：

```text
minghm_commitment_q1_type ... q6_type
minghm_commitment_q1_beneficiary ... q6_beneficiary
minghm_commitment_q1_cost ... q6_cost
minghm_commitment_q1_deadline ... q6_deadline
minghm_commitment_q1_status ... q6_status
```

`status` 固定为 `0 proposed / 1 accepted / 2 fulfilled / 3 broken / 4 waived / 5 expired`。承诺兑现或背叛必须写入联盟和相关人物记忆，再清理危机槽。

### 7.4 参与者

使用六种 situation group/聚合角色：

```text
initiator
executor
beneficiary
veto_actor
mediator
burden_bearer
```

每组保存支持强度，不保存群众人物列表。关键人物总量受 `17A` 相关人物上限约束。

## 13. 调用顺序不变量

年度结算固定：

```text
validate state
→ settle active projects/reforms that have annual effects
→ settle each registered region exactly once
→ aggregate national region values
→ update C1—C8 source deltas
→ recalculate class caches
→ recalculate six alliances
→ update seven transition drivers
→ review historical stage
→ decay memories
→ invariant check
→ mark annual tick complete
```

不得把 stage review 放在 class/region 聚合前，也不得在同一年由区域循环内重复重算全国。

## 14. 初始化合同

初始化必须分步且幂等：

1. 检查 DLC、版本、适用政权和主头衔；
2. 若 schema 旧，先迁移；
3. 设置 initialization lock；
4. 登记 8 个区域 anchor；
5. 从低成本本体/头衔条件给八轴设基线；
6. 初始化 C1—C8 七维；
7. 派生联盟、制度和 P0/P1 初始阶段；
8. 清空改革、项目、危机槽；
9. 运行不变量检查；
10. 最后设置 initialized 并清锁。

任何中断都不能留下 `initialized = yes` 的半成品。

## 15. Schema 迁移登记

首版目标：

| schema | 内容 | 迁移行为 |
|---:|---|---|
| 1 | core、8 区域、C1—C8、A1—A6、P0—P9 | 新存档初始化 |
| 2 | 改革/项目固定槽 | 为旧档建立空槽，不重算历史 |
| 3 | 危机与承诺槽、组织记忆 | 活动旧事件按保守规则收口 |

实际开发可调整版本内容，但一旦公开发布不得复用已使用 schema 数字。迁移逐级执行：`1→2→3`，不直接写一个无法测试的 `1→3` 大跳跃。

## 16. 硬不变量

调试 trigger/effect 必须能检查并尽量安全修复：

- 所有标准量在 `0..100`；
- 每国区域数 `1..12`，标准为 8；
- 区域 index 唯一且 anchor 有效；
- 活跃改革 ≤2，项目 ≤3，危机不超过全国/区域上限；
- 当前阶段、制度和枚举值合法；
- 每个活动槽的 type/stage/region 等必填字段存在；
- 空槽不存在遗留 type、预算和 saved scope；
- 同一人物不重复计入同一危机角色；
- 缓存 dirty 时 UI 显示旧缓存加“待更新”，不临时全量重算；
- tick marker 不超前于当前日期；
- initialized 与 schema/区域/阶级基础状态一致；
- state title 失去适用性时进入休眠/收口，不继续高频调度。

## 17. 数据扩展流程

增加新数据项时，提交必须同时修改：

1. 本字典的名称、载体、范围、所有者、初始和清理；
2. schema 迁移；
3. 唯一公共写入 effect；
4. 不变量检查；
5. debug dump；
6. 本地化或 UI（若玩家可见）；
7. 对应功能与存档测试；
8. 最终实现反查索引。

只增加变量而没有清理/迁移/测试视为未完成。
