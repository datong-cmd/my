# MVP 垂直切片：土地清丈、区域执行与财政危机

> MVP 不是缩小版全 Mod，而是一条用于证明底层技术和核心玩法的完整链。
>
> 验证目标：玩家的改革不是按按钮得增益，而是改变信息、征收、地方权力、阶级联盟和未来危机；AI 也能在相同规则下运行。

## 1. 玩家体验目标

一个适用国家在开局几年内出现财政压力和税基不可见问题。玩家可：

- 维持旧有地方中介，以低行政成本换取税基流失；
- 联合官僚推动清丈，承担预算、人才和士绅反制；
- 与地方士绅妥协，只清查特定区域/特定地类；
- 借助宫廷或非常监察强推，快速提高可见度但损害制度信任；
- 以商人垫款和信用融资，换取特许与未来财政依赖；
- 在危机中兑现、修改或背弃承诺。

结局不是“清丈成功/失败”二值，而是：实际入库、土地暴露、基层负担、地方自主、官僚能力、阶级组织和政治记忆的组合。

## 2. MVP 明确包含

### 2.1 技术基础

- `minghm` namespace、schema 1、初始化和迁移空框架；
- state title、8 个 region anchors、固定改革槽和一个危机载体；
- 月/季/年调度与幂等 marker；
- DLC/适用政权检查；
- 原生国家国库 adapter；
- debug dump、validate、repair 和 force scenario；
- 静态索引和 error.log 差分流程。

### 2.2 状态

- 区域八轴全部存在，但本切片重点改变 `land_concentration`、`state_direct_control`、`extraction_pressure`、`commercialization`；
- C1—C8 七维全部有基线，但事件重点操作 C1/C2/C3/C5/C7/C8；
- A1—A6 全部可见，重点是 A1/A2/A3/A4；
- P0、P1、P2、P3、P4 可进入；P5—P9 只保留合法枚举，不在 MVP 触发；
- 一个土地清丈改革类型；
- 一个财政—地方阻力危机类型；
- 承诺类型：地方留成、官僚增俸、产权确认、免追旧欠、商人偿付/特许、基层减负。

### 2.3 玩家入口

- 一个 game rule：启用/禁用 MVP；
- 一个开局介绍事件；
- 一个国家概览 decision/轻量界面；
- 一个“提出清丈”decision；
- 一个选择试点区域和推动联盟的事件；
- 危机 K0—K6 的 situation/action 或降级事件链；
- 一个年度结果摘要；
- 调试模式下的诊断 decision。

### 2.4 AI

- 是否提出清丈；
- 选择试点、支持联盟和融资方式；
- 面对地方阻力时妥协、增资、强制、延期或回撤；
- 危机结算时兑现/背弃承诺；
- 路线惯性和最近成功/背叛记忆；
- 信息受 visibility 和自身 state access 限制。

## 3. MVP 明确不包含

- 完整科举、官职、案件和锦衣卫系统；仅用简化官僚/非常监察选项；
- 完整军队、军工、藩属和边疆；
- 真实人口、商品价格、逐县地亩或逐户税册；
- 所有明代历史事件和历史人物硬绑定；
- P5—P9、革命、内战与复辟；
- 复杂自定义 GUI；原生 situation/decision/event 足够时不扩界面；
- 新 3D 模型、单位模型和大规模原创美术；
- 对参考模组的运行时依赖；
- 多版本、多大型 Mod 的全面兼容。

## 4. 开局基线

### 4.1 区域映射

首版以 8 个功能区而非逐省建模：

| 索引 | 功能区 | 建议初始特征 | 主要验证 |
|---:|---|---|---|
| 1 | 京畿—宫廷中心 | 国家直控高、提取高 | A1/C1、公私财政 |
| 2 | 北方边防—军粮区 | 军役、运输压力 | 财政压力传播 |
| 3 | 运河—漕运走廊 | 运输整合高 | 粮运和中央执行 |
| 4 | 江南财赋—手工业区 | 商品化、土地集中较高 | 清丈收益与士绅反制 |
| 5 | 东南沿海—海贸区 | 商业与信用高 | 商人融资/特许 |
| 6 | 中部粮产—水利区 | 小农与地主租佃混合 | C7 负担转嫁 |
| 7 | 西南土司—山地网络 | 国家直控低 | 政策执行变形上限 |
| 8 | 西北军镇—屯田区 | 军户、强制劳动高 | 强推与逃役 |

实际 landed title 映射在代码阶段按目标地图和“溥天之下”头衔确认；功能和索引稳定，具体边界允许配置化。

### 4.2 数值初始化原则

- 来自本体的低成本信号只用于基线：发展、控制、建筑、政体、原生资源和重要地理标签；
- 不把 development 直接等同工业能力或阶级力量；
- 同类开局加入小幅受控差异，不用全随机；
- 八轴总量不要求守恒；它们表达不同结构维度；
- C1—C8 power 不要求和为 100，各集团可同时强或弱；
- grievance 与 power 分开；弱而不满不等于立即能行动。

## 5. 改革生命周期

MVP 的清丈改革使用七阶段：

| 阶段 | 名称 | 玩家/AI 任务 | 主要状态变化 |
|---:|---|---|---|
| 0 | 提案 | 是否公开议程 | visibility、联盟初始立场 |
| 1 | 结盟 | 选择推动者、资金与承诺 | support、commitment、政治预算 |
| 2 | 试点 | 选一区域和执行者 | progress、capacity、distortion |
| 3 | 扩散 | 扩大、分区或冻结 | 多区域传播、opposition |
| 4 | 反制 | 应对隐田、诉讼、拖延、基层转嫁 | polarization、grievance、memory |
| 5 | 结算 | 统一、妥协、形式化、回撤 | 区域/阶级/财政多维结果 |
| 6 | 后效 | 数年先例、信用与反扑 | stage driver、route inertia |

改革进度不是时间自动上涨。季度变化至少由实施能力、联盟支持、区域难度、反制和已兑现承诺共同决定。

## 6. 推动联盟与不同方案

| 方案 | 主要推动者 | 短期优势 | 必付代价 | 内生风险 |
|---|---|---|---|---|
| 官僚清丈 | A2/C3 | 规则和持续执行 | 预算、增俸、任期保护 | 文书造假、官僚自保、基层刚性 |
| 宫廷强推 | A1/C1 + 非常监察 | 快速、可绕过阻挠 | 内帑/恩宠、合法性 | 抄没寻租、公私混同、恐惧信息 |
| 士绅协商 | A3/C2 | 地方知识与低行政成本 | 留成、免追、产权确认 | 自查自报、负担转嫁、中央收益有限 |
| 商人融资 | A4/C5 | 现金与账簿能力 | 偿付、商税/特许承诺 | 债务、特许垄断、财政被俘获 |
| 基层动员 | A6/C7/C8 | 暴露隐田、提高社会可见度 | 减负、保护、参与承诺 | 地权冲突、扩大化、精英恐慌 |

首版玩家可把一种作为主方案、一种作为辅助联盟。不能把所有优势一次叠满；辅助联盟增加承诺和内部分歧。

## 7. 政策执行公式

概念公式：

```text
implementation =
  sponsor_capacity
  + bureaucratic_capacity
  + information_capacity
  + fulfilled_commitments
  + regional_state_control
  - local_opposition
  - land_concentration_resistance
  - administrative_overload
  - crisis_disruption
  ± bounded_uncertainty
```

`implementation` 控制实际完成多少；`distortion` 控制成本由谁承担和账面与实际偏差。两者不得合为一个成功率。

区域效果分三步：

1. 账面输出：根据改革阶段和政策强度；
2. 执行折损：由国家能力、地区结构和反制决定；
3. 分配变形：由推动联盟、劳动强制、地方权力和承诺决定。

## 8. 危机触发

清丈不会每次必然产生同一危机。满足组合条件时启动“财政—地方阻力危机”：

```text
改革进入试点或扩散
AND（地方反对高 OR 承诺负担高 OR 执行变形高）
AND（财政压力、税基差异、基层负担中至少两项成立）
AND 没有同类型活动危机
AND 冷却结束
```

强推提高危机可见度和极化；隐蔽妥协可能降低可见度但积累严重度。危机不是改革“失败惩罚”，而是把结构矛盾转成可博弈对象。

## 9. 危机 K0—K6 内容

| 阶段 | 最少事件/行动 | 核心选择 |
|---:|---|---|
| K0 压力积累 | 账册差异、地方拖延 | 调查、容忍、加派、暂停 |
| K1 信息暴露 | 隐田/基层摊派证据 | 公开、内部处置、压下消息 |
| K2 议程形成 | 四种解释争夺 | 归罪贪官、制度漏洞、财政紧急、地方权利 |
| K3 资源动员 | 官员、资金、士绅、商人、基层 | 承诺、任命、借款、监察、赦免 |
| K4 公开对抗 | 罢市、抗查、诉讼、请愿或镇压 | 妥协、分区、强制、换执行者 |
| K5 结算 | 统一册籍/地方协议/形式改革/取消 | 多维结果和承诺处理 |
| K6 后效 | 数年摘要与政治记忆 | 进入下一阶段候选、冷却 |

## 10. 关键选项示例

### 10.1 公开账册差异

- 成本：短期合法性风险和极化；
- 支持：A2、基层动员；
- 反对：A3、受牵连官员；
- 收益：visibility、information capacity；
- 风险：若执行能力低，公开问题却无法解决；
- 长期：提高公共财政先例，或制造改革无能记忆。

### 10.2 免追旧欠换取重新登记

- 成本：放弃一次性追缴；
- 支持：地方士绅、部分小业主；
- 反对：激进清丈和财政紧急派；
- 收益：降低 opposition，提高长期税基可见；
- 风险：大户先隐后报、将旧欠转给无保护农户；
- 承诺：产权确认与不追溯，背弃会重创未来合作。

### 10.3 交由非常监察抄核

- 成本：内帑/政治预算、制度合法性；
- 支持：宫廷集权、被排挤官员；
- 反对：士绅与规则官僚；
- 收益：短期 information/coercion；
- 风险：抄没替代税制、机构自利、诬告；
- 长期：C1 state access/coercion 上升，公共国家可能被私产化。

### 10.4 商人垫款

- 成本：利息、未来税收或市场特许；
- 支持：A4/C5、财政急进派；
- 反对：地方既得者、反商舆论；
- 收益：缓解 time pressure；
- 风险：财政信用危机、特许垄断、C5 state access 快速上升。

## 11. 结果矩阵

### 11.1 统一册籍但基层转嫁

- `state_direct_control` 与 `fiscal_capacity` 上升；
- C3 state_access/legitimacy 上升；
- C2 受损但可能通过地方执行保留 organization；
- C7/C8 grievance 上升；
- 未来可进入赋役/民变危机；
- 不是“完美成功”。

### 11.2 地方协商型成功

- 可见税基中度改善；
- 地方执行成本低；
- C2 组织和议价先例增强；
- 中央 fiscal capacity 只有限提高；
- 区域差异持续，未来统一成本上升。

### 11.3 宫廷抄核型成功

- 短期国库和 visibility 大增；
- C1 coercion/state_access 上升；
- C3 规则合法性受损；
- `state_private_capture` 上升；
- 下一任统治者可能无法维持，产生继承断裂。

### 11.4 商人信用型成功

- 现金和账簿能力提高；
- C5 economic_base/state_access 上升；
- 商业化区域扩张；
- 债务和特许承诺积累；
- 后续可能导向商业法权或寡头俘获。

### 11.5 政治失败但留下信息

- 改革回撤；
- visibility 和部分册籍仍保留；
- C2/A3 获得抵抗经验；
- A2/C3 信誉下降但可能总结经验；
- 下一次改革更昂贵，却不从零开始。

### 11.6 强制失败

- 国库损失、基层负担、士绅/官僚抵抗同时上升；
- 危机进入公开对抗；
- 可退入 P4，不在 MVP 进入完整革命；
- 玩家仍能用妥协、换人、赦免和缩小范围收口。

## 12. 阶段变化边界

MVP 只开放：

| 转移 | 组合条件示例 |
|---|---|
| P0→P1 | 财政/土地张力、可见度或外部压力持续 |
| P1→P2 | 至少两个联盟形成互斥改革议程 |
| P2→P3 | 改革试点、执行机构和资源承诺成立 |
| P3→P2 | 试点回撤但国家未进入全面危机 |
| P2/P3→P4 | 国家危机、精英分裂和公开对抗组合成立 |
| P4→P2/P3 | 妥协或能力恢复，危机收口 |

进入 P4 不自动发动革命。P5—P9 需后续里程碑解锁，并在 UI 说明 MVP 边界。

## 13. AI 行为档案

### 13.1 输入

- 统治者性格与生存风险；
- 主/次联盟；
- 自身可见的财政与地方信息；
- 当前路线惯性、承诺和背叛记忆；
- 国库、官僚、信息和强制能力；
- 区域的估计难度，不读隐藏精确 distortion；
- 外部战争/继承压力。

### 13.2 典型但非锁死倾向

| AI 状态 | 倾向 | 可改变条件 |
|---|---|---|
| 高官僚能力、A2 强 | 官僚试点 | 财政枯竭或 C2 过强则寻联盟 |
| 高宫廷控制、时间压力高 | 非常监察强推 | 合法性过低或继承临近则收缩 |
| 地方 C2 强、中央弱 | 士绅协商 | 危机公开化/战争可转强制或商融 |
| C5 信用强、商业区重要 | 商人融资 | default memory 高则拒绝 |
| C7/C8 组织高、合法性低 | 基层揭露/参与 | coercive threat 高可能隐蔽组织 |

AI 每季度只审查活动改革；是否新提案按低频议程检查。

## 15. UI 最小信息

国家概览显示：

- 当前 P 阶段与七驱动趋势；
- 八阶级力量/不满的简化条；
- 六联盟支持与承诺；
- 八区域的 3—4 个关键趋势而非全部数字墙；
- 活动改革和危机；
- 国库、官僚、信息、强制、合法性；
- “本年度数据待结算”标记。

## 16. 数值平衡初始方法

MVP 不追求一次给出最终常数。采用参数组：

| 参数组 | 内容 | 调整依据 |
|---|---|---|
| `baseline` | 八轴、阶级、联盟开局基线 | 不同开局可解释性 |
| `threshold` | 阶段、改革、危机组合阈值 | 触发频率和路线多样性 |
| `delta` | 事件即时变化 | 玩家能感知但不一键翻盘 |
| `decay` | 记忆、压力和组织衰减 | 路线依赖持续时间 |
| `capacity` | 执行公式权重 | 国家能力与政治支持区分 |
| `ai` | 效用、误判、惯性 | AI 路线稳定且能适应 |

所有常数集中为 named scripted values/triggers。每次平衡修改记录观察局结果，不在事件文件散改数字。

## 18. MVP 发布门槛
