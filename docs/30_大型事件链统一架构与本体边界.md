# 30 大型事件链统一架构与本体边界

> 本文冻结六条大事件链的共同技术边界。各链见 `30A–30F`。目标不是复述明史，而是使用 CK3 本体、TGP 和《变身大明》已经证明可用的事件工具，形成可继承、可失败、可清理的长期政治博弈。

## 1. 六条主链

| 类型 | 事件链 | 主要系统 | 典型持续时间 |
|---:|---|---|---:|
| 1 | 新君临朝与三司定局 | 继承、皇权、内外廷、军政支持 | 1–3 年 |
| 2 | 考成新政与朝堂反攻 | 官僚、改革、朋党、监察 | 3–6 年 |
| 3 | 边战失利与军制重整 | 战争结果、军职、军功、军事自主 | 2–5 年 |
| 4 | 格致试制与技术国家 | 改革网络、军备组织、商业政治影响 | 4–8 年 |
| 5 | 奏疏风暴与厂卫大案 | 朋党、都察院、刑部、大理寺、锦衣卫 | 1–4 年 |
| 6 | 民变、双重权力与新秩序 | 民众动员、精英分裂、革命/反革命 | 2–6 年 |

全国同时最多一条；它可包含普通政策、案件或危机，但必须占用 `minghm_major_chain_active` 锁。其他符合条件的主链只设置一次候选 flag，当前链结束后年度检查再决定是否启动。

## 2. 使用的原生边界

| 需求 | CK3 载体 | 核验依据 |
|---|---|---|
| 玩家可见长期链 | `common/story_cycles`、`create_story`、visualization counters | TGP `tgp_story_cycle_*`、EP3 `story_cycle_admin_eunuch` |
| 继承/终止 | `on_owner_death`、`end_story`、继承者重新 `create_story` | `tgp_story_cycle_tai_migrations.txt` |
| 远程奏报 | `type = letter_event` | TGP ministry/faction/interaction events；参考模组 `liuguan_events` |
| 宫廷抉择 | `character_event` | 本体与参考模组常规事件 |
| 重大阶段转折 | 主题化 `character_event` | 复用本体/TGP 背景、主题与音效，不虚构独立事件类型 |
| 后台验证 | `hidden = yes` event | TGP 与参考模组大量现成范式 |
| 延迟 | `trigger_event = { id = ... days/months = ... }` | 本体事件链通用范式 |
| 事实入口 | `on_title_gain`、战争结束、决议、政策/案件结算 effect | 本体 on_action 与前置已有入口 |

不使用活动（activity）模拟廷议，不创建新 scheme，不启动新 situation，不覆盖参考模组事件。

## 3. 权威数据与 story 分工

`title:h_greatming` 保存：

| 字段 | 范围/类型 |
|---|---|
| `minghm_major_chain_active` | flag |
| `minghm_major_chain_type` | 1..6 |
| `minghm_major_chain_stage` | 0..6 |
| `minghm_major_chain_support` | 0..200 |
| `minghm_major_chain_resistance` | 0..200 |
| `minghm_major_chain_urgency` | 0..100 |
| `minghm_major_chain_commitment` | 0..100 |
| `minghm_major_chain_route` | 0..6 |
| `minghm_major_chain_lead/opponent/mediator` | 可空 saved scopes |
| `minghm_major_chain_next_date` | 兜底期限 |

这些是唯一新增的活动临时字段，链结束全部删除。八项长期值、历史阶段、工业组织阶段仍按 `24/25/26A` 保存。

`story_minghm_major_chain` 只给玩家显示：链名、阶段、支持、阻力、紧迫度、主持人和主要反对者。创建语法使用本体已证实的：

```text
create_story = {
  type = story_minghm_major_chain
  save_scope_as = chain_story
}
```

AI 皇帝不必创建可见 story，事件仍从 title 状态推进。story 无随机 `effect_group`；只设 360 天孤儿验证，避免额外频繁轮询。

## 4. 统一事件节奏

- 每链 6–10 个必经事件、最多 4 个条件插曲；
- 两个可见事件间隔通常 90–360 天，不连续弹窗；
- 开场和最终结算最多各一个大画面，中间主要用 character/letter events；
- 每次选择只改 2 个活动值和最多 2 个长期值；
- 随机修正 `0..10`，主要结果由公开数字决定；
- 选项必须显示支持/阻力方向、皇权成本、长期代价和将锁定的承诺。

## 5. 角色来源与表现

按 `25A` 复用：首辅/大学士、六部、左都御史、大理寺、锦衣卫、司礼监、五军职位、品级、朋党和军功人物。群众、士绅联名、商人、军中基层、工匠与灾民默认匿名文本，不生成人物。

每链最多保存主持人、反对者、调停者和 5 名辅助角色。人物死亡或离廷时：优先从对应现有官位补位；无法补位则降低支持/执行力，不凭空生成替代者。

## 6. 继承、失效与清理

- 皇帝死亡：story `end_story`；title 活动状态保留。
- 新 holder 的 `on_title_gain`：若链仍 active，重新创建 story，并触发对应“新君承接旧案”事件；新君可延续、修改承诺或终止并承担反噬。
- `h_greatming` 无 holder：暂停延迟推进，清人物 saved scopes；复国后进入“档案重启”而不补算多年。
- lead/opponent 死亡：事件换人或把人物冲突转为制度冲突。
- 所有成功、失败、撤回、战争失效和超时出口调用 `minghm_end_major_chain_effect`。

## 7. 美术/主题

直接使用参考模组已验证主题：`zouzhang`（奏疏/党争/司法）、`guanwei`/`guanwei_shijian`（任官/考成）、`crown`（继承）、`jinyiwei`（厂卫），以及 TGP/本体 `administrative`、`realm`、`war`。背景复用 `dadian/gongdian/shufang/jiedao/minzhai/chenglou`。不复制资产、不新增 3D。

## 8. AI 共同规则

AI 只比较：预期差值、性格、当前历史阶段、皇权成本、危机紧迫度和长期路线偏好。AI 不读取隐藏未来结果，不为了“历史正确”固定选项。相同局势下不同皇帝可走妥协、强推、镇压或改革，但不能选择不满足资源门槛的行动。

## 9. 共同测试

- 一次仅一条主链；候选不会覆盖活动链；
- 玩家 story 与 title 权威状态一致，AI 无 story 也能完成；
- 继承、死亡、离廷、入狱、大明失主和旧存档均有出口；
- 每链至少一个成功、妥协、失败和反噬结局；
- 100 年观察中主链不死锁、不刷事件、不线性生成人物；
- 非明从不触发；原科举、朋党、弹劾、厂卫和战争事件仍独立运行。
