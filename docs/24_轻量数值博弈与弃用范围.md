# 24 轻量数值博弈与弃用范围

> 本文覆盖旧策划中与施工版冲突的部分。兼容扩展优先保证性能、可解释性和长期博弈，不制作财政、区域生产、市场或人口经济沙盘。

## 1. 硬性删减

下列内容不进入代码：

- 国库、内帑、税种、收支平衡和财政部门模拟；
- 分区域产量、土地、矿业、作坊、物流和商品链；
- 市场价格、资本账户、私人投资和跨区贸易；
- 人口数量、职业人口、迁徙及逐伯爵领社会结构；
- 为上述系统服务的地图循环、人物循环、月度结算和自定义界面。

上述早期研究文档已从 `main` 删除。工业化与商业力量只保留 `26A` 的政治—组织抽象，不实现经济过程。

## 2. 唯一模型：全国级力量对比

所有长期数值只放在 `title:h_greatming`，核心控制在 8 项：

| 数值 | 范围 | 含义 |
|---|---:|---|
| `minghm_state_capacity` | 0..100 | 朝廷把决定贯彻下去的能力 |
| `minghm_reform_momentum` | 0..100 | 改革组织、干部和政治承诺的合力 |
| `minghm_conservative_power` | 0..100 | 既有制度受益者的阻挠能力 |
| `minghm_popular_mobilization` | 0..100 | 城乡民众被组织进政治行动的程度 |
| `minghm_elite_cohesion` | 0..100 | 统治集团协调一致程度 |
| `minghm_military_autonomy` | 0..100 | 军事集团脱离文官—皇帝控制的程度 |
| `minghm_commercial_influence` | 0..100 | 商人及货币关系的政治影响，不代表市场规模 |
| `minghm_crisis_pressure` | 0..100 | 战争、灾害和政治冲突的综合压力 |

皇权继续读取《变身大明》的 `huang_quan_value`，不复制为第九项长期数值。朋党、官位、厂卫只在发生政策或危机时提供临时修正。

## 3. “数字对比”玩法

每项行动只比较两边总值：

```text
推动力 = 基础值 + 皇权投入 + 改革势头 + 支持官署 + 动员 + 0..10随机
阻力   = 政策难度 + 守成力量 + 反对朋党 + 军事/地方阻挠 + 危机干扰
差值   = 推动力 - 阻力
```

| 差值 | 结果 |
|---:|---|
| `≥ 25` | 完全实现，并积累少量制度能力 |
| `5..24` | 妥协实现，部分目标落空 |
| `-15..4` | 纸面通过，执行失败或延期 |
| `< -15` | 反噬，反对力量增长并产生危机事件 |

界面必须展示双方总值和前三项修正，让玩家能够推算风险。随机数只负责打破完全确定，不应盖过力量对比。

## 4. 阶级斗争的抽象办法

不追踪每一阶级的人数和财富。阶级变化表现为“政治影响与组织能力”的此消彼长：

- 政策会让一至两个全国数值上升，同时使至少一个对立数值或风险上升；
- 没有永久最优路线。例如依靠皇权压倒士绅可快速改革，却可能削弱官僚协作并提高军事或内廷自主；
- 商业影响上升只说明其能进入政治交易，不模拟资本、价格或企业；
- 民众动员上升既能推动深层变革，也会降低朝廷完全控制结果的能力；
- 历史阶段由数值组合和重大事件判定，不按年份自动升级。

这仍体现历史唯物主义：政治选择受现存力量关系制约，并反过来重组力量关系；但载体是少量可玩的指标，而非虚构精确经济统计。

## 5. 更新频率与对象上限

- 长期数值：每季度最多结算一次；无活动政策/危机时不运行复杂 effect。
- 政策、重大危机、改革案件和大型事件链共用一个前台内容锁，同一时间只推进一种复杂内容。
- 大型链需要案件或政策阶段时复用自己的临时字段，不另启动第二槽。
- 临时参与者：每项内容最多 8 名关键人物；候选池最多 12 名。
- 朋党输入：最多读取 4 名党魁/代表。
- 禁止 `any_living_character`、全地图伯爵领循环、逐日/月全局脉冲。

预计常态每年仅 4 次全国轻结算；无政策时只做简单条件判断。

## 6. 不做复杂 UI

MVP 使用：

- 决议显示当前八项数值；
- 事件选项显示推动力、阻力、差值和主要修正；
- character modifier 表示临时主持人/反对者；
- 长期链使用事件、决议和可见 story；只读局势概览使用 `29` 的事件 widget，不覆盖公共 GUI。

不制作经济地图模式、仪表盘、可滚动人口表、商品面板或实时曲线。2D 优先复用本体/DLC/参考模组图标；本阶段不需要任何新 3D。

## 7. AI 简化

AI 不做多步预测，只比较：

`预期差值 + 性格/路线偏好 + 危机紧迫度 − 皇权成本 − 失败风险`

低于阈值则妥协或放弃；高于阈值才强推。每年最多评估一次新政策，避免决议频繁轮询。

## 8. 性能验收红线

- 观察模式运行 100 年，脚本错误日志无持续刷屏；
- 开关本模组对非明国家无可见决议、变量和周期性人物处理；
- 无活动政策/危机时，季度 effect 只进行 Ming 存在性和活动 flag 判断；
- 存档中不存在区域经济、商品、人口或逐人物阶级变量；
- 任一新机制若必须突破对象或更新频率上限，先删减规则，不以“以后优化”为由合入。
