# 事件、触发器、效果与本地化模板

> 文档性质：后续 AI 编写 CK3 脚本时使用的结构模板和验收合同。
>
> 注意：代码块表达结构和命名，不替代实机语法验证。凡涉及原生 situation、DLC 国库、功绩、任命等接口，必须经 [13](13_CK3本体与溥天之下技术映射.md) 与本地脚本确认。

## 1. 一条完整机制调用链

```text
本体 on_action / decision / interaction / situation action
→ 低成本资格 trigger
→ 保存稳定 state / region / crisis / actor scopes
→ 玩家事件或 AI 入口
→ option 调公共 can_* trigger 和统一 AI utility
→ 单一事务 effect
→ 原子状态 changes + 本体 adapter 调用
→ dirty / memory / next deadline
→ 通知、tooltip、后续事件
→ 季度/年度调度聚合
→ debug invariant + 测试
```

任何机制说明若只有“事件弹出—加修正”，缺少入口、状态、AI、清理或聚合，都还不能交给代码 AI 直接实现。

## 2. 文件头模板

每个逻辑文件使用：

```text
# MINGHM — <module> — <purpose>
# Target: CK3 1.19.0.6 (Scribe)
# Owner: <module>
# Public interfaces: <list>
# Persistent writes: <list or none>
# Frequency: event/month/quarter/year/five-year/UI
# Evidence: D: docs/<doc>; S1: <verified local file if applicable>
# Compatibility: additive file; no vanilla path replacement
```

事件文件第一条有效定义前声明一次：

```text
namespace = minghm
```

不得在同一文件中重复 namespace，也不得让多个模块混用一个无主题的 `minghm_events.txt`。

## 3. Scripted trigger 模板

### 3.1 公共资格 trigger

结构示意：

```text
minghm_can_start_land_survey_reform_trigger = {
    # Root: state title
    # Reads: initialized, reform slots, fiscal/bureaucratic capacity, stage
    # Writes: none
    minghm_is_initialized_trigger = yes
    NOT = { has_title_flag = minghm_reform_operation_lock_flag }
    minghm_has_empty_reform_slot_trigger = yes
    minghm_is_reform_type_active_trigger = {
        REFORM_TYPE = land_survey
        EXPECTED = no
    }
    minghm_meets_land_survey_minimum_trigger = yes
}
```

要求：

- 根 scope 和读取项在注释中明确；
- trigger 内没有 effect、副作用或随机；
- 高频 trigger 只读缓存或固定对象；
- 复杂失败理由拆分为命名子 trigger，供 tooltip 复用；
- 对参数值的拼接/替换语法从本体或参考模组已运行例子复制模式，不能凭空发明；
- `NOT`、`NOR` 等块保持小而可读，避免三层以上嵌套。

### 3.2 显示和可用性分离

```text
is_shown = {
    minghm_is_applicable_state_trigger = yes
}

is_valid = {
    minghm_can_start_land_survey_reform_trigger = yes
}
```

- `is_shown` 决定玩家是否应知道入口；
- `is_valid` 决定现在能否执行；
- `is_valid_showing_failures_only` 或等价位置显示具体缺口；
- 不因资源不足永久隐藏重要机制。

### 3.3 阈值 trigger

阈值集中命名：

```text
minghm_has_adequate_bureaucratic_capacity_trigger
minghm_has_high_land_concentration_trigger
minghm_crisis_is_public_trigger
```

不要在 20 个事件里分别写 `compare_variable >= 60`。若平衡调整，修改单一 trigger/value 即可。

## 4. Scripted value 模板

### 4.1 单一分项

```text
minghm_land_survey_implementation_value = {
    # Root: state title
    # Output: 0..100
    # Reads only cached national values plus one registered region
    value = 0
    # Add named components here.
    min = 0
    max = 100
}
```

公式必须在头部注释列出策划来源。脚本内使用命名分项，而不是无法解释的一串常数。

### 4.2 AI 选项效用

```text
minghm_ai_option_utility_value = {
    # Root: acting character or organization leader
    # Parameters: OPTION_PROFILE, STATE, optional CRISIS
    # Output: bounded relative score, not a probability
    value = 0
    # material_gain
    # institutional_goal
    # survival_security
    # legitimacy_effect
    # ally_commitment
    # route_consistency
    # minus resource_cost / risk / betrayal / uncertainty
}
```

最后由 `ai_chance/ai_will_do` 的本体合法语法读取。随机误判应是小幅 bounded random 或信息不确定惩罚，不直接把效用替换为骰子。

### 4.3 性能限制

禁止 value：

- 每次被 GUI 调用就遍历所有 vassal、courtier、county 或 world character；
- 在循环内再次调用包含同类循环的 value；
- 为得到一个显示值临时重算八区域、八阶级和六联盟；
- 依赖未设变量而静默按 0 处理。

## 5. 原子 effect 模板

### 5.1 数值变更

```text
minghm_change_region_axis_effect = {
    # Root: registered region title
    # Params: AXIS, AMOUNT, REASON
    # Preconditions: valid anchor, valid axis, initialized state
    # Writes: one source axis; region/state dirty
    # Failure: no state change; debug warning in debug mode

    if = {
        limit = { minghm_is_valid_region_anchor_trigger = yes }
        # Change the selected registered field.
        # Clamp 0..100.
        add_title_flag = minghm_region_dirty_flag
        # Request national aggregate refresh through core interface.
    }
}
```

只有此 effect 可直接写区域八轴。不同政策以参数和原因码调用，不各自复制 clamp 与 dirty 逻辑。

### 5.2 复合事务

```text
minghm_start_land_survey_reform_effect = {
    # Root: state title
    # Params: SPONSOR_ALLIANCE, optional PILOT_REGION
    # Transaction: validate → lock → allocate → cost → memory → unlock
    if = {
        limit = { minghm_can_start_land_survey_reform_trigger = yes }
        add_title_flag = minghm_reform_operation_lock_flag

        minghm_allocate_reform_slot_effect = {
            REFORM_TYPE = land_survey
            SPONSOR = $SPONSOR_ALLIANCE$
        }
        minghm_add_state_treasury_effect = {
            AMOUNT = <negative named value>
            REASON = land_survey_startup
        }
        minghm_record_reform_commitments_effect = { ... }
        minghm_request_politics_refresh_effect = yes

        remove_title_flag = minghm_reform_operation_lock_flag
    }
    else = {
        # No partial cost. Debug-only failure record.
    }
}
```

如果 CK3 参数替换不允许把 value 直接作为该 effect 参数，使用本体确认的临时 scope value/变量模式；不要保留 `<...>` 占位符进入发布代码。

### 5.3 清理 effect

每个动态槽必须有单独清理接口：

```text
minghm_clear_reform_slot_effect
minghm_clear_project_slot_effect
minghm_close_crisis_effect
minghm_remove_relevant_actor_effect
```

清理顺序：结算长期后果 → 解除 modifier/参与组 → 清 saved scopes → 清全部字段 → 清 active/lock → 请求 UI/缓存刷新。只移除 active flag 会制造幽灵状态。

## 6. On-action 模板

### 6.1 本体入口只做调度

```text
<verified_vanilla_on_action> = {
    on_actions = {
        minghm_<module>_<entry>
    }
}

minghm_<module>_<entry> = {
    trigger = {
        minghm_is_applicable_state_trigger = yes
    }
    effect = {
        # Resolve stable state title.
        # Call one public scheduler/event entry.
    }
}
```

采用 additive on_action 文件和本体允许的合并结构；实际键名必须从本体 `game/common/on_action` 或参考模组已确认入口复制。若同名文件/定义会覆盖，改用已确认的嵌套注册方式，而不是直接覆盖。

### 6.2 调度防重复

```text
if = {
    limit = { minghm_can_run_annual_tick_trigger = { YEAR = <current> } }
    minghm_run_annual_tick_effect = { YEAR = <current> }
}
```

标记在完整成功后写入。若中途失败，不把年度 tick 伪装成完成；恢复工具可安全重跑幂等步骤。

## 7. 事件登记表模板

每个事件先登记：

| 字段 | 必填内容 |
|---|---|
| ID | `minghm.<号段内整数>` |
| 文件 | 相对路径 |
| 类型 | character/title/situation；popup/hidden |
| 根 scope | 精确 scope 类型 |
| 入口 | on_action/decision/effect/前置事件 |
| 资格 | 公共 trigger |
| saved scopes | 名称、类型、失效替代 |
| 幂等锁 | 键和设置/清除位置 |
| 冷却 | 人物/国家/危机何处保存 |
| options | 选项 ID 与事务 effect |
| AI | 使用的 option profile/utility |
| 后续 | 事件 ID、延迟、取消条件 |
| 状态写入 | 只列公共 effects |
| 本地化 | title/desc/options/tooltips |
| 测试 | 测试用例 ID |

示例：

| 字段 | 示例 |
|---|---|
| ID | `minghm.1801` |
| 文件 | `events/minghm_reform_events.txt` |
| 类型 | character popup |
| 根 scope | ruler character |
| 入口 | 清丈 decision |
| 资格 | `minghm_can_start_land_survey_reform_trigger` on saved state |
| saved scopes | `minghm_state_title`、可选 `minghm_pilot_region` |
| 幂等锁 | `minghm_reform_operation_lock_flag` |
| options | A 正式试点；B 扩大联盟；C 暂缓但付政治代价 |
| AI | `land_survey_pilot` / `land_survey_coalition` / `defer_reform` |
| 测试 | `REFORM-LS-001..006` |

## 8. 玩家事件模板

结构示意：

```text
minghm.1801 = {
    type = character_event
    title = minghm.1801.t
    desc = minghm.1801.desc
    theme = <verified theme>

    # Root: ruler character
    # Saved: minghm_state_title, optional minghm_pilot_region
    # Entry: decision/effect only; never fires by raw monthly random pulse

    trigger = {
        exists = scope:minghm_state_title
        scope:minghm_state_title = {
            minghm_can_start_land_survey_reform_trigger = yes
        }
    }

    immediate = {
        # Revalidate volatile scopes; select bounded fallback if pilot region invalid.
    }

    option = {
        name = minghm.1801.a
        trigger = { ... }
        custom_tooltip = minghm.1801.a.tt
        scope:minghm_state_title = {
            minghm_start_land_survey_reform_effect = { ... }
        }
        ai_chance = {
            # Read shared utility/profile; no hidden omniscience.
        }
    }

    option = {
        name = minghm.1801.b
        # Alternative coalition, different real cost and beneficiaries.
    }

    option = {
        name = minghm.1801.c
        # Deferral is playable but has credibility/time consequences.
    }
}
```

发布代码不保留省略号。模板只是结构，代码 AI 必须展开为已登记接口。

## 9. 隐藏事件模板

隐藏事件仅用于：

- 延迟结算；
- scope 转换；
- 教程状态；
- 无需玩家选择的已承诺后果。

不得用隐藏事件掩盖可影响博弈的重大选择。

```text
minghm.<id> = {
    hidden = yes
    # Root and entry documented.
    trigger = { ... }
    immediate = {
        if = {
            limit = { <idempotent precondition> }
            <one public effect>
        }
    }
}
```

每个隐藏事件同样登记调用者、重复锁和清理。

## 10. 危机事件模板

### 10.1 阶段入口

通用 crisis effect 先迁移 K0—K6，再触发类型叙事事件。叙事事件不得自行设置 stage。

```text
minghm_advance_crisis_stage_effect = {
    TARGET_STAGE = 2
    REASON = accounts_public
}
trigger_event = minghm.<registered crisis narrative event>
```

### 10.2 危机选项合同

每个选项至少声明：

| 项目 | 要求 |
|---|---|
| 资源 | 钱、职位、合法性、时间、军队、信息、组织或承诺中的实际成本 |
| 受益者 | 至少一个阶级/联盟/区域 |
| 承担者 | 谁付税、失地、失权、失业、伤亡或风险 |
| 支持者 | 为什么支持，依据何种可见信息 |
| 反对者 | 有什么反制，而非只减 opinion |
| 信息 | 哪些是已知，哪些是估计 |
| 执行 | 通过哪个公共 effect，如何受能力/区域变形影响 |
| 升级 | 对 severity/visibility/polarization/time 的方向 |
| 承诺 | 是否新增、兑现或背叛 q1—q6 |
| 长期 | 对阶级、制度、阶段和记忆的影响方向 |

### 10.3 危机恢复

关键人物死亡或 scope 失效时：

1. 若角色是可替代执行者，从已登记相关人物池选取；
2. 若是不可替代的谈判方，降低谈判连续性并回到合法前一阶段；
3. 若主国家不再存在，按 successor/fragmentation 规则结算；
4. 若 situation 载体损坏，调用 `minghm_recover_invalid_crisis_effect`；
5. 恢复不能给玩家双重奖励或逃掉已付成本。

## 11. Decision 模板

decision 是机制入口，不是结算容器：

```text
minghm_propose_land_survey_decision = {
    picture = <reused verified asset>
    desc = minghm_propose_land_survey_decision_desc
    selection_tooltip = minghm_propose_land_survey_decision_tooltip

    is_shown = { minghm_is_applicable_ruler_trigger = yes }
    is_valid_showing_failures_only = {
        scope:minghm_state_title = {
            minghm_can_start_land_survey_reform_trigger = yes
        }
    }
    effect = {
        # Save state and open event. No direct multi-variable settlement.
    }
    ai_check_interval = <low frequency verified format>
    ai_potential = { ... }
    ai_will_do = { ... }
}
```

AI check 频率服从 `17A`；重大改革不允许 daily/weekly 轮询。

## 12. Character interaction 模板

interaction 适合任命、拉拢、承诺、调查和组织关系，不适合全国年度计算。

必须区分：

- 发起者资格；
- 目标资格与候选池；
- 可接受度/抵抗度；
- 花费；
- 成功、拒绝、揭露和反制；
- AI 是否知道相关事实；
- 冷却与重复锁；
- 死亡/离职时清理。

同一人物不能同时占用互斥关键角色；用公共 role trigger 验证。

## 13. Situation action 与界面

优先复用原生 situation 的参与组、阶段和 action 展示。自定义 GUI 只补充：

- 六个危机仪表；
- 阶级/联盟摘要；
- 区域结构趋势；
- 改革承诺和执行变形；
- 项目所有制、资金、劳动和供给闭环。

GUI 点击只触发 decision/interaction/event/public effect，不直接写变量。每帧显示只读缓存，不重算全国。

## 14. 本地化键规范

### 14.1 事件

```text
minghm.1801.t
minghm.1801.desc
minghm.1801.a
minghm.1801.a.tt
minghm.1801.b
minghm.1801.b.tt
minghm.1801.c
minghm.1801.c.tt
```

动态描述用：

```text
minghm.1801.desc_default
minghm.1801.desc_high_opposition
minghm.1801.desc_low_capacity
```

不要把完整条件塞进一个难以维护的超长字符串。

### 14.2 数据标签

```text
minghm_class_c1_name
minghm_class_dimension_economic_base
minghm_alliance_a4_name
minghm_stage_p3_name
minghm_stage_p3_desc
minghm_crisis_stage_k4_name
minghm_region_axis_land_concentration
minghm_relation_pr6_name
```

### 14.3 Decision/interaction/modifier

沿对象数据库键使用 CK3 约定后缀，具体后缀从本体同类型文件确认：

```text
minghm_propose_land_survey_decision
minghm_propose_land_survey_decision_desc
minghm_propose_land_survey_decision_tooltip
minghm_propose_land_survey_decision_confirm

minghm_reform_pilot_modifier
minghm_reform_pilot_modifier_desc
```

### 14.4 语言要求

- 选项正文先写政治行动，不先写数值，例如“以漕粮和官俸换取地方配合”。
- tooltip 再列实际成本与影响方向。
- 不把“阶级”机械塞入所有角色对白；人物用其时代语言表达，系统面板解释结构关系。
- “生产力发展”必须具体为矿冶、运输、技术、组织、能源、熟练劳动或市场能力。
- “进步/反动”不是永久阵营标签，应说明在当前矛盾中的作用和代价。
- 未知数值显示范围、趋势或“情报不足”，不显示虚假精确值。

## 15. Tooltip 模板

一个重大选项推荐显示：

```text
即时成本：<国库/政治预算/时间/合法性>
主要支持：<联盟与理由>
主要反对：<联盟与可用反制>
执行条件：<官僚/财政/区域/组织>
已知风险：<执行变形、军头化、资本俘获、劳动冲突等>
承诺：<对象、内容、期限>
可能方向：<危机/改革/阶段趋势，不承诺随机结果>
```

短选项可以省略不相关项，但不能隐瞒确定成本。

## 16. 本地化文件模板

```yaml
l_simp_chinese:
 minghm.1801.t: "丈量天下"
 minghm.1801.desc: "<完成的叙事文本>"
 minghm.1801.a: "先从可控之地试行"
 minghm.1801.a.tt: "<成本、支持、反对和执行风险>"
```

要求：

- 文件名 `minghm_<module>_l_simp_chinese.yml`；
- 首行语言标识正确；
- UTF-8 with BOM；
- 冒号、引号、转义遵循本体范例；
- 占位符、scope 函数、颜色和图标代码从本体已验证用法复制；
- 静态检查重复 key、缺 key、孤儿 key和非法字符。

## 17. 事件文本资料边界

史料和小说只提供：

- 冲突主体；
- 可争夺资源；
- 制度限制；
- 行动语言和政治想象；
- 可能的妥协、反制和意外后果。

不得长段复制史料、论文或小说文本。角色对白与描述应重新创作，并允许非历史人物在相同结构条件下触发。

## 18. Debug 接口模板

每个模块至少提供：

```text
minghm_debug_dump_<module>_effect
minghm_debug_validate_<module>_trigger
minghm_debug_repair_<module>_effect
minghm_debug_force_<mechanism>_effect   # debug game rule only
```

dump 输出：state instance、schema、活动槽、关键值、dirty、最后 tick、相关 scope 是否有效。正常玩家不开 debug 时不得写高频日志。

修复 effect 采用保守原则：clamp 越界、清幽灵槽、重建缓存；不凭空补偿资源、不自动宣布改革成功。

## 19. 自动检查建议

代码仓库至少建立以下只读/测试脚本：

| 检查 | 阻断条件 |
|---|---|
| 事件 ID 扫描 | 重复、跨号段、无 namespace |
| 前缀扫描 | 新数据库键无 `minghm_` |
| 定义/调用索引 | 未定义调用、孤立关键接口 |
| 本地化对照 | 缺失、重复、非法语言头 |
| 变量写入审计 | 非所有者直接写入持久状态 |
| 同路径覆盖对照 | 允许清单外碰撞 |
| 高频遍历扫描 | month/UI 中出现禁用全量 iterator |
| 占位符扫描 | `<...>`、`BLOCKED`、测试哨兵进入发布包 |
| error.log 差分 | 新解析错误、无效 scope、未定义键 |

静态扫描不能证明脚本可运行，仍需 `17A` 的实机与百年观察局。

## 20. 测试用例模板

```text
ID: REFORM-LS-001
机制: 土地清丈改革启动
前置: schema 1+, 适用国家, 两个改革槽为空, 国库/官僚能力达到登记阈值
动作: 玩家选择试点区域并确认
期望即时:
  - 仅一个 reform slot active
  - 原生国库扣除一次
  - sponsor/承诺/期限完整
  - 区域轴尚未被直接宣布为最终变化
期望季度:
  - progress/opposition/distortion 按当前能力更新一次
  - AI 参与者按有限信息行动
期望年度:
  - 试点区域执行变形进入八轴
```

3. 新增/读取/写入变量清单；
4. schema 与迁移影响；
